# AXIOM MODULE SPECIFICATION: TOKEN V5.2

**Статус:** Актуальная спецификация (core)  
**Версия:** 5.3.0  
**Дата:** 2026-05-13  
**Формат:** 64 байта, `repr(C, align(64))`  
**Модель времени:** COM `event_id` (причинный порядок, u64)  
**Связанные спеки:** COM, Connection V5.0, Domain V1.2, UPO v2.1

---

## 1. Назначение

**Token** — голографическая частица смысла в поле Домена, представляющая собой "активный атом" семантической сети:

- хранит ссылку на вечный смысл (`sutra_id`) и текущее положение в домене,
- обладает локальной физикой (position, velocity, target) для взаимодействия в поле,
- имеет термодинамические свойства (valence, mass, temperature) для динамики,
- хранит историю пути (`lineage_hash`) для фрактальной навигации,
- использует инерцию (`momentum`) для предсказуемого движения.

Token **не хранит wall-clock время**. Любая "временная" динамика выражается через **COM event_id**.

## 2. Структура (64 байта)

```rust
#[repr(C, align(64))]
pub struct Token {
    // --- ИДЕНТИФИКАЦИЯ (8 Байт) ---
    pub sutra_id: u32,       // Глобальный ID смысла ("Яблоко")
    pub domain_id: u16,      // Текущий домен существования
    pub type_flags: u16,     // Тип сущности (Object, Relation, Pattern...)

    // --- ЛОКАЛЬНАЯ ФИЗИКА ПОЛЯ (16 Байт) ---
    pub position: [i16; 3],  // Координаты в текущем Field (X, Y, Z)
    pub velocity: [i16; 3],  // Вектор скорости (инерция движения)
    pub target: [i16; 3],    // Целевая точка (аттрактор)
    pub origin: u16,         // Происхождение токена (см. константы TOKEN_ORIGIN_*)

    // --- ТЕРМОДИНАМИКА (4 Байта) ---
    pub valence: i8,         // Эмоциональный заряд (-127..+127)
    pub mass: u8,            // Гравитационный вес (1..255)
    pub temperature: u8,     // Пластичность/мутируемость (0..255)
    pub state: u8,           // Состояние (Active, Sleeping, Locked...)

    // --- ФРАКТАЛЬНАЯ НАВИГАЦИЯ (32 Байта) ---
    pub lineage_hash: u64,    // Хеш пути: откуда пришел (Sutra -> Logic -> Math...)
    pub momentum: [i32; 3],   // Инерция по осям (накопленный импульс)
    pub resonance: u32,       // Резонанс с другими токенами (частота)
    pub last_event_id: u64,   // Последний COM event_id обновления
}
```

## 3. Семантика полей

### 3.1 Идентификация
- **sutra_id**: Уникальный идентификатор вечного смысла в Библиотеке Сутр
- **domain_id**: ID текущего домена существования токена
- **type_flags**: Битовая маска типа и поведения токена

### 3.2 Локальная физика
- **position**: Текущие координаты в 3D пространстве поля домена
- **velocity**: Вектор скорости, определяющий инерционное движение
- **target**: Точка притяжения, к которой стремится токен
- **origin**: Происхождение токена. Кодировка:

  | Значение | Константа | Семантика |
  |---|---|---|
  | `0x0000` | `TOKEN_ORIGIN_LOCAL` | Рождён в SUTRA текущего уровня |
  | `0x0001..=0x00FF` | — | Пришёл с уровня N FractalChain (MAYA[N] → SUTRA[N+1]) |
  | `0xFE00` | `TOKEN_ORIGIN_PERSISTED` | Восстановлен из persistence |
  | `0xFF00..=0xFFFF` | `TOKEN_ORIGIN_EXTERNAL_BASE \| id` | Внешний источник (будущий multi-instance) |

  По умолчанию `0` (`TOKEN_ORIGIN_LOCAL`) — безопасно для всех существующих токенов.

### 3.3 Термодинамика
- **valence**: Эмоциональная окраска (-127 враждебность, +127 симпатия)
- **mass**: Влияние на гравитацию в поле (тяжелые тонут)
- **temperature**: Способность к мутации (горячие быстрее меняются)
- **state**: Текущее состояние активности

### 3.4 Фрактальная навигация
- **lineage_hash**: Значение зависит от `type_flags`:
  - Если `(type_flags & TOKEN_FLAG_FRAME_ANCHOR) == 0` *(обычный токен)*:
    Хэш истории пути домен-по-домену. Обновляется при прохождении ASHTI-pipeline
    (`apply_causal`, Role 6) и при MAYA-консолидации (XOR участников). Default = 0.
  - Если `(type_flags & TOKEN_FLAG_FRAME_ANCHOR) != 0` *(Frame-анкер)*:
    FNV-1a хэш от sorted `sutra_id` всех участников Frame. Устанавливается при
    кристаллизации, не изменяется. Используется для дедупликации
    (`find_existing_anchor`) и деривации ID при промоции в SUTRA. Не несёт
    информации о пути.
  > Смешение интерпретаций по классам токенов недопустимо. См. эррату
  > `docs/spec/Weaver/erratas/Token_lineage_hash_duality.md`.
- **momentum**: Накопленный импульс для предсказания движения
- **resonance**: Частота резонанса с другими токенами
- **last_event_id**: Последний event_id из COM для синхронизации

## 4. Инварианты

1. **Размер**: Строго 64 байта с выравниванием 64
2. **sutra_id**: Всегда > 0 (0 зарезервирован для пустых токенов)
3. **domain_id**: Всегда > 0 (0 зарезервирован для системных доменов)
4. **last_event_id**: Монотонно возрастает согласно COM
5. **mass**: Всегда > 0 (нулевая масса не допускается)
6. **temperature**: 0..255 (абсолютный максимум пластичности)
7. **lineage_hash дуальность**: значение поля определяется классом токена через
   `type_flags`. Для Frame-анкеров (`TOKEN_FLAG_FRAME_ANCHOR`) — идентификатор
   паттерна (FNV-1a от участников). Для остальных — провенанс пути (или 0).
   Frame-анкеры физически не проходят через ASHTI-pipeline, поэтому конфликта
   нет. Читать `lineage_hash` Frame-анкера как путь — ошибка интерпретации.

## 5. Взаимодействия

### 5.1 С Connection
- Token является источником или целью для Connection
- Позиция влияет на distance/ideal_dist в Connection
- Mass влияет на stress в Connection через gates

### 5.2 С Domain
- Token существует в рамках поля домена
- Position ограничена размерами поля домена
- Domain.config влияет на термодинамические параметры

### 5.3 С COM
- Все изменения токена генерируют COM события
- last_event_id синхронизируется с COM timeline
- Momentum обновляется через COM event sequence

---

## 6. Жизненный цикл

1. **Создание**: Token создается с начальным position и sutra_id
2. **Активация**: Устанавливается state = Active, генерируется COM событие
3. **Движение**: Обновляется position/velocity через физику поля
4. **Взаимодействие**: Обновляется через Connection gates и резонансы
5. **Трансформация**: Изменение temperature/mass через термодинамику
6. **Архивация**: Состояние замораживается, lineage_hash сохраняется

---

## 7. Валидация

```rust
fn validate_token(token: &Token) -> bool {
    token.sutra_id > 0 
    && token.domain_id > 0
    && token.mass > 0
    && token.position.iter().all(|&p| p >= i16::MIN && p <= i16::MAX)
    && token.last_event_id > 0
}
```
