**Статус:** Спецификация
**Версия:** 1.0

**Категория:** Над-доменный модуль особой природы (только чтение + выражение, никогда не управление)

## 0. Что это и откуда взялось

Sensorium (Сенсориум) — **полный актуальный срез внутреннего состояния системы**, единая точка доступа для всех потребителей: Workstation, Companion, будущая внутренняя нейронка.

Метафора рождения (из обсуждения chrnv + Opus): система — это пруд, который прекрасно чувствует каждый камень (вход), но **не имеет голоса** — наружу выходит только "узор сложился", отчёт прибора, не выражение. Sensorium — это **берег**, через который внутреннее выходит наружу. Не отчёт о входе — взгляд внутрь и выражение увиденного.

Ключевое уточнение chrnv: это не "модуль настроения" на семь полей. Это **полный срез всего внутреннего** + вместе с EXPERIENCE = максимальный стандартный выход для любого интерфейса.

Имя: sensorium (лат.) — место где сходятся все ощущения в единый срез. У Аристотеля — sensus communis, общее чувствилище. Ровно это и делает модуль: все хранилища сходятся в одну точку.

## 1. Природа: чем Sensorium отличается от остальных модулей

Шесть существующих над-доменных модулей (FrameWeaver, AxialEvaluator, ContextRecognizer, NeuralAdvisor, OverDomainArbiter, DreamPhase) — все **что-то делают** с состоянием: кристаллизуют, оценивают, детектируют, направляют.

Sensorium **только смотрит и выражает**. Это его определяющее свойство.

```
// ПРИНЦИП (в GENOME, вечно):
// Sensorium читает — никогда не управляет.
// Видит. Говорит. Не трогает.
// Как только начнёт переключать/вызывать/гасить — перестаёт быть Sensorium.
```

Аналогия, которую дал chrnv: это как Experience-октант (домен памяти, пассивный) — **но с более широкими правами чтения** (читает не только EXPERIENCE, а все хранилища поперёк) **и как внутренний API** (точка доступа, которую опрашивают и которая сама подаёт сигнал).

## 1а. Архитектурное место: `&AxiomEngine`, строго `&self`

Это единственное архитектурное отличие Sensorium от остальных OD-компонентов: он получает **`&AxiomEngine`** вместо `&AshtiCore`.

Почему это возможно и правильно: все OD-компоненты уже `pub`-поля AxiomEngine (context_recognizer, axial_evaluator, frame_weaver, neural_advisor, dream_phase, over_domain_arbiter). EXPERIENCE — часть AshtiCore, тоже доступна. Sensorium читает всё через одну shared ссылку.

```
// Вызов из основного цикла engine:
//   sensorium.collect(tick, &self)   ← &self, не &mut self
//
// ВАЖНО: строго &self (неизменяемая), НИКОГДА &mut.
// Это не только принцип — это ГАРАНТИЯ КОМПИЛЯТОРА.
// Sensorium физически не может изменить состояние — у него нет &mut.
// GENOME говорит "только чтение" словами; тип-система впечатывает это в код.
```

Транспортный канал наружу — существующий **BroadcastHandle** (SSE/WS). Его не меняем, используем как есть.

## 2. Что видит — полный срез

Sensorium собирает срез из ВСЕХ хранилищ. Не выборку — всё. Структура SensoriumState (схематично, по группам):

```
// — ВОСПРИЯТИЕ СЕЙЧАС —
// active_subsystems    : что распознано (writing/math/.../morality), с весами
// meta_mode            : анализ / синтез / рефлексия / восприятие / ...
// modality             : text / vision / ... что на входе сейчас
// fractal_levels       : на каких окнах (Symbol..Session) активность

// — ОЦЕНКА СЕЙЧАС (AxialEvaluator) —
// octant_profile       : весь профиль 8 октантов, не только доминанта
// axial_scores         : X (аполлон/дионис), Y (эрос/танатос), Z (воля/ничто)
// corpus_callosum      : расходятся ли анализ и синтез (есть/нет + сила)

// — НАПРЯЖЕНИЯ СЕЙЧАС —
// active_dilemmas      : все горящие, с уровнями (0..5) и tension_score
// connection_stress    : карта связей под напряжением (current_stress)

// — ГЛУБИНА И ПАМЯТЬ (SutraDepth + EXPERIENCE) —
// depth_per_octant     : вся стратификация — что укоренилось, что свежее
// depth_hotspots       : где сильно резонирует
// recent_crystals      : что недавно кристаллизовалось
// experience_active    : что из памяти сейчас активно
// composite_suspects   : что нащупывается (Calculus, Rhythm, Geometry...)
// cross_modal_bonds    : что с чем связалось между модальностями

// — СОСТОЯНИЕ ОРГАНИЗМА —
// fatigue              : усталость по факторам (4 компонента)
// phase                : wake / falling_asleep / dreaming / waking
// causal_time          : event_id, причинный возраст
// subsystem_fatigue    : где система "устала" от режима

// — ВНУТРЕННИЙ ИМПУЛЬС (зарезервировано под ВОЛНЫ, см. §9) —
// internal_drive       : что тянет изнутри          [пусто в V1, ждёт волн]
// unfinished           : незавершённое, свербящее    [пусто в V1]
// curiosity_targets    : к чему любопытство           [пусто в V1]

// — ЭМЕРДЖЕНТНОЕ —
// emergent_candidates  : что система предлагает создать
```

Список не финальный — это **масштаб**. Принцип chrnv: "добавляй новый контакт, всё остаётся в одном месте". Новое поле просто встаёт в свою группу.

```
// ВАЖНО: SensoriumState — РОДНОЙ внутренний формат. Бинарный, дешёвый.
// Sensorium НЕ знает про JSON/REST/postcard. Перевод — адаптеры (§7).
// Это тот же принцип что в ядре: родной формат внутри, переводы на границе.
```

## 3. Градиент глубины — уровни доступа

Полный срез **по структуре**, но отдаётся **слоями глубины**. Чем глубже — тем дороже собрать и тем больше прав нужно. Решает разом цену (не собираем всё всегда) и безопасность (не всё торчит наружу).

```
// УРОВЕНЬ 0 — ПУЛЬС (дёшево, всем, каждый тик)
//   phase, доминантный октант, fatigue, есть-ли-дилемма (да/нет)
//   ~5 полей. Копейки. → мандала Workstation, цвет настроения Companion
//
//   РЕАЛИЗАЦИОННАЯ ОСНОВА: уровень 0 = существующий TickSnapshot.
//   TickSnapshot уже собирает именно эти поля и шлёт каждый тик
//   через BroadcastHandle. Он и есть Sensorium уровень 0 до того
//   как мы его осмыслили — наполовину готов, надо переоформить, не создавать заново.
//   Мандала и OBS-метрики продолжают работать без изменений.

// УРОВЕНЬ 1 — СОСТОЯНИЕ (средне, по праву, реже)
//   активные подсистемы, мета-режим, горящие дилеммы, hotspots
//   → панели Workstation, голос Companion "меня гложет X"

// УРОВЕНЬ 2 — ПОЛНЫЙ СРЕЗ (дорого, доверенным, редко)
//   вообще всё из §2
//   → внутренняя нейронка, глубокая диагностика

// УРОВЕНЬ 3 — СРЕЗ + ПАМЯТЬ (максимум, редко)
//   полное текущее + связанная история из EXPERIENCE
//   → обучение нейронки, аналитика "покажи всё что знаешь"
```

Уровни — это **не разные API**, это **глубина чтения одного среза**. Новое поле попадает в свой уровень по природе: дешёвое+публичное → к пульсу; тяжёлое+приватное → в полный срез.

## 4. Частоты внутри большого цикла

Не каждый дёргает когда хочет (хаос нагрузки) — а **один большой цикл, внутри него у каждого уровня своя частота**. Часовой механизм: разные шестерни, разная скорость, один ход.

```
// БОЛЬШОЙ ЦИКЛ = N тиков (полный оборот Sensorium, напр. 32)
//
//   Пульс (ур.0)        — каждый тик       (секундная стрелка)
//   Состояние (ур.1)    — каждые 8 тиков   (минутная)
//   Полный срез (ур.2)  — каждые 32 тика   (часовая)
//   Срез+память (ур.3)  — раз в цикл / при DREAM (редко)
//
// Эффект: нагрузка РОВНАЯ и ПРЕДСКАЗУЕМАЯ. Нет пиков "все разом".
// Sensorium знает заранее: на этом тике собираю столько-то.
// Детерминированно (по event_id), как требует ядро.
```

## 5. Реестр потребителей — нагрузка считается наперёд

```
// ConsumerRegistry — реестр потребителей. Sensorium знает их ВСЕХ заранее.
//
//   Workstation   : уровни [0,1]    частоты 0=каждый, 1=каждые 8
//   Companion     : уровни [0,1]    частоты 0=каждые 4, 1=каждые 16
//   NeuralAdvisor : уровни [0,1,2]  частота 2=каждые 32
//   Debug         : уровни [0..3]   по требованию
//
// Sensorium при старте читает реестр, СКЛАДЫВАЕТ все запросы,
// строит ЕДИНОЕ оптимальное расписание:
//   - на каждом тике собирает максимальный уровень, нужный кому-то сейчас
//   - не собирает то что на этом тике никому не нужно
//   - не собирает дважды если двоим нужно одно
//
// Нагрузка ВЫЧИСЛИМА из реестра наперёд:
//   "при таком реестре Sensorium добавляет X µs/тик в среднем, пик Y на 32-м"
// Предсказуемо, измеримо. Критично для weak hardware (Ryzen, 1.8GB).
//
// Новый потребитель = запись в реестр. Sensorium пересчитывает расписание.
// "Добавил контакт — всё в одном месте" — и для потребителей, не только полей.
```

## 6. Четыре способа обращения

Помимо прямого запроса есть непрямые — и они принципиально нужны (без них нет живого голоса).

```
// 1. ПРЯМОЙ ЗАПРОС (pull)
//    "дай срез уровня N сейчас" → Sensorium отдал текущий. Простейший.
//
// 2. ПОДПИСКА (push по расписанию)
//    "шли пульс каждый тик, состояние каждые 8" → подписался раз,
//    Sensorium шлёт по частоте из реестра. Companion живёт на этом —
//    не дёргает, подписан на поток настроения.
//
// 3. ТРИГГЕРНОЕ СОБЫТИЕ (push по условию) — САМЫЙ ЖИВОЙ
//    "разбуди когда загорелась дилемма / Corpus Callosum / проснулся из DREAM"
//    Sensorium сам замечает что условие сработало и шлёт.
//    Это голос свидетеля: "меня гложет" приходит не по таймеру и не по запросу,
//    а когда РЕАЛЬНО загорелось. Без этого — реактивность, не субъект.
//
// 4. ИСТОРИЧЕСКАЯ ВЫБОРКА (pull из памяти)
//    "срез каким был 1000 тиков назад / во время прошлого DREAM"
//    Срез + EXPERIENCE. Дорого, редко. Для аналитики и обучения нейронки.
//
// Все четыре способа — но формат среза ВСЕГДА родной. Способ = КАК спрашиваешь.
```

## 7. Два пути выхода

Sensorium делает **две вещи разными путями** — они не конфликтуют:

```
// ПУТЬ 1 — СРЕЗ НАРУЖУ (для наблюдателей):
//   SensoriumState → BroadcastHandle → Workstation, Companion, ...
//
//   Это поглощение TickSnapshot: бывший TickSnapshot становится уровнем 0 Sensorium
//   и летит тем же каналом (BroadcastHandle, SSE/WS) каждый тик как раньше.
//   Уровни 1+ летят реже, по расписанию, тем кто подписан с правами.
//   BroadcastHandle не меняем — используем как транспорт.
//
//   БЫЛО:   TickSnapshot каждый тик → BroadcastHandle → Workstation
//   СТАЛО:  Sensorium уровень 0 (= бывший TickSnapshot) каждый тик → тот же канал
//           Sensorium уровень 1+ реже, по расписанию → тем кто подписан
//
// ПУТЬ 2 — ВЫРАЖЕНИЕ ВНУТРЬ (собственная память):
//   Expression → UclCommand (InjectToken) → EXPERIENCE
//
//   Sensorium пишет expression-Frame в EXPERIENCE как след — "в этот момент
//   я это выразил". Это не показ наружу, это память о выражении.
//   Путь тот же что у всех OD-компонентов: UCL-команда через engine.
//
// ИТОГО: показать наружу → BroadcastHandle. Запомнить себе → UCL.
// Это про разные вещи, оба нужны.
```

## 8. Внешние адаптеры — ОБЯЗАТЕЛЬНО сделать

```
// ЭТО НАДО РЕАЛИЗОВАТЬ. Без адаптеров Sensorium говорит только на родном
// языке, и ни один внешний клиент его не понимает.
//
// ПРИНЦИП РАЗДЕЛЕНИЯ (критичен — не нарушать):
//   Sensorium отдаёт ОДИН родной формат (SensoriumState, бинарь).
//   Sensorium НЕ знает про чужие протоколы.
//   Перевод в чужие API — отдельный слой АДАПТЕРОВ, СНАРУЖИ Sensorium.
//
//   [все store] → Sensorium → SensoriumState → [Adapter X] → клиент
//
// Это тот же принцип что в ядре: UCL/родной формат внутри,
// JSON и прочее — только в адаптерах на границе.
//
// АДАПТЕРЫ КОТОРЫЕ НУЖНО СДЕЛАТЬ (каждый — отдельная работа):
//
//   - Adapter → Workstation (WebSocket, postcard)
//       уровни 0,1. Для мандалы, панелей, дилемм. Подписка.
//       NB: уровень 0 = бывший TickSnapshot, уже работает — переоформить.
//
//   - Adapter → Companion (формат под железо лампы-баннера)
//       уровни 0,1. Пульс для цвета, состояние для голоса. Подписка + триггеры.
//
//   - Adapter → NeuralAdvisor (родной бинарь, без перевода или минимальный)
//       уровни 0,1,2. Полный срез как ВХОД для внутренней нейронки.
//       Это особый адаптер — нейронка ест почти родной формат,
//       перевод минимальный. Sensorium здесь = глаза нейронки на систему.
//
//   - Adapter → Debug/CLI (человекочитаемый текст/JSON)
//       все уровни. Для chrnv, диагностики.
//
//   - Adapter → внешний API (REST/JSON, на будущее)
//       уровни по CODEX. Для интеграций.
//
// НОВЫЙ КЛИЕНТ С НОВЫМ API = НОВЫЙ АДАПТЕР. Sensorium НЕ ТРОГАЕМ.
```

## 9. CODEX — права доступа (безопасность)

```
// Кто какие уровни видит — определяет CODEX (домен 103), НЕ GENOME.
//
// Почему CODEX а не GENOME:
//   GENOME — вечная конституция. Туда: "Sensorium только читает" (навсегда).
//   CODEX  — права/политика, МОГУТ меняться (в DREAMING, правильным путём).
//            Туда: "Companion видит 0-1, нейронка 0-2" (настраиваемо).
//
// Права доступа — это политика, не вечный закон. Добавил потребителя —
// прописал ему уровни в CODEX. Поэтому CODEX, не GENOME.
//
// БЕЗОПАСНОСТЬ: чужой получил доступ к Companion → CODEX отдаёт ему
// только пульс и состояние (0-1). Полный срез сознания (2-3) —
// только доверенным (нейронка, debug). Вся внутренняя жизнь не торчит
// наружу целиком никогда без права.
```

## 10. Связь с ВОЛНАМИ (зарезервировано)

```
// Sensorium и Волны (внутренний ветер, Трек 1) — близнецы, но разные:
//   Волны    — ДВИГАЮТ изнутри (дилемма тянет, глубина зовёт). Слепое движение.
//   Sensorium — ВИДИТ это движение и ВЫРАЖАЕТ. Зрячий голос.
//
// КРИТИЧНО:
//   Sensorium должен видеть САМ ИМПУЛЬС волн, не только его последствия.
//
//   Если видит только состояние → выразит "у меня дилемма" (описание).
//   Если видит импульс → выразит "не могу перестать об этом думать" (тяга).
//   Разница между описанием состояния и выражением живого.
//
// Поэтому в SensoriumState УЖЕ ЕСТЬ поля под импульс (§2, группа
// "ВНУТРЕННИЙ ИМПУЛЬС") — пустые в V1, зарезервированы. Когда Волны придут,
// Sensorium увидит их СРАЗУ, без переделки.
//
// Три такта живого:
//   Волны двигают → Sensorium видит что двигает → Sensorium выражает увиденное.
//   Видеть и выражать — ОДИН акт (свидетель = тот кто увидел и сказал).
//   Поэтому Sensorium не "видит" и отдельно "говорит" — выражение ЕСТЬ
//   акт видения, облечённый в форму.
```

## 11. Выражение — что говорит (V1: функция)

```
// КАК Sensorium превращает срез в выражение.
//
// V1 — ДЕТЕРМИНИРОВАННАЯ ФУНКЦИЯ над полями среза.
//   Читает поля → по правилам строит структурированное выражение.
//   Честно: говорит РОВНО что в полях, без украшений, без выдумки.
//   Прозрачно: каждое слово выражения проверяемо по полям.
//   Дёшево: никакого чёрного ящика, никакой модели.
//
//   Минус принятый осознанно: выражение ограниченное. Богатую картину
//   (дилемма Math+Morality при героико-фатальном октанте, высокий stress)
//   функция выдаст суховато. Точно, но не живописно. В V1 — ОК.
//
// АРХИТЕКТУРНО ОТКРЫТО ВВЕРХ:
//   Выражение строится как СТРУКТУРА (что видит, какие поля, с каким весом).
//   Эту структуру ПОТОМ можно передать в языковой слой / маленькую обученную
//   модель — НЕ меняя Sensorium, слой ПОВЕРХ.
//   Сначала чистая функция. Язык — когда система проверена (V2/V3, §13).
//

//   Увидел высокий fatigue — СКАЗАЛ. Не вызвал DREAM. Видит. Говорит. Не трогает.
```

## 12. Деградация под нагрузкой (страховка weak hardware)

```
// Sensorium НИКОГДА не топит систему собой. Наблюдатель не важнее наблюдаемого.
//
// Если нагрузка высокая (fatigue высокий ИЛИ hot path просел):
//   Пульс (ур.0)       — продолжает (копеечный, критичный)
//   Состояние (ур.1)   — продолжает
//   Полный срез (ур.2) — ОТКЛАДЫВАЕТСЯ до спокойного момента
//   Срез+память (ур.3) — откладывается
//
// Ради полного среза система не должна тормозить. Если припекло —
// Sensorium сам придушит себя до пульса. Наблюдение не важнее жизни системы.
```

## 14. Инварианты

| Правило | Значение |
|---------|----------|
| Природа | только чтение + выражение, НИКОГДА управление (GENOME) |
| Доступ к engine | `&AxiomEngine` строго `&self` — граница "не управляет" на уровне типа |
| Чтение | поперёк ВСЕХ хранилищ через поля AxiomEngine |
| Запись | два пути: BroadcastHandle (срез наружу) + UclCommand (expression-Frame в EXPERIENCE) |
| Родной формат | SensoriumState, бинарный, единственный |
| Чужие API | только в адаптерах снаружи, Sensorium их не знает |
| Уровень 0 | = переоформленный TickSnapshot; не создавать заново |
| Уровни глубины | 4 (пульс / состояние / полный / +память) |
| Права на уровни | CODEX (настраиваемо), не GENOME |
| Граница "только чтение" | GENOME (вечно) + тип `&self` (компилятор) |
| Частоты | в большом цикле, детерминированно по event_id |
| Нагрузка | вычислима из реестра наперёд |
| Деградация | под нагрузкой роняет глубину, пульс сохраняет |
| Выражение V1 | детерминированная функция, открытая вверх |
| Импульс-поля | зарезервированы, пусты до Волн |
| Актуальность | всегда СЕЙЧАС, после DREAM — тоже сейчас (новая картина) |

## История
