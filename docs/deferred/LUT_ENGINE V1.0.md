**Математическое ядро (LUT_Engine)**.

Мы заменяем вычисления с плавающей точкой (деление, экспоненты, логарифмы) на чтение из заранее подготовленных массивов в памяти процессора. Это даст ядру ту самую детерминированную наносекундную скорость.

Вот жесткая спецификация этого модуля.

---

# AXIOM MODULE SPECIFICATION: LUT_ENGINE V1.0

**Статус:** Core Math Engine

**Версия:** 1.0.0 (Zero-Cost Interpolation)

**Назначение:** Замена дорогостоящей математики (`f32::exp`, деление) на кэш-дружественные `Look-Up Tables` (LUT).

## 1. Параметры памяти и Разрешение (Resolution)

Чтобы таблицы были молниеносными, они должны целиком помещаться в L1/L2 кэш процессора.

Мы устанавливаем размер одной таблицы в **4096 элементов** (`4096 * 4 байта = 16 КБ`).



```Rust
// core/src/math/lut.rs

pub const LUT_SIZE: usize = 4096;

// Физические пределы системы (для нормализации)
pub const MAX_DISTANCE: f32 = 1000.0;   // Максимальный радиус взаимодействия
pub const MAX_TICKS: f32 = 10000.0;     // Максимальное время для полного остывания

// Предвычисленные множители для избежания деления в рантайме
const DIST_SCALE: f32 = (LUT_SIZE - 1) as f32 / MAX_DISTANCE;
const TIME_SCALE: f32 = (LUT_SIZE - 1) as f32 / MAX_TICKS;
```

## 2. Физические Таблицы (The Tables)

Ядро Axiom использует три фундаментальные кривые. На этапе компиляции (или инициализации) система заполняет эти массивы чистой математикой.

1. **`GRAVITY_LUT`** (Закон обратных квадратов): $F \propto \frac{1}{r^2}$. Используется для притяжения токенов.
    
2. **`DECAY_LUT`** (Термодинамическое остывание): $E(t) = e^{-\lambda t}$. Используется для затухания энергии волны и старения связей.
    
3. **`SIGMOID_LUT`** (Функция активации): $S(x) = \frac{1}{1 + e^{-x}}$. Используется мембранами для мягкой фильтрации порогов.
    

Rust

```Rust
lazy_static! {
    pub static ref GRAVITY_LUT: [f32; LUT_SIZE] = build_gravity_table();
    pub static ref DECAY_LUT:   [f32; LUT_SIZE] = build_decay_table();
    pub static ref SIGMOID_LUT: [f32; LUT_SIZE] = build_sigmoid_table();
}

fn build_gravity_table() -> [f32; LUT_SIZE] {
    let mut table = [0.0; LUT_SIZE];
    for i in 0..LUT_SIZE {
        // Дистанция от 0.0 до MAX_DISTANCE
        let r = (i as f32) / DIST_SCALE;
        // Защита от деления на ноль в эпицентре (r + 1.0)
        table[i] = 1.0 / ((r + 1.0) * (r + 1.0)); 
    }
    table
}
```

## 3. Механизм Нормализации и Интерполяции (LERP)

Если дистанция равна `10.5`, мы не можем просто взять индекс `10` или `11` — физика будет "дерганой" (лестничный эффект). Мы должны смешать два соседних значения массива пропорционально.

**Секрет скорости:** Линейная интерполяция (LERP) требует только сложения и умножения. Никакого деления.

Rust

```Rust
/// Получить силу гравитации для любой дистанции за ~2 наносекунды
#[inline(always)]
pub fn get_gravity(distance: f32) -> f32 {
    // 1. Нормализация (f32 -> index)
    let scaled = distance * DIST_SCALE;
    let index = scaled as usize;

    // Ограничитель (clamp) - если ушли за пределы массива
    if index >= LUT_SIZE - 1 {
        return 0.0; 
    }

    // 2. Вычисление дробной части для смешивания
    let fract = scaled - index as f32;

    // 3. Чтение из таблицы (попадание в кэш)
    let val1 = GRAVITY_LUT[index];
    let val2 = GRAVITY_LUT[index + 1];

    // 4. LERP: v1 + fract * (v2 - v1)
    val1 + fract * (val2 - val1)
}

/// Получить коэффициент затухания энергии в зависимости от прошедших тиков
#[inline(always)]
pub fn get_decay(ticks: f32) -> f32 {
    let scaled = ticks * TIME_SCALE;
    let index = scaled as usize;

    if index >= LUT_SIZE - 1 {
        return 0.0; // Полное затухание
    }

    let fract = scaled - index as f32;
    let val1 = DECAY_LUT[index];
    let val2 = DECAY_LUT[index + 1];

    val1 + fract * (val2 - val1)
}
```

## 4. Как это используется в Ядре

Когда SWE-агент будет писать `Field Engine` (обновление позиций Токенов), код гравитации будет выглядеть кристально чисто и работать с околосветовой скоростью:

Rust

```Rust
// Внутри цикла расчета симуляции (Tick):
let distance = calculate_distance(&token_a.pos, &token_b.pos);

// Мгновенный LUT-запрос вместо дорогого f32::powf
let gravity_factor = lut::get_gravity(distance);

// Применение силы (масса * коэффициент Домена * гравитация)
let force = token_b.mass * config.get_gravity() * gravity_factor;
```

---

### Что мы получили:

1. **Предсказуемость:** Вызов `lut::get_gravity()` всегда занимает одинаковое количество тактов процессора (О(1)), в отличие от сложных математических сопроцессоров.
    
2. **Точность:** Разрешение в 4096 шагов с интерполяцией дает погрешность менее $0.01\%$, что абсолютно невидимо для макро-поведения системы.
    
3. **Инкапсуляция:** Вся тяжелая математика спрятана. Если нам понадобится изменить кривую затухания времени с экспоненты на логарифм — мы меняем только формулу в функции сборки массива `build_decay_table()`, а логика ядра остается нетронутой.
