# CAUSAL ORDER MODEL (COM)

## Спецификация времени как упорядоченности изменений

**Статус:** архитектурное решение  
**Применимость:** Grid / Token + Connection  
**Совместимость:** UPO stateless  
**Связанные спеки:** Token V5.0, Connection V5.0, Domain V1.2, UPO v2.1

---

## 1. Принцип

В системе отсутствует глобальная временная ось.

Вместо этого вводится **монотонный причинный порядок изменений**.

Время определяется как порядок применения событий.

Формально:

```
stateₙ₊₁ = F(stateₙ, eventₙ)
```

где `n` — порядковый номер события.

---

## 2. Основная сущность: Event

Каждое изменение состояния фиксируется как событие.

```rust
#[repr(C)]
pub struct Event {
    pub event_id: u64,        // Монотонный причинный индекс
    pub domain_id: u16,       // Домен, к которому относится событие
    pub event_type: u16,      // Тип события (UpdateToken, UpdateConnection, etc.)
    pub payload_hash: u64,    // Хеш содержимого (для валидации/детерминизма)
    pub _reserved: [u8; 8],   // Расширение
}
```

### Инварианты:

- `event_id` строго монотонно возрастает.
    
- Никакое событие не может иметь id меньше предыдущего.
    
- Порядок событий полностью определяет эволюцию системы.
    

---

## 3. Локальность времени

Домены могут обновляться асинхронно.

Однако:

- Все события, независимо от домена, получают глобальный `event_id`.
    
- Причинный порядок общий.
    
- Нет понятия «одновременности» — только последовательность.
    

---

## 4. Механизм генерации event_id

```rust
pub struct CausalClock {
    counter: AtomicU64
}
```

При создании события:

```
event_id = counter.fetch_add(1)
```

Это не время в секундах.  
Это индекс причинности.

---

## 5. Связь с UPO

UPO не хранит время.  
Он получает текущее состояние поля на момент вызова.

Если требуется маркировка следа:

- в `DynamicTrace.event_id` записывается event_id последнего обработанного события.
    

Это обеспечивает:

- воспроизводимость,
    
- детерминизм,
    
- возможность отката.
    

---

## 6. Детерминизм

Система детерминирована при условии:

1. Идентичного начального состояния.
    
2. Идентичной последовательности `Event`.
    

Это позволяет:

- воспроизводить поведение,
    
- делать снапшоты,
    
- делать форк состояния,
    
- проводить симуляции альтернативных веток.
    

---

## 7. Отсутствие глобального времени

В системе не существует:

- real_time
    
- delta_time
    
- глобального tick
    

Есть только:

- состояние
    
- события
    
- порядок применения
    

---

## 8. Асинхронность доменов

Домены могут:

- генерировать события независимо,
    
- обновляться по внутренним правилам,
    
- иметь разную плотность событий.
    

Но все события включаются в общий причинный поток.

---

## 9. Оптимизация: Snapshot

Так как журнал событий может расти бесконечно, вводится механизм:

```rust
pub struct Snapshot {
    pub snapshot_id: u64,     // Последний event_id
    pub state_hash: u64,
}
```

Snapshot фиксирует состояние после применения всех событий до `snapshot_id`.

После snapshot:

- возможно архивирование предыдущих событий,
    
- возможна компрессия.
    

---

## 10. Философский инвариант

Время = причинность.

Если нет событий — нет изменения — нет времени.

Система в статическом состоянии не «стареет».

---

## 11. Следствия для Grid

- Grid не является пространством во времени.
    
- Grid — это пространство состояний.
    
- Эволюция Grid — это последовательность трансформаций состояния.
    
- Направление времени определяется только порядком событий.
    

---

## 12. Ограничения

- Невозможно иметь истинную параллельную одновременность.
    
- Любая конкуренция должна разрешаться до присвоения `event_id`.
    
- Требуется строгий контроль над генерацией событий.
    

---

# Итог

Causal Order Model:

- сохраняет UPO stateless,
    
- обеспечивает асинхронность,
    
- сохраняет детерминизм,
    
- устраняет глобальную временную ось,
    
- делает причинность первичной структурой системы.
    

В Grid время перестаёт быть координатой.  
Оно становится порядком изменения структуры.

Это архитектура, в которой эволюция — это поток трансформаций, а не движение по часам.
