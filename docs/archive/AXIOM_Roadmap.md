# AXIOM — Roadmap

**Статус:** Дорожная карта
**Дата:** 2026-06-03
**Назначение:** свести воедино всё спроектированное — три пункта плана (Кросс / Волны / Сенсориум) плюс смежные ветки — с зависимостями, порядком и статусом. Карта дороги, не спецификация.
**Опирается на:** все спеки сессии + STATUS/BLUEPRINT/RESULTS

## 0. Где мы стоим

Движок V7, 1631 тест, hot path 24 µs. Шесть над-доменных модулей работают (FrameWeaver, AxialEvaluator V3, ContextRecognizer V6/7, NeuralAdvisor V3, OverDomainArbiter V3, DreamPhase V1.1). Девять подсистем познания детектируются. TextPerceptor 2-path жив, динамика течёт (Cascading), дилеммы кристаллизуются (DilemmaDetector V2.0).

**Система умеет слушать.** Не умеет — хотеть (внутренний импульс) и говорить (выражение). И наполовину умеет видеть мир не только текстом (Vision частичен).

Три пункта плана закрывают именно эти пробелы:

```
КРОСС     — связать символ с миром (grounding, мультимодальность)
ВОЛНЫ     — поднять изнутри (внутренний импульс, субъектность)
СЕНСОРИУМ — видеть себя и выразить (полный срез + голос)
```

## 2. Три пункта — суть и зависимости

### КРОСС — Cross-Modal Binding
**Что:** связывание Frame разных модальностей по co-activation. Текстовое "красный" ↔ визуальный кластер. Grounding символа в перцепте.
**Природа:** под-механизм ContextRecognizer (не модуль).
**Зависит от:** Vision pipeline (нужны две реальные модальности для полного теста; на одной — половинчато).
**Даёт:** систему понимающую мир, не только символы про символы.
**Независим** от Волн и Сенсориума — отдельная ветка.

### ВОЛНЫ — Internal Drive
**Что:** внутренний ветер. Импульс изнутри без внешнего входа. Три источника — незавершённые дилеммы, глубокий резонанс, незавершённые связи. internal_dominance_factor (реактивное↔когнитивное).
**Природа:** над-доменный модуль.
**Зависит от:** готового материала (дилеммы ✓, SutraDepth ✓, FrameWeaver candidates ✓) — всё на месте.
**Даёт:** субъектность. Система перестаёт быть зеркалом.

### СЕНСОРИУМ — полный срез + голос
**Что:** единая точка доступа ко всему внутреннему. Видит и выражает. 4 уровня глубины, частоты, реестр потребителей.
**Природа:** над-доменный модуль особой природы (только чтение, `&self`).
**Зависит от:** готовых хранилищ (всё на месте) для среза. Для **живого голоса** — от Волн (импульс-поля).
**Даёт:** голос. Нужен для Companion.

### Связь трёх (живой организм)

```
        КРОСС ──────────── обогащает чем система оперирует
          (символ↔мир)      (не только текст)
                                  │
   ВОЛНЫ ───дуют──→ СЕНСОРИУМ ───выражает──→ наружу
   (тяга изнутри)   (видит ветер)            (Companion/Workstation)

   Волны двигают → Сенсориум видит что двигает → выражает увиденное.
```

Сенсориум без Волн — честный, но суховатый срез (описание состояния).
Волны без Сенсориума — ветер, которого никто не видит.
Вместе — внутренняя жизнь и голос о ней.
Кросс — независим, идёт сбоку, расширяет входной материал.

## 3. Порядок реализации

Рекомендуемый порядок с обоснованием. Финальное решение — за chrnv.

```
ФАЗА 1 — СЕНСОРИУМ V1.0 (срез из готового)
  Почему первым:
    - материал весь на месте (хранилища готовы)
    - уровень 0 = переоформить TickSnapshot (наполовину готов)
    - даёт голос СРАЗУ, пусть суховатый (функция-выражение)
    - не зависит ни от чего нового
  Результат: можно спросить систему и услышать честный срез.
            Workstation/Companion получают единую точку доступа.

ФАЗА 2 — ВОЛНЫ V1.0 (оживить голос)
  Почему вторым:
    - материал готов (дилеммы, глубина, связи)
    - заполняет импульс-поля Сенсориума → голос оживает
    - "не могу перестать думать" вместо "у меня дилемма"
  Результат: система обретает внутреннюю жизнь. Сенсориум выражает тягу,
            не только состояние. Субъект, не зеркало.
  ↳ Сенсориум V2.0 активируется автоматически (импульс-поля заполнены).

ФАЗА 3 — КРОСС V1.0 (расширить мир) — МОЖЕТ ИДТИ ПАРАЛЛЕЛЬНО
  Почему может параллелиться:
    - независим от Волн/Сенсориума (другая ветка)
    - другие руки/сессия могут вести
  Зависимость: нужен замкнутый Vision pipeline для полного теста.
  Результат: grounding символов в перцептах. Путь к клинописи.

ПАРАЛЛЕЛЬНО ВСЕМУ — VISION PIPELINE (инженерный долг)
  Замкнуть L0VisionPerceptor → кристаллизация символа → грамматика.
  Доказать UGS вне текста. Разблокирует полный тест Кросса.
  Не концептуальный прорыв — закрытие долга. Отдельные руки.
```

Краткая логика: **голос → жизнь голосу → мир для голоса**. Сначала Сенсориум (есть из чего), потом Волны (оживляют его), Кросс и Vision — сбоку.

## 4. Смежные ветки (не из тройки, но в очереди)

```
DilemmaDetector V2.1 — углубление дилемм
  Заготовка готова (приложение Cross_Modal). Сигналы B (stress) + C (Corpus
  Callosum), отсев ложных, HeldInTension. Вся инфраструктура есть.
  Триггер возврата: когда дилеммы дают ложные срабатывания, ИЛИ когда
  cross-modal расхождение (text≠vision) потребует HeldInTension.
  → мост: Кросс естественно тянет за собой V2.1.

DilemmaDetector V3.0 — аксиогенез (рождение ценностей)
  ОТЛОЖЕНО осознанно: требует длительной живой работы (накопить
  неразрешимые конфликты за месяцы). На двухдневных дилеммах — мёртвая
  спека. Время — позже. Смыкается с Волнами V3 (ветер рождающий) и
  ContextRecognizer V8 roadmap.

ContextRecognizer V6→V9 — углубление восприятия
  V6/V7 в коде. V8 (мета-уровень, генеративные подсистемы), V9 (активная
  нейронка) — план есть (Roadmap V6-V9). Идут по мере надобности, не срочно.

Embeddings / AGENT-TD-01 — восприятие неявного
  Большая отдельная работа. Нужна для 6 нулевых подсистем (metaphor, style,
  geometry без маркеров). Lookup-таблица — предшественник, embeddings —
  замена. Откладывается до явной потребности.

Внутренняя нейронка — финал
  Sensorium V3.0 + NeuralAdvisor активная фаза. Нейронка читает полный срез
  как вход, система видит себя своими глазами. Самый дальний горизонт.
```

## 6. Вехи (что считается "получилось")

```
ВЕХА 1 — Голос. Сенсориум V1.0 работает.
  Спросил систему → услышал честный срез. Workstation читает единую точку.
  Критерий: SensoriumState отдаётся по уровням, TickSnapshot переоформлен.

ВЕХА 2 — Жизнь. Волны V1.0 работают.
  Система в тишине поднимает то что тянет. Сенсориум выражает тягу.
  Критерий: internal_dominance_factor растёт в тишине, импульс-поля
            Сенсориума заполняются, голос меняется с "дилемма" на "гложет".

ВЕХА 3 — Мир. Кросс V1.0 + Vision pipeline.
  Символ связан с перцептом. UGS доказан на картинке (клинопись/синтетика).
  Критерий: CROSS_MODAL_BOND создаётся, текст↔образ связываются.

ВЕХА 4 — Companion. Все три + железо.
  Лампа-баннер на столе выражает внутреннюю жизнь голосом.
  Критерий: Companion читает Сенсориум, выражает состояние+тягу человеку.
  Это горизонт — ради него всё.
```

## 7. Принципы, которые держим во всех ветках

```
- Token/Connection 64B HARD — тяжёлые данные в отдельные store
- UCL единственный путь мутации, GUARDIAN валидирует
- Над-доменные читают состояние, пишут через UCL
  (Сенсориум исключение: читает &AxiomEngine, но &self — не управляет)
- Причинное время (event_id), нет wall-clock в ядре
- Значимые изменения (новые подсистемы, ценности, cross-modal bonds) —
  через chrnv approval. Система не само-меняется бесконтрольно.
- GENOME вечен (границы), CODEX настраиваем (права)
- weak hardware: всё считает нагрузку наперёд, деградирует под давлением
- спека и код синхронны — не допускать drift
```
