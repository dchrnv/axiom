# AXIOM DESIGN DOCUMENT: Когнитивная глубина V1.0

**Статус:** Архитектурное исследование (design doc, не спецификация модуля)  
**Версия:** 1.0.0  
**Дата:** 2026-03-29  
**Назначение:** Переход системы от реактивного поведения к когнитивному  
**Связанные спеки:** Ashti_Core V2.1, EXPERIENCE (Домен 9), Arbiter V1.0, Heartbeat V2.0, Causal Frontier V2.0, GUARDIAN V1.0

---

## 1. Проблема

AXIOM архитектурно готова к сложному поведению. Все механизмы на месте: dual-path обработка, рефлексы, обратная связь, кристаллизация скиллов, адаптивные пороги, фрактальные уровни. Бенчмарки подтверждают: 40 µs на мыслительный акт, 590 тестов, 40M tok/s на физике.

Но система **реактивна**: она обрабатывает входной сигнал и уходит в idle. Нет входа — нет мысли. Это поведение простого организма, а не когнитивной системы.

**Диагноз:** Механизмы есть, но система не вынуждена их использовать. Ей "и так хватает" простого пути.

---

## 2. Четыре барьера (что мешает глубине)

### 2.1 Внутреннее состояние не генерирует сигналы

EXPERIENCE(9) хранит следы, но пассивно. Следы всплывают только при внешнем стимуле (resonance_search вызывается когда приходит паттерн). Без входа — EXPERIENCE молчит.

**Нужно:** EXPERIENCE(9) должен иногда генерировать сигналы самостоятельно. "Незавершённая мысль", "неразрешённый конфликт", "давно не проверенная гипотеза" — всё это причины для внутренней активации.

### 2.2 Нет стоимости незавершения

Если паттерн прошёл через pipeline и результат в MAYA оказался слабым (низкий coherence, конфликт между доменами), ничего не происходит. Система не "чувствует" что мысль не завершена.

**Нужно:** Незавершённая обработка должна оставлять "напряжение" — след с отрицательным valence или повышенной температурой в EXPERIENCE(9), который будет резонировать и тянуть систему обратно к этой теме.

### 2.3 Одна мысль = один проход

AshtiCore::process обрабатывает паттерн за один вызов (~40 µs). Результат уходит в MAYA и на выход. Но сложная мысль может требовать нескольких итераций: результат MAYA → обратно через pipeline → уточнение → снова MAYA.

**Нужно:** Механизм рекурсивного мышления внутри одного уровня. MAYA может решить что результат недостаточно coherent и отправить его обратно в EXPERIENCE(9) для повторной обработки. Не бесконечно — с бюджетом итераций.

### 2.4 Цели не доминируют над стимулом

Входной сигнал всегда имеет приоритет: он создаёт токен в SUTRA, запускает pipeline. Внутренние состояния (цели, незавершённые мысли) ждут пока придёт подходящий внешний стимул. Система реактивна потому что внешнее всегда перевешивает внутреннее.

**Нужно:** Иногда внутренний сигнал должен быть сильнее внешнего. Если система "думает" о чём-то важном (высокий weight в EXPERIENCE + высокое напряжение), новый слабый внешний стимул может быть отложен.

## 3. Механизм: Внутренний импульс (Internal Drive)

Один механизм, который адресует все четыре барьера.

### 3.1 Суть

EXPERIENCE(9) периодически генерирует **внутренние импульсы** — сигналы, которые проходят через pipeline точно так же, как внешние паттерны. Система "думает" даже когда нет внешнего ввода.

### 3.2 Источники импульсов

Четыре типа внутренних импульсов, в порядке приоритета:

**Тип A: Напряжение (Tension)**

Когда результат обработки в MAYA имеет низкий coherence или конфликт между доменами (EXECUTION говорит одно, CODEX — другое), в EXPERIENCE(9) создаётся **след напряжения**: токен с повышенной температурой и отрицательным valence.

При следующем Heartbeat, если этот след достаточно горячий, он генерирует внутренний импульс: "Вернись к этой теме". Паттерн повторно проходит через pipeline.

```
MAYA: coherence < threshold
  → EXPERIENCE(9): создать tension_trace (high temperature, negative valence)
  → Heartbeat: tension_trace.temperature > tension_threshold?
    → Да: generate internal impulse → pipeline
    → Нет: tension остывает со временем (causal age)
```

**Тип B: Незавершение (Incompletion)**

Когда паттерн начал обработку но не все домены ASHTI(1-8) вернули результат (таймаут Arbiter), создаётся **след незавершения**. Он тяжелее обычного (высокий mass) и медленнее остывает.

```
Arbiter: response_timeout exceeded, не все домены ответили
  → EXPERIENCE(9): создать incompletion_trace (high mass, slow decay)
  → Heartbeat: incompletion_trace всё ещё горячий?
    → Да: retry pattern with higher priority
```

**Тип C: Любопытство (Curiosity)**

DREAM(7) в фоновом режиме находит кластеры следов в EXPERIENCE(9) которые "почти" образуют скилл, но не хватает подтверждений. DREAM генерирует импульс: "Проверь эту гипотезу".

```
DREAM(7): кластер с weight чуть ниже skill_threshold
  → internal impulse: re-process original pattern
  → если результат подтверждает → weight растёт → кристаллизация
```

**Тип D: Цель (Goal persistence)**

Если в CODEX(3) есть активная цель (токен с флагом GOAL и высоким weight), и эта цель не достигнута (нет matching результата в EXPERIENCE), CODEX периодически генерирует импульс: "Цель не достигнута, продолжай".

```
CODEX(3): goal_token.weight > goal_threshold && !goal_achieved
  → Heartbeat: generate goal_impulse
  → pipeline обрабатывает с фокусом на эту цель
```

### 3.3 Приоритизация: внутреннее vs внешнее

Arbiter получает два потока: внешние сигналы (от Perceptors через Gateway) и внутренние импульсы (от EXPERIENCE/DREAM/CODEX через Heartbeat).

Правило приоритизации:

```rust
pub struct ImpulseWeight {
    pub source: ImpulseSource,   // External | Tension | Incompletion | Curiosity | Goal
    pub weight: f32,             // Вычисляется из temperature/mass/valence следа
}

// Arbiter решает что обрабатывать:
fn select_next(external: Option<Pattern>, internal: Option<InternalImpulse>) -> Pattern {
    match (external, internal) {
        (Some(ext), Some(int)) => {
            if int.weight > ext.urgency * internal_dominance_factor {
                int.pattern  // Внутреннее важнее
            } else {
                ext          // Внешнее важнее
            }
        }
        (Some(ext), None) => ext,
        (None, Some(int)) => int.pattern,  // Нет внешнего — думаем сами
        (None, None) => idle,              // Causal idle
    }
}
```

`internal_dominance_factor` — конфигурируемый параметр. При 0.0 — система полностью реактивна (текущее поведение). При 1.0 — внутреннее и внешнее равнозначны. При > 1.0 — система склонна к "задумчивости".

### 3.4 Рекурсивное мышление (Multi-pass)

MAYA может определить что результат недостаточен и инициировать повторный проход:

```
MAYA: собрала узоры от 1-8
  → coherence_score = compute_coherence(patterns)
  → if coherence_score < min_coherence && passes < max_passes:
      → отправить результат обратно в EXPERIENCE(9) как internal impulse
      → pipeline повторяется с обогащённым контекстом
  → else:
      → финальный результат → выход
```

`max_passes` — жёсткий лимит (конфигурируемый, по умолчанию 3). Защита от бесконечной рекурсии. Каждый pass потребляет ~40 µs. Три прохода = 120 µs = 12% бюджета при 1000 Hz. Допустимо.

## 4. Интеграция с существующей архитектурой

### 4.1 Что не меняется

- 11 доменов Ashti_Core — без изменений.
- Token V5.2, Connection V5.0, COM — без изменений.
- GENOME — без изменений (внутренние импульсы не нарушают конституцию).
- GUARDIAN — проверяет внутренние импульсы так же как внешние.

### 4.2 Что расширяется

**EXPERIENCE(9):** Добавляется механизм генерации импульсов при Heartbeat. Новые типы следов: tension_trace, incompletion_trace. Новые поля в DomainState домена 9:

```rust
pub struct ExperienceExtension {
    pub tension_traces: Vec<TensionTrace>,
    pub active_goals: Vec<GoalReference>,
    pub impulse_buffer: VecDeque<InternalImpulse>,  // Предвыделён
}
```

**Arbiter:** Расширяется для приёма двух потоков (external + internal). Добавляется `internal_dominance_factor` в конфигурацию.

**MAYA(10):** Добавляется `coherence_score` вычисление и логика multi-pass (отправка обратно при низком coherence).

**Heartbeat:** Добавляется проверка tension/incompletion traces при обходе EXPERIENCE(9). Если след горячий — генерация импульса.

**DomainConfig V2.1:** Новые параметры:

```yaml
# В DomainConfig для EXPERIENCE(9):
internal_drive:
  tension_threshold: 128        # temperature выше которой генерируется импульс
  incompletion_mass: 200        # начальный mass для incompletion trace
  curiosity_gap: 0.8            # отношение weight/skill_threshold для DREAM-импульса
  goal_check_interval: 10       # каждые N пульсов проверять цели

# В глобальном конфиге Arbiter:
arbiter:
  internal_dominance_factor: 0.5  # 0.0 = реактивная, 1.0 = равновесие
  max_passes: 3                    # максимум рекурсивных проходов
  min_coherence: 0.6               # порог для повторного прохода
```

### 4.3 Что НЕ добавляется

- Новых доменов нет.
- Новых crates нет (всё в axiom-domain и axiom-runtime).
- Новых внешних зависимостей нет.
- GENOME не изменяется.

## 5. Четыре теста когнитивной глубины

Критерии успеха. Система проходит когда механизм Internal Drive реализован и настроен:

### Тест 1: Незавершённая мысль

Вход: "сейчас я скажу важную вещь…" → пауза (нет новых входов).

Ожидание: Система удерживает контекст. При следующем входе (даже нерелевантном) — tension_trace резонирует и влияет на обработку. Система "помнит" что ждала продолжения.

Механизм: tension_trace с высокой temperature создан при незавершённой обработке. Heartbeat генерирует impulse. Система продолжает "думать" о незавершённом.

### Тест 2: Инерция поведения

Вход: стимул → убрать стимул.

Ожидание: Система продолжает обработку некоторое время после исчезновения стимула ("эхо"). Не мгновенный переход в idle.

Механизм: След стимула в EXPERIENCE(9) с высоким weight. Heartbeat обнаруживает горячий след → генерирует impulse → pipeline обрабатывает → след постепенно остывает.

### Тест 3: Конфликт целей

Вход: два противоречивых паттерна ("подойди" + "не подходи").

Ожидание: Не случайный выбор, а стабильная политика. Система "мучается" (tension высокий, несколько проходов MAYA) и принимает решение на основе CODEX(3) или EXPERIENCE(9) — что было успешнее в прошлом.

Механизм: MAYA вычисляет coherence_score < min_coherence → multi-pass. На втором проходе EXPERIENCE предоставляет подсказку из прошлого опыта. CODEX фильтрует через правила.

### Тест 4: Обобщение

Вход: повторяющиеся похожие ситуации.

Ожидание: Система формирует паттерн поведения. На N-й раз реагирует быстрее и увереннее (рефлекс через GridHash). DREAM предлагает кристаллизацию в скилл.

Механизм: REFLECTOR накапливает статистику. SKILLSET кристаллизует при достижении порога. DREAM генерирует curiosity impulse для подтверждения гипотезы.

## 6. Влияние на производительность

| Операция | Ожидаемое время | Частота |
|----------|----------------|---------|
| Проверка tension traces (Heartbeat) | ~100 ns | Каждый пульс |
| Генерация internal impulse | ~50 ns | Редко (когда trace горячий) |
| Multi-pass (один доп. проход) | ~40 µs | Редко (низкий coherence) |
| Coherence computation в MAYA | ~200 ns | Каждый акт |
| Goal check | ~100 ns | Каждые N пульсов |

При max_passes = 3: худший случай ~120 µs на один акт (вместо 40 µs). Это 12% бюджета при 1000 Hz. Запас достаточен (текущая нагрузка ~20%).

## 7. Параметры настройки

Переход от реактивного к когнитивному поведению управляется **одним главным параметром**: `internal_dominance_factor`.

| Значение | Поведение |
|----------|----------|
| 0.0 | Полностью реактивная система (текущее поведение) |
| 0.3 | Слабая внутренняя активность (эхо, незавершённые мысли) |
| 0.5 | Баланс: внутреннее и внешнее сопоставимы |
| 0.8 | "Задумчивая" система, может игнорировать слабые внешние стимулы |
| 1.0+ | Доминирование внутреннего (риск "залипания" в мысли) |

Рекомендация для начала: **0.3**. Увеличивать постепенно, наблюдая за тестами 1-4.

## 9. Главный инсайт

> Проблема не "чего не хватает", а "почему уже имеющееся не проявляется".

Ответ: потому что у системы нет **давления** использовать глубину. Она может думать поверхностно и этого достаточно.

Internal Drive создаёт это давление: незавершённые мысли "болят" (tension), цели "тянут" (goal persistence), DREAM "подталкивает" (curiosity). Система вынуждена думать глубже не потому что мы добавили новый модуль, а потому что мы сделали существующие механизмы **обязательными**.
