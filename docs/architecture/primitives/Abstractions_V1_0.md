## 0. Что эта спека описывает

Abstractions — **мета-подсистема** AXIOM, определяющая как уровень абстракции закодирован в структурах системы. В отличие от Writing, Mathematics, Time, Values — это не доменная подсистема с собственными якорями-примитивами. Это **формальное отображение** между теоретическими уровнями абстракции (L0–L7) и внутренними структурами AXIOM.

Главный результат этой спеки — точная таблица соответствия:

```
Уровень абстракции L0–L7  ←→  FrameComposition C0–C5_Plus
```

И семь **абстракционных якорей** — мета-анкоры в SUTRA, к которым притягиваются Frame-ы соответствующей глубины.

**Эта спека НЕ описывает:**
- Как конкретно кристаллизует FrameWeaver — это `FrameWeaver_V1_2.md`
- Floor Lowering механизм — это `Primitives_And_Abstractions.md §4`
- Конкретные примитивы Writing/Math/Time — их собственные спеки

## 1. Главный принцип Abstractions

**Уровень абстракции = глубина дерева композиции от C0.**

Это ключевое решение: абстрактность не задаётся вручную, не хранится в поле `abstraction_level: u8` — она **вычисляется** из структуры composition_of дерева FrameWeaver.

```
C0  — сырые L0-примитивы (stroke, edge, before/after, pain-signal)
C1  — первичные кластеры (буква, интервал, моральное основание)
C2  — объектная абстракция (слово, ритм, культурная норма)
C3  — категоризация (концепт, грамматическое правило, ценность)
C4  — свойства и отношения (теорема, дилемма, нарратив)
C5  — абстрактные теории (математическая система, этическая система)
C5+ — конструктивизм (создание новых категорий, аксиогенез)
```

Это не произвольная нумерация — она отражает **реальную когнитивную иерархию**, подтверждённую эмпирически: каждый уровень строится из предыдущего и не сводится к нему.

## 2. Abstraction-Anchors-7: семь мета-якорей

В отличие от других примитивов — эти якоря не доменные. Они не принадлежат Writing, Mathematics, Time или Values. Они **универсальны**: каждый Frame в SUTRA, независимо от домена, может гравитировать к одному из них в зависимости от своей глубины.

Все живут в SUTRA (domain_id=100), STATE_LOCKED. Масса 160–200. Позиции — в **высокой** части пространства по оси Z (абстрактность = высокая воля / высокий потенциал).

---

### A0 — Raw (сырое)

**Философия:** Нулевой уровень. Сигнал без интерпретации. Пиксели, фонемы, сенсорный шум. Ещё не паттерн — только данные. Архетип: "Сигнал есть".

```yaml
- id: "abstraction_raw"
  word: "сырое"
  aliases: ["raw", "сырой", "необработанный", "сигнал", "данные", "стимул"]
  tags: ["meta", "abstraction", "a0_raw", "layer:L0"]
  position: [2000, 2000, 1000]
  # Низкий X, Y, Z: сырое — не упорядочено, не связано, не активно
  shell: [30, 20, 0, 0, 0, 0, 0, 0]
  # L1=30 физическое, L2=20 сенсорное — A0 живёт в теле
  mass: 160
  temperature: 9
  state: STATE_LOCKED
```

---

### A1 — Pattern (паттерн)

**Философия:** Первичный кластер. Повторяющееся различие из шума. Edge, stroke, tone. Ещё нет имени, нет значения — только "это появляется снова". Архетип: "Это встречается".

```yaml
  position: [5000, 5000, 4000]
  shell: [10, 25, 5, 0, 15, 0, 0, 5]
  # L2=25 сенсорное (паттерн воспринимаем), L5=15 когнитивный (распознавание)
  mass: 170
  temperature: 7
  state: STATE_LOCKED
```

### A2 — Entity (сущность)

**Философия:** Объект с постоянством. Кружка, которая остаётся кружкой при повороте. Человек, которого узнаёшь после разлуки. Идентичность сквозь изменения. Архетип: "Это — то же самое".

```yaml

  position: [8000, 8000, 7000]
  shell: [5, 15, 0, 5, 20, 0, 5, 10]
  # L5=20 когнитивный (постоянство = когнитивный конструкт)
  mass: 175
  temperature: 6
  state: STATE_LOCKED
```

---

### A3 — Category (категория)

**Философия:** Класс сущностей. "Это — собака" (а не "вот этот пёс"). Иерархия: собака → млекопитающее → животное. Категоризация — первый шаг к языку: слово = категория. Архетип: "Это относится к классу".

```yaml
- id: "abstraction_category"
  word: "категория"

  shell: [0, 5, 0, 0, 30, 5, 0, 15]
  # L5=30 когнитивный доминирует, L8=15 абстрактный (категория = уже отрыв от конкретного)
  mass: 180
  temperature: 5
  state: STATE_LOCKED
```

---

### A4 — Schema (схема)

**Философия:** Абстракция свойств и отношений. Не "что это" (A3), а "как это связано с другим". Роли: агент, пациенс, инструмент. Отношения: больше/меньше, причина/следствие. Schema — основа аналогии: "если A относится к B как X к Y, то..." Архетип: "Это — отношение".

```yaml
  position: [11000, 11000, 12000]
  shell: [0, 0, 0, 0, 35, 0, 0, 20]
  # L5=35 когнитивный, L8=20 абстрактный
  mass: 185
  temperature: 4
  state: STATE_LOCKED
```

**Нюанс:** A4 — главный уровень для аналогий и метафор. Когда система обнаруживает что два разных Frame-а имеют одинаковую Schema — возможен перенос знания между доменами.

### A5 — Theory (теория)

**Философия:** Полностью оторванная от конкретного структура. Правила, аксиомы, законы. Истинность внутренняя (дедуктивная), не эмпирическая. Теория групп работает и для поворотов куба, и для сложения чисел, и для симметрий физики. Архетип: "Это — закон".

```yaml
  shell: [0, 0, 0, 0, 25, 0, 5, 35]
  # L8=35 абстрактный доминирует, L5=25 когнитивный
  mass: 190

  state: STATE_LOCKED
```

### A6 — Constructor (конструктор)

**Философия:** Абстракция как инструмент. Способность создавать новые категории под задачу. Изобретать классификацию, которой раньше не было. Конструктивизм: не открываем мир, а строим описание мира. Архетип: "Я создаю категорию".

```yaml

  # L8=40 максимальный абстрактный
```

**Нюанс:** A6 — это уровень аксиогенеза (`Dilemmas_V1_0.md §5`). Когда система через разрешение дилеммы создаёт новую ценность — это Frame на уровне A6.

## 3. Отображение: уровни абстракции ↔ FrameComposition

| Уровень (Абстракции.md) | FrameComposition | Якорь AXIOM | Пример |
|---|---|---|---|
| L0 — Сенсорный шум | **C0** | abstraction_raw | edge detection, RGB pixels |
| L1 — Первичные кластеры | **C1** | abstraction_pattern | буква, нота, моральный сигнал |
| L2 — Объектная абстракция | **C2** | abstraction_entity | слово, мелодия, культурная норма |
| L3 — Категоризация | **C3** | abstraction_category | концепт, жанр, ценность |
| L4 — Свойства и отношения | **C4** | abstraction_schema | теорема, нарратив, дилемма |
| L5 — Мета-абстракции | **C5** | abstraction_theory | математическая система, этика |
| L6 — Абстрактные теории | **C5** | abstraction_theory | (часть C5) |
| L7 — Конструктивизм | **C5+** | abstraction_constructor | аксиогенез, новая категория |

**Важно:** L5 и L6 из Абстракции.md оба отображаются на C5 — разница между ними в глубине дерева внутри C5, не в отдельном FrameComposition уровне. C5+ = аксиогенный уровень.

## 4. Как FrameComposition вычисляется из дерева

```
Frame X  composed_of: [A, B, C]
  A  composed_of: [p1, p2]        ← C1 (1 уровень от примитивов)
  B  composed_of: [q1]            ← C1
  C  composed_of: [A, B]          ← C2 (2 уровня)

Глубина дерева X от C0:
  p1, p2, q1 — C0 якоря (Writing primitives)
  A, B       — C1 (глубина 1)
  C          — C2 (глубина 2)
  X          — C3 (глубина 3)
```

Правило: `composition_level(X) = max_depth(X.composed_of_tree_from_C0)`

Это **автоматически** вычисляется из `composed_of: Vec<u32>` (V7-A1). Никакой ручной разметки.

## 5. Изменение mass и temperature с уровнем абстракции

По мере роста абстракции Frame-ы:
- **Увеличивают mass** (притягивают больше — более стабильные концепты)
- **Снижают temperature** (менее хаотичны, более структурированы)

| FrameComposition | Типичная mass | Типичная temperature |
|---|---|---|
| C0 | 50–100 | 8–10 |
| C1 | 100–130 | 6–8 |
| C2 | 130–160 | 5–7 |
| C3 | 160–180 | 4–6 |
| C4 | 180–200 | 3–5 |
| C5 | 200–240 | 2–4 |
| C5+ | 240+ | 1–3 |

Эти диапазоны ориентировочны; точные значения определяются при создании Frame-анкоров через FrameWeaver.

## 6. Связь с другими подсистемами

### 6.1 С FrameWeaver

FrameWeaver V1.3 уже имеет `composition_level: FrameComposition`. V7-A1 добавляет `composed_of: Option<Vec<u32>>` — без этого вычисление глубины дерева невозможно. Abstractions_V1_0 **требует V7-A1**.

### 6.2 С Universal Grounding Stack

Абстракционные якоря A0–A6 — это "вертикальная ось" UGS. Горизонтальная ось — когнитивные домены (Writing, Time, Values...). Любой Frame находится на пересечении: доменный примитив × абстракционный уровень.

### 6.3 С Floor Lowering

Floor Lowering (`Primitives_And_Abstractions.md §4`) — это снижение пола на 1–3 порога: система временно опускается до более низкого уровня абстракции, чтобы пересобрать более точную картину. Это движение **вниз** по шкале A0–A6.

### 6.4 С Dilemmas

Аксиогенная дилемма (Dilemmas V Type) разрешается на уровне A6 (Constructor). Новая ценность = новый Frame на C5+. Это единственный путь наверх по оси абстракции через событие (не через накопление).

## 10. Домены AshtiCore

Путь C0→C5+ зеркалит движение токена через домены — это не метафора, а структурное совпадение.

| Уровень абстракции | Домен | Причина |
|---|---|---|
| A0 — Raw (C0) | **D8 VOID** | сырой сигнал без интерпретации — аномалия по умолчанию |
| A0 → A1 переход | **D5 PROBE** | PROBE активно исследует, превращая VOID в гипотезу |
| A1 — Pattern (C1) | **D5 PROBE** | гипотеза-паттерн рождается в исследовании |
| A2 — Entity (C2) | **D7 DREAM** | сущности с постоянством кристаллизуются в фоновой рекомбинации |
| A3 — Category (C3) | **D4 MAP** | подтверждённые категории = карта мира |
| A4 — Schema (C4) | **D6 LOGIC** | схемы отношений = дедуктивная структура |
| A5 — Theory (C5) | **D3 CODEX** | теории = законы системы, конституционный уровень |
| A6 — Constructor (C5+) | **D7→D10** | новые категории рождаются в DREAM, выводятся через MAYA |

**Полный путь абстракции:**
```
D8 VOID → D5 PROBE → D7 DREAM → D4 MAP → D6 LOGIC → D3 CODEX → D0 SUTRA
  A0           A1        A2         A3        A4          A5         (архив)
```

Этот путь — это не просто абстракция. Это **онтологический путь** любого знания в AXIOM: от аномалии через исследование и консолидацию до закона и абсолютной истины.
