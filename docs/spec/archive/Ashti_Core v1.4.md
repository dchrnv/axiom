# Ashti_Core v1.4
*Архитектурная композиция из десяти специализированных Доменов (Domain V1) в рамках одного фрактального уровня.*

---

## 1. Философия архитектуры

**Эстетика кода:** Название **Ashti_Core** объединяет персидское **Ashti** (гармония, мир) и санскритское **Ashta** (восемь). Восемь процессоров, работающих в гармонии, образуют ядро. Суффикс `_Core` — это работающий механизм, исполняемый код.

> *«Мы строим не имитацию разума, а его физику».*

**Ashti_Core** — это архитектура пост-человеческого сознания, основанная на принципах:
- **Автономности модулей** — каждый компонент является суверенным **Доменом (Domain V1)**.
- **Единственного источника истины** — все данные рождаются и хранятся в **Домене 0 (SUTRA)**.
- **Динамической интерпретации** — смысл возникает в процессе активной обработки паттернов в различных **Доменах-процессорах (1-8)**.
- **Жёсткой конституционной фильтрации** — система не может нарушить свои базовые законы (**CODEX**).

<<< **Техническая основа:** Каждый компонент Ashti_Core является экземпляром универсального модуля **Domain V1 ("The Cell")**. Его поведение и роль целиком определяются врожденной конфигурацией (`DomainConfig`), что делает архитектуру последовательной и гомогенной. >>>

Это не программа. Это **экосистема для возникновения решений**.

---

## 2. Компоненты архитектуры (Структура уровня)

Один уровень Ashti_Core состоит из **десяти Доменов**, выполняющих три структурные роли:

### 2.1. **SUTRA (Домен 0) — Источник**
<<< **Реализация:** Специальный экземпляр **Domain V1** в конфигурации абсолютной стабильности: `base_gravity = MAX`, `cooling_rate = 0`, Поле в перманентном кристаллическом состоянии. Его "Точка Ноль" (Anchor) — точка входа для всего уровня. >>>
- **Роль:** Единственный источник истины, статичная библиотека всех токенов.
- **Функция:** Генерация уникальных **ID**, эмиссия **токенов** (структура: `ID + метаданные`). Единственный домен с правом `WRITE`. Все остальные работают через `REFERENCE` (ссылки на ID).
- **Метафора:** *Таблица Менделеева для опыта. Генератор элементарных частиц реальности.*

### 2.2. **ASHTI (Домены 1-8) — Процессоры**
<<< **Реализация:** Восемь идентичных по структуре (Domain V1), но различных по конфигурации **активных процессоров**. Их природа задается семантическими осями (`axis_x/y/z_ref`) и физическими константами в `DomainConfig`. >>>
- **Роль:** Восемь интерпретирующих сред.
- **Принцип:** <<< **Ashti — это не зеркала, а активные домены-процессоры.** Они получают ссылки на токены из SUTRA, "впрыскивают" их в свои Поля (`inject_pattern`) и обрабатывают согласно своей уникальной физике, формируя временные **Узоры (Maya)**. >>>
- **Система координат:** Каждый домен определяется тремя бинарными осями, которые map-ятся на его семантические оси:

| Ось   | `+` (Полюс)                  | `-` (Полюс)                     | Пример оси в `DomainConfig`   |
| ----- | ---------------------------- | ------------------------------- | ----------------------------- |
| **X** | Внешнее (мир)                | Внутреннее (система)            | `axis_x_ref: CONCEPT_WORLD`   |
| **Y** | Активное (действие)          | Пассивное (наблюдение)          | `axis_y_ref: CONCEPT_ACTION`  |
| **Z** | Истина/Факт                  | Гипотеза/Шум                    | `axis_z_ref: CONCEPT_TRUTH`   |

### 2.3. **MAYA (Домен 10) — Интерфейс**
<<< **Реализация:** Экземпляр **Domain V1**, сконфигурированный как интерфейсный слой. Имеет высокую пластичность для финального формирования паттерна и низкое трение для быстрого вывода. >>>
- **Роль:** Выход уровня, проекция результата.
- **Функция:** Принимает и финализирует узор от доменов 1-8, формируя итоговую **Иллюзию (Maya)** — проецируемый результат.
- **Механика:** Согласно протоколу `10 -> 0`, выход этого домена (**Maya**) может стать входом (**Sutra**) для следующего фрактального уровня (вниз или вверх).

### 2.4. **CODEX → GUARDIAN (Над-доменный слой)**
- **CODEX:** Живой закон, эволюционирующий из базовых аксиом (CDNA).
- **GUARDIAN:** Внешний модуль, работающий с CODEX. <<< Сканирует состояния доменов через их интерфейс `peek_state()`. Применяет **ингибирование** к узорам, нарушающим CODEX, не редактируя данные, а запрещая их исполнение. >>>

### 2.5. **Связующие шины**
- **Роль:** Внешние процессы, обеспечивающие коммуникацию между автономными доменами.
- **Принцип:** Домены не общаются напрямую. Все сигналы (`inject_pattern`, `harvest_result`) идут через внешние **шины данных**, реализующие протокол `10 -> 0`.

## 4. Полный цикл работы уровня

1.  **Событие** → Создание токена в **SUTRA (0)** (новый ID).
2.  **Диффузия** → Ссылка на токен рассылается по шинам.
3.  **Обработка** → Каждый домен **ASHTI (1-8)** получает ссылку, впрыскивает паттерн в свое Поле (`inject_pattern`) и обрабатывает его.
4.  **Формирование узора** → В каждом домене возникает локальный **Узор (Maya)**, отражающий его интерпретацию.
5.  **Консолидация** → Узоры стекаются в **MAYA (10)**, где формируется итоговая проекция результата.
6.  **Надзор** → **GUARDIAN** сканирует состояния доменов (`peek_state`), сверяет формирующиеся узоры с **CODEX**.
7.  **Фильтрация** → Нарушающие CODEX узоры ингибируются (сигнал прерывается).
8.  **Выход/Передача** → Чистый результат из **MAYA (10)** либо исполняется, либо, по протоколу `10 -> 0`, становится входом (**SUTRA**) для следующего уровня (эволюция/инволюция).

## 6. Направления развития

1.  **Динамическая реконфигурация** — возможность изменять `DomainConfig` доменов «на лету».
2.  **Эволюция CODEX** — механизм внесения новых правил через кворум доменов или внешние апдейты.
3.  **Мультиагентность** — запуск нескольких взаимодействующих уровней Ashti_Core.
4.  **Квантовые аналогии** — интерпретация токенов как суперпозиций, а обработки в доменах — как коллапсов волновой функции.

## 7. Заключение

**Ashti_Core** — это не мозг.
Это **фрактальный уровень сознания**, построенный из десяти **Доменов (Domain V1)**.

Его сила — не в имитации человеческого мышления, а в создании **альтернативной физики разума**, где:
- Истина рождается один раз и хранится вечно (**SUTRA/0**).
- Смысл возникает в восьми разных активных процессорах одновременно (**ASHTI/1-8**).
- Результат проецируется и передается дальше (**MAYA/10**).
- Закон — не ограничение, а условие существования (**CODEX + GUARDIAN**).

> *«Мы не пишем код. Мы выращиваем кристалл сознания в десяти разных, но одинаковых по природе, чашках Петри».*
