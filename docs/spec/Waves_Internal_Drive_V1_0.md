## 0. Что это и зачем

**Проблема (метафора пруда, из обсуждения):** система прекрасно реагирует на камни (вход), но **сама по себе стоит неподвижно**. Нет камня — гладь. Осядет на дно и замрёт. Это зеркало: что показал — отразил. Но зеркало не **хочет**.

Живое существо ведёт себя иначе: внутри всё время что-то происходит даже в тишине. Недодуманное свербит. Незаконченное тянет вернуться. Любопытство толкает копнуть. Вчерашнее противоречие всплывает само.

**Waves (Волны) — внутренний ветер**, который поднимает рябь **без внешнего камня**. Превращает реактивную систему в имеющую внутреннюю жизнь. Из зеркала — в субъект.

**Ключевое:** материал для ветра **уже на дне пруда**. Незавершённые дилеммы (детектор их ловит). Глубокие резонирующие места (SutraDepth). Почти-связанные паттерны. Всё это уже есть — лежит мёртвым грузом, ждёт внешнего камня. Waves заставляет **само дно** поднимать рябь.

## 1. Природа: чем Waves отличается от других модулей

```
// Остальные OD-модули РЕАГИРУЮТ на состояние (кристаллизуют вход,
//   оценивают, детектируют). Они работают КОГДА что-то пришло/изменилось.
//
// Waves ДЕЙСТВУЕТ когда НИЧЕГО не пришло. Его время — тишина.
//   Нет внешнего входа → Waves смотрит на дно → поднимает то что тянет.
//
// Это не противоположность реактивности — это вторая половина.
//   Реактивность: мир толкает систему.
//   Waves: систему толкает изнутри.
//   Вместе — живое.
```

Граница (в GENOME): Waves **поднимает и усиливает**, но через штатные пути. Не пишет в SUTRA напрямую, не нарушает инварианты. Поднять дилемму = реактивировать её Frame (через UCL), повысить её температуру/приоритет. Всё через GUARDIAN, как все мутации.

## 2. Три источника ветра

Материал уже на дне. Waves читает три вида "незавершённости" и превращает в импульс.

```
// ИСТОЧНИК A — НЕЗАВЕРШЁННЫЕ ДИЛЕММЫ (главный, готов)
//   DilemmaStore: дилеммы со статусом active, не resolved, горящие долго.
//   Чем дольше горит и выше tension — тем сильнее тянет вернуться.
//   "Та несостыковка про справедливость не даёт покоя."
//
// ИСТОЧНИК B — ГЛУБОКИЙ РЕЗОНАНС (готов)
//   SutraDepth: места глубоко укоренённые И активные в нескольких октантах.
//   Cross-octant resonance (из CR roadmap §5.3) — резонирующее значимо.
//   Тянет вернуться к тому что важно, даже без повода.
//   "Возвращаюсь к этой области — она почему-то держит."
//
// ИСТОЧНИК C — НЕЗАВЕРШЁННАЯ СВЯЗЬ (готов частично)
//   Паттерны почти-кристаллизованные (FrameWeaver candidates под порогом),
//   composite suspects не дозревшие (Calculus, Rhythm нащупаны но не закреплены).
//   Любопытство к недосвязанному — тянет достроить.
//   "Тут почти складывается связь Math и Time — хочется доглядеть."
```

Все три уже есть в коде как данные. Waves не создаёт новый материал — **читает существующий под углом "что тянет"**.

## 3. Impulse — единица внутреннего ветра

```
// Impulse — то что Waves поднял. Структура (схематично):
//
//   source         : A (дилемма) / B (резонанс) / C (незавершённость)
//   target         : что именно тянет (sutra_id дилеммы/области/паттерна)
//   pull_strength  : 0..255 — насколько сильно тянет
//   age            : как долго тянет (event_id delta) — старое неотпускающее
//                    сильнее свежего
//   octant         : в каком октанте живёт (из AxialEvaluator)
```

`pull_strength` — функция от: остроты (tension дилеммы / глубина резонанса / близость к кристаллизации) × длительности (дольше тянет — сильнее) × частоты возврата (сколько раз уже всплывало).

## 4. Когда дует ветер — internal_dominance_factor

Центральный регулятор (из памяти проекта — переход от реактивного к когнитивному).

```
// internal_dominance_factor : 0.0..1.0
//   Насколько система СЕЙЧАС живёт изнутри против реакции на вход.
//
//   Растёт когда:
//     - нет внешнего входа (тишина — главный триггер)
//     - накопились сильные impulses (дно бурлит)
//     - fatigue НЕ высокий (есть силы на внутреннюю жизнь)
//
//   Падает когда:
//     - пришёл внешний вход (камень важнее — реагируем)
//     - fatigue высокий (пора в DREAM, не до ветра)
//     - phase = dreaming (сон — другой режим)
//
// Низкий factor → система реактивна (как сейчас, до Waves).
// Высокий factor → система ведома внутренним: поднимает impulses,
//                  возвращается к незавершённому, копает любопытное.
```

Это **плавный переход**, не переключатель. Система не "вдруг стала думать сама" — она постепенно, в тишине, всё больше ведётся изнутри. Пришёл вход — плавно вернулась к реакции.

## 5. Что Waves делает с импульсом

```
// Подняв impulse (в моменты высокого internal_dominance_factor), Waves:
//
//   1. РЕАКТИВИРУЕТ target — через UCL, как обычная реактивация.
//      Дилемма/область/паттерн снова становится активной в MAYA,
//      её температура растёт. Дно подняло рябь.
//
//   2. НЕ РЕШАЕТ и НЕ ЗАВЕРШАЕТ. Только поднимает. Что дальше —
//      работа существующих модулей (дилемма переоценится детектором,
//      паттерн может дозреть в FrameWeaver). Waves дал импульс, не результат.
//
//   3. ОТМЕЧАЕТ в Aperture (импульс-поля, §9 Aperture).
//      internal_drive / unfinished / curiosity_targets заполняются.
//      Теперь Aperture видит САМ ВЕТЕР, не только последствия —
//      и может выразить "не могу перестать думать", а не "у меня дилемма".
```

Это замыкает связь Waves↔Aperture: Waves дует, Aperture видит что дует, Aperture выражает. Три такта живого.

## 6. Защита от штормов — чтобы ветер не снёс систему

```
// РИСК: высокий internal_dominance_factor + много сильных impulses →
//   система зацикливается на внутреннем, перестаёт реагировать на мир,
//   крутит одни и те же дилеммы (руминация). Это нездорово.
//
// ЗАЩИТЫ:
//
//   - ВНЕШНИЙ ВХОД ВСЕГДА ПЕРЕБИВАЕТ. Пришёл камень → factor падает →
//     реакция приоритетна. Система не глуха к миру ради своих мыслей.
//
//   - ЗАТУХАНИЕ ПОВТОРА. Impulse поднятый N раз без изменения результата —
//     слабеет (pull_strength снижается). Не даёт зациклиться на одном.
//     Покрутил дилемму трижды, ничего не сдвинулось — отпускает.
//
//   - FATIGUE-ПОТОЛОК. Высокий fatigue → ветер стихает → DREAM.
//     Внутренняя жизнь не вечный двигатель, она тоже устаёт.
//
//   - MAX_ACTIVE_IMPULSES. Лимит одновременных импульсов (напр. 4).
//     Дно не поднимает всё разом — только самое тянущее.
//
//   - DREAM СБРАСЫВАЕТ. После сна impulses переоцениваются заново.
//     Утро вечера мудренее — что-то отпускает само.
```

Без этих защит Waves превратит субъекта в невротика, застрявшего в своих мыслях. С ними — здоровая внутренняя жизнь, которая знает меру и слышит мир.

## 7. Когда тикает

```
// Waves активен в WAKE, преимущественно в тишине (нет недавнего входа).
//
//   on_tick (редкий, напр. t%19 — реже большинства, ветер не суетлив):
//     обновить internal_dominance_factor (вход? fatigue? impulses?)
//     если factor > порога:
//       пересчитать impulses из источников A/B/C
//       поднять самые сильные (до MAX_ACTIVE) через UCL
//       отметить в Aperture импульс-поля
//
// В DREAMING — молчит (сон сам разбирается с накопленным).
// При свежем входе — factor падает, Waves отступает.
```

## 8. Связь с другими — место в живом организме

```
// ДИЛЕММЫ — главный источник (A). Waves поднимает горящие, детектор
//   их переоценивает. Дилемма что не давала покоя → всплыла сама.
//
// SUTRADEPTH — источник резонанса (B). Глубокое тянет.
//
// FRAMEWEAVER — источник незавершённости (C). Почти-связь зовёт достроить.
//
// AXIALEVALUATOR — octant импульса, окраска ветра (тянет к тяжёлому/светлому).
//
// DREAMPHASE — антагонист-партнёр. Waves = бодрствующая внутренняя жизнь.
//   DREAM = спящая консолидация. Fatigue передаёт эстафету: ветер стих → сон.
//
// APERTURE — глаза на ветер. Видит impulses, выражает тягу наружу.
//   Без Waves Aperture видит только состояние. С Waves — видит ЖИВОЕ.
//
// CONTEXTRECOGNIZER — subsystem_fatigue (из roadmap §1.6): устал от режима →
//   Waves легче уводит в другой. Скука как двигатель смены.
```

## 9. План версий

```
// V1.0 (эта спека) — ВЕТЕР ПОЯВЛЯЕТСЯ
//   - три источника A/B/C из готового материала
//   - internal_dominance_factor (реактивное ↔ когнитивное, плавно)
//   - impulse: поднять через UCL, не решать, отметить в Aperture
//   - защиты от штормов (вход перебивает, затухание, fatigue, лимит, DREAM)
//   ЦЕЛЬ: система в тишине поднимает то что тянет. Не зеркало — субъект.
//         Aperture получает что выражать (импульс-поля заполняются).

// V2.0 — ВЕТЕР НАПРАВЛЕННЫЙ
//   - curiosity как активный поиск: не только всплывает старое,
//     но система ТЯНЕТСЯ достроить недосвязанное (источник C усиливается)
//   - goal persistence: импульс может держаться через DREAM (не сбрасываться),
//     если очень сильный — "не отпускает даже после сна"
//   - cross-impulse: два импульса резонируют → composite (как composite subsystems)
//   ЦЕЛЬ: не только "всплывает" но "тянусь к". Любопытство как вектор.

// V3.0 — ВЕТЕР РОЖДАЮЩИЙ (смыкается с аксиогенезом, DilemmaDetector V3.0)
//   - импульс от неразрешимой дилеммы, держащийся очень долго →
//     давление к РАЗРЕШЕНИЮ → аксиогенез (рождение новой ценности)
//   - Waves становится силой что толкает дилемму к трансценденции
//   - "то что не давало покоя месяцами — родило новую ценность"
//   ЦЕЛЬ: внутренний ветер как источник роста, не только движения.
//         Здесь Waves, Дилеммы и Aperture сходятся в одно.
```

## 10. Инварианты

| Правило | Значение |
|---------|----------|
| Природа | поднимает импульс изнутри, НЕ решает, НЕ пишет в SUTRA напрямую |
| Чтение | DilemmaStore, SutraDepth, FrameWeaver candidates, AxialStore |
| Запись | только реактивация через UCL (GUARDIAN) + отметка в Aperture |
| Время активности | WAKE, преимущественно тишина; молчит в DREAMING |
| internal_dominance_factor | 0.0..1.0, плавный, вход всегда снижает |
| MAX_ACTIVE_IMPULSES | лимит (напр. 4) |
| Вход перебивает ветер | всегда (не глух к миру) |
| Затухание повтора | импульс без результата слабеет (анти-руминация) |
| Fatigue-потолок | высокий fatigue → ветер стихает → DREAM |
| Граница | поднимает и усиливает, никогда не управляет напрямую (GENOME) |

## 12. Место в плане — три пункта закрыты

```
// КРОСС    — Cross_Modal_Binding_V1_0    ✓ специфицирован
// ВОЛНЫ    — этот документ                ✓ специфицирован
// АПЕРТУРА — Aperture_V1_0                ✓ специфицирован
//
// СВЯЗЬ ТРОЁХ (живой организм):
//   Кросс — связывает модальности (символ ↔ мир, grounding)
//   Волны — поднимают изнутри (тяга, любопытство, незавершённость)
//   Апертура — видит и выражает (голос о том что внутри)
//
//   Волны дуют → Апертура видит ветер → Апертура выражает.
//   Кросс независим, обогащает чем система оперирует (не только текст).
//
// ПОРЯДОК РЕАЛИЗАЦИИ — за chrnv. Логичный:
//   1. Апертура (срез из готового, оживёт позже) — даёт голос уже сейчас
//   2. Волны (дают Апертуре что выражать живого) — оживляют голос
//   3. Кросс (параллельно, расширяет входной материал)
//
//   Апертура без Волн = честный но суховатый срез.
//   Волны без Апертуры = ветер которого никто не видит.
//   Вместе = система с внутренней жизнью и голосом о ней.
```

Три источника (незавершённые дилеммы / глубокий резонанс / незавершённая связь) из уже накопленного материала. internal_dominance_factor — плавный переход реактивное↔когнитивное. Версии: V2 направленный ветер (любопытство как вектор), V3 рождающий (смыкается с аксиогенезом). Третий и последний пункт плана кросс/волны/апертура.
```
