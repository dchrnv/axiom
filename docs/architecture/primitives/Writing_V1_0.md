## 0. Что эта спека описывает (и чего НЕ описывает)

Writing — это **одна из подсистем познания** в AXIOM. Не основа, не универсальный механизм, а специализированная подсистема, отвечающая за письменность всех языков мира.

В AXIOM будут существовать другие подсистемы познания (Mathematics, Time, Values, Dilemmas, Abstractions, Morality, Understanding) — каждая со своими примитивами, инвариантами, правилами композиции. Все они **накладываются на одно пространство SUTRA** (вариант "б" по архитектурному выбору), а не дублируют инфраструктуру.

**Эта спека:**
- Определяет 7 примитивов письменности
- Определяет правила композиции букв и иероглифов из примитивов
- Определяет, как TextPerceptor использует эти примитивы для восприятия текста

**Эта спека НЕ определяет:**
- Координатное пространство (это `INVARIANTS.md §6` и `SPACE_V6_0`)
- Оси Аполлон/Эрос/Воля (это `INVARIANTS.md §6`)
- АШТИ как гравитационный центр (это `INVARIANTS.md §6`, `SPACE_V6_0 §3.3`)
- Octants как архетипы (это общая семантика пространства)
- Механизм якорей и гравитации (это `SPACE_V6_0`)
- Shell L1-L8 (это `Shell_V3_0`)
- DomainConfig переопределения осей (это `DomainConfig_V2_1`)

Всё это **уже существует** и используется. Writing просто опирается на это.

## 1. Главный принцип Writing

**Все письменные системы Земли строятся из 7 атомарных графических форм.**

Это эмпирическое наблюдение (см. `Письменность.md`): латиница, кириллица, иероглифы (ханьцзы/кандзи), арабская вязь, хангыль, деванагари — все собираются из одного набора графических примитивов с разными правилами соединения, толщины и порядка начертания.

В AXIOM эти 7 примитивов живут в SUTRA (domain_id=100) как `STATE_LOCKED` якорные токены. Конкретная буква, иероглиф или символ — это **Frame**, кристаллизуемый FrameWeaver из этих примитивов.

---

## 2. Primitives-7: семь графических атомов

Все примитивы живут в SUTRA, имеют `mass` в диапазоне 160–200 (тяжелее обычных токенов 50-200, легче осевых якорей 255), `state = STATE_LOCKED`, низкую `temperature` (3–8 — структурное, не хаотичное).

Координаты примитивов — в положительной области `[0..32767]³` согласно `INVARIANTS.md §6` (все контентные якоря).

---

### P1 — dot (·) Точка

**Философия:** Чистое существование. Ноль измерений, только идентичность. Начало любого знака — перо касается бумаги. Архетип: "Я есть".

```yaml
- id: "prim_dot"
  word: "точка"
  aliases: ["dot", "начало", "семя"]
  tags: ["primitive", "writing", "p1_dot"]
  position: [0, 0, 6000]
  # X=0, Y=0 — точка вне дихотомий порядка и связи
  # Z=6000 — лёгкая Воля: точка заявляет о себе самим существованием
  shell: [10, 5, 0, 0, 20, 0, 0, 30]
  # L1=10 материальность, L5=20 "нечто одно", L8=30 абстрактный элемент
  mass: 200
  temperature: 5
  state: STATE_LOCKED
```

---

### P2 — vline (|) Вертикальная линия

**Философия:** Устойчивость, противостояние гравитации, ствол. 人 в японском начинается с vline. Архетип: "Я стою".

```yaml
- id: "prim_vline"
  word: "столб"
  aliases: ["vline", "вертикаль", "ствол", "стержень"]
  tags: ["primitive", "writing", "p2_vline"]
  position: [10000, 3000, 12000]
  # X=10000 Аполлон: вертикаль упорядочена
  # Y=3000 лёгкий Эрос: стержень поддерживает
  # Z=12000 Воля: стоять = утверждать себя
  shell: [30, 5, 20, 5, 5, 0, 5, 0]
  mass: 180
  temperature: 3
  state: STATE_LOCKED
```

---

### P3 — hline (—) Горизонтальная линия

**Философия:** Горизонт, покой, течение времени, основание. 一 (один) в китайском. Архетип: "Время идёт".

```yaml
- id: "prim_hline"
  word: "горизонт"
  aliases: ["hline", "горизонталь", "основание", "покой"]
  tags: ["primitive", "writing", "p3_hline"]
  position: [8000, 0, 1000]
  # X=8000 Аполлон: горизонталь упорядочена
  # Y=0 нейтральна
  # Z=1000 близко к АШТИ: горизонталь = лежачее = пассивное
  shell: [20, 5, 5, 0, 5, 0, 25, 5]
  # L7=25 темпоральное (горизонт = течение времени)
  mass: 180
  temperature: 3
  state: STATE_LOCKED
```

---

### P4 — dslash (/) Диагональ вверх-вправо

**Философия:** Движение, подъём, направленность вперёд. ノ (но) в японском, 掠 в каллиграфии. Архетип: "Движение к".

```yaml
- id: "prim_dslash"
  word: "подъём"
  aliases: ["dslash", "диагональ", "движение", "взлёт"]
  tags: ["primitive", "writing", "p4_dslash"]
  position: [3000, 5000, 10000]
  # X=3000 лёгкий Дионис: движение динамично
  # Y=5000 Эрос: движение к цели
  # Z=10000 Воля: активное действие
  shell: [10, 10, 25, 5, 5, 0, 10, 5]
  # L3=25 моторное (движение)
  mass: 160
  temperature: 8
  state: STATE_LOCKED
```

---

### P5 — bslash (\\) Диагональ вниз-вправо

**Философия:** Падение, нисхождение, завершение движения. 撇 (piě) в китайской каллиграфии. Архетип: "Движение от".

```yaml
- id: "prim_bslash"
  word: "спуск"
  aliases: ["bslash", "падение", "нисхождение", "завершение"]
  tags: ["primitive", "writing", "p5_bslash"]
  position: [3000, 1000, 8000]
  # X=3000 лёгкий Дионис
  # Y=1000 лёгкий Танатос: уход
  # Z=8000 Воля: ещё активно
  shell: [10, 10, 25, 5, 5, 0, 10, 0]
  mass: 160
  temperature: 8
  state: STATE_LOCKED
```

---

### P6 — hook (㇇) Крюк, поворот

**Философия:** Соединение, переход, узел. Концевые завитки каллиграфии. Архетип: "Я связываю".

```yaml
- id: "prim_hook"
  word: "крюк"
  aliases: ["hook", "поворот", "соединение", "узел"]
  tags: ["primitive", "writing", "p6_hook"]
  position: [6000, 8000, 5000]
  # X=6000 средний Аполлон: крюк структурирует, но имеет изгиб
  # Y=8000 Эрос: крюк создаёт связь между элементами
  # Z=5000 средняя Воля
  shell: [10, 5, 15, 0, 10, 5, 0, 10]
  # L3=15 моторное (поворот), L6=5 социальное (связь)
  mass: 170
  temperature: 5
  state: STATE_LOCKED
```

---

### P7 — arc (⌒) Дуга

**Философия:** Непрерывность, плавность, цикл. Дуга окружности, изгиб. Архетип: "Я плавно меняюсь".

```yaml
- id: "prim_arc"
  word: "дуга"
  aliases: ["arc", "кривая", "изгиб", "цикл"]
  tags: ["primitive", "writing", "p7_arc"]
  position: [4000, 6000, 6000]
  # X=4000 средний Дионис: кривизна = плавная динамика
  # Y=6000 Эрос: дуга соединяет, охватывает
  # Z=6000 средняя Воля
  shell: [10, 15, 10, 5, 10, 0, 5, 10]
  # L2=15 сенсорное (плавность ощутима), L7=5 темпоральное (изменение во времени)
  mass: 170
  temperature: 6
  state: STATE_LOCKED
```

## 3. Композиция: от примитивов к буквам, иероглифам, словам

### 3.1 Буква = Frame первого уровня

Буква — это Frame, кристаллизуемый FrameWeaver из 2–8 примитивов. Frame-анкер живёт в EXPERIENCE (domain_id=109) с `TOKEN_FLAG_FRAME_ANCHOR` (см. `INVARIANTS.md §9`).

Примеры композиции:

```
A = dslash + bslash + hline           (3 примитива)
R = vline + arc + dslash               (3 примитива)
Ж = vline + dslash + dslash + bslash + bslash  (5 примитивов)

人 (человек)  = dslash + bslash           (2 примитива, японский/китайский)
山 (гора)     = vline + vline + vline + hline  (4 примитива)
永 (вечность) = dot + hline + vline + dslash + hook + bslash + arc + dslash  (8 примитивов)

ب (ба, арабский) = hline + dot
لا (лам-алиф)    = vline + vline + dslash (связка)

Математические символы (используются те же примитивы):
+ = vline + hline
= = hline + hline
∫ = arc + vline
```

**Принципиально:** один и тот же примитив `vline` используется и в `R` (латиница), и в `Ж` (кириллица), и в `山` (иероглиф), и в `+` (математический оператор). Это нормально — примитивы универсальны, специфика возникает на уровне Frame и его связей.

### 3.2 Слово = Frame второго уровня

Слово — это Frame из Frame-ов букв. Позиция слова рассчитывается как центроид букв + гравитация семантических якорей.

### 3.3 Концепт = Frame третьего уровня

"Вечность" (английское `eternity`) и `永远` (китайское) — разные пути в графе примитивов, но финальная позиция Frame "Вечность" близкая для обоих языков, потому что оба притягиваются к одним семантическим якорям (L7 Temporal + L8 Abstract в Shell).

Это и есть смысл AXIOM: язык не имеет значения, значение имеет **семантическая гравитация**.

## 4. TextPerceptor: путь от текста к примитивам

### 4.1 Текущий путь (FNV-1a fallback)

```
текст → FNV-1a(текст) & 0x7FFF → позиция в [0..32767]³ → не семантично
```

Это работает, но не использует примитивы. Текст просто хешируется в позицию.

### 4.2 Целевой путь (после внедрения primitives.yaml)

```
текст → разбиение на символы
     → каждый символ → разбиение на примитивы
     → позиция символа = центроид примитивов + гравитация Shell-якорей
     → позиция слова = центроид символов + гравитация семантических центров
```

Это требует:
1. Файла `config/anchors/primitives.yaml` с 7 примитивами Writing
2. Таблицы декомпозиции символов в примитивы (по каждой письменной системе)
3. Изменений в `TextPerceptor` для использования этой таблицы

### 4.3 Промежуточный путь (уже работает)

```
текст → совпадение с якорем → позиция якоря (семантично)
      → иначе → FNV-1a fallback в [0..32767]³
```

Этот путь продолжает работать, пока примитивы не внедрены.

## 5. Сопряжение с другими подсистемами познания

Writing — одна из подсистем. Другие подсистемы (Mathematics, Time, Values, и т.д.) будут добавлены отдельными спецификациями. Все они **живут в одном SUTRA**.

### Принцип сосуществования

Один и тот же токен может одновременно интерпретироваться разными подсистемами:

- Символ `X` в письменности = латинская буква X (Writing)
- Символ `X` в математике = переменная (Mathematics)
- Символ `X` во времени = неизвестный момент (Time)

В архитектуре варианта "б" (см. главный архитектурный выбор) интерпретация определяется **активным контекстом** обработки. Контекст указывает, какие cognitive domain активны.

### Будущее направление (вариант "в", отложено)

В будущей версии каждый токен сможет иметь **профиль по cognitive domains** — отдельную структуру (типа Shell V3.0, но другого назначения), показывающую, насколько токен принадлежит каждому из cognitive доменов. Это даст:

- Постепенное добавление новых cognitive domains без переписывания
- Лучшее различение "это математика" от "это математика и письменность"
- Более тонкую работу с многозначностью

Сейчас этого нет, но архитектура не закрывает путь к этому. Это направление развития.

7 якорей по схеме раздела 2. Инжектируется при старте через существующий механизм `inject_anchor_tokens` (см. `Anchor_Tokens_V1_0`).

```rust
    pub axes: Vec<Anchor>,
    pub layers: Vec<Anchor>,
    pub domains: Vec<Anchor>,
    pub primitives: Vec<Anchor>,  // НОВОЕ

```

### 6.3 Таблица декомпозиции в TextPerceptor

```rust
pub struct CharDecomposition {
    pub character: char,
    pub primitives: Vec<&'static str>,  // id примитивов
}
```

Таблица создаётся для часто встречающихся символов (латиница, кириллица, базовые иероглифы). Полное покрытие — задача будущих версий.

### 6.4 Алгоритм TextPerceptor

```
fn text_to_positions(text: &str) -> Vec<TokenPosition> {
    for char in text.chars() {
        if let Some(decomp) = lookup_decomposition(char) {
            let primitive_positions = decomp.primitives.iter()
                .map(|id| get_primitive_anchor(id).position)
                .collect();
            let centroid = compute_centroid(primitive_positions);
            // дальше: гравитация Shell-якорей корректирует позицию
            positions.push(centroid);
        } else {
            // fallback на FNV-1a
            positions.push(fnv1a_position(char));
        }
    }
}
```

## 9. Домены AshtiCore

Примитивы Writing не изолированы — каждый этап их жизненного цикла проходит через конкретный домен AshtiCore.

| Примитив / процесс | Домен | Причина |
|---|---|---|
| P1–P7 (все примитивы) | **D0 SUTRA** | STATE_LOCKED якоря — кристаллизованная память |
| TextPerceptor: обработка входа | **D1 EXECUTION** | реализация "здесь и сейчас", контакт с реальностью |
| Нераспознанный символ (FNV fallback) | **D8 VOID** | нет резонанса ни с одним якорем → аномалия |
| Гипотеза нового Frame | **D5 PROBE** | PROBE активно исследует незнакомые паттерны |
| Консолидация Frame | **D7 DREAM** | фоновая рекомбинация → устойчивые связи |
| Подтверждённый Frame (буква, слово) | **D4 MAP** | факт принят в карту реальности |
| Валидация по правилам письменности | **D3 CODEX** | конституционный фильтр проверяет соответствие |
| Омонимия, амбигуация | **D2 SHADOW** | симулятор угроз генерирует альтернативные интерпретации |
| Итоговая форма текста | **D10 MAYA** | финализация + проекция вышестоящему уровню |

**Критический путь символа:**
```
Входной символ → D1 EXECUTION → (известен?) → D4 MAP / D0 SUTRA
                                 ↓ (неизвестен)
                           D8 VOID → D5 PROBE → D7 DREAM → D4 MAP
```
