# AXIOM — Semantic Foundation V1.0

**Статус:** Концептуальная спецификация (не реализована)  
**Дата:** 2026-05-13  
**Назначение:** Детальное описание семантического фундамента: трёхосевое пространство,
семь примитивных токенов, схема композиции, якорное размещение.  
**Основано на:** `docs/architecture/Как объяснить машине.md`, `docs/spec/такое 2/`

---

## 1. Главный принцип

Все смыслы — числа, буквы, понятия, эмоции — строятся из одних и тех же **семи
атомарных форм** (Primitives-7). Они живут в SUTRA как тяжёлые якорные токены.
Более сложные объекты кристаллизуются через FrameWeaver как Frame-ы из примитивов.

Три оси пространства имеют философское значение — не произвольная геометрия, а
измерения смысла. Место токена в пространстве — это его семантическая характеристика.

Итог: **буква, иероглиф, математическая операция и абстрактное понятие** — это разные
комбинации одних и тех же примитивов в одном пространстве. "Вечность" по-английски и
"永远" по-китайски притягиваются к одной точке — потому что собраны из одних атомов.

## 2. Трёхосевое пространство (уже реализовано в axes.yaml)

Token.position = [X: i16, Y: i16, Z: i16], нормализованные координаты [-1.0 .. +1.0].

### Ось X: Аполлон (+) ↔ Дионис (-)

| Полюс | Координата | Смысл | Метрика |
|-------|-----------|-------|---------|
| Аполлон (+30000) | X = +1.0 | Порядок, структура, симметрия, предсказуемость | Низкая энтропия |
| Дионис (-30000) | X = -1.0 | Хаос, спонтанность, поток, случайность | Высокая энтропия |

**Примеры:**
- Математическое доказательство → X≈+0.9
- Поэзия (сонет) → X≈+0.5 (форма есть, но есть свобода)
- Поток сознания → X≈-0.7
- Белый шум → X≈-1.0

**В Axiom:** ось X = `temperature` (горячее = хаотичнее) + плотность синтаксических
связей в Shell L5. Токены с низкой temperature дрейфуют к Аполлону; высокой — к Дионису.

---

### Ось Y: Эрос (+) ↔ Танатос (-)

| Полюс | Координата | Смысл | Метрика |
|-------|-----------|-------|---------|
| Эрос (+30000) | Y = +1.0 | Связь, синтез, рост, созидание, притяжение | Плотность связей |
| Танатос (-30000) | Y = -1.0 | Разрыв, разрушение, изоляция, забвение | Разрежение связей |

**Примеры:**
- "Я люблю" → Y≈+0.9
- "Мы строим" → Y≈+0.7
- "Он ушёл" → Y≈-0.3 (нейтральное разделение)
- "Убить", "забыть", "удалить" → Y≈-0.8

**В Axiom:** ось Y = `valence` поля + количество активных Connection у токена.
Токены с высоким valence и плотными связями дрейфуют к Эросу.

---

### Ось Z: Воля (+) ↔ Ничто (-)

| Полюс | Координата | Смысл | Метрика |
|-------|-----------|-------|---------|
| Воля (+30000) | Z = +1.0 | Сила сигнала, агентность, императив, присутствие | Высокая magnitude |
| Ничто (-30000) | Z = -1.0 | Тишина, пассивность, отсутствие, нулевой сигнал | Низкая magnitude |

**Примеры:**
- "Сделай это!" → Z≈+0.95 (императив)
- "Я есть" → Z≈+0.8
- "Оно случилось..." (пассивный залог) → Z≈-0.3
- Тишина, пустота, "меня нет" → Z≈-0.9

**В Axiom:** ось Z = `mass` токена + `state` (Active vs Sleeping).
Тяжёлые активные токены дрейфуют к Воле; лёгкие спящие — к Ничто.

---

### Восемь октантов (комбинации осей)

Комбинируя три полюса, получаем 8 семантических октантов. Каждый — архетип:

| Октант | X | Y | Z | Архетип | Пример |
|--------|---|---|---|---------|--------|
| +++ | Аполлон | Эрос | Воля | **Величие** | Военный парад, архитектура |
| ++- | Аполлон | Эрос | Ничто | **Мудрость** | Медитация, созерцание |
| +-+ | Аполлон | Танатос | Воля | **Власть** | Закон, суд, запрет |
| +-- | Аполлон | Танатос | Ничто | **Равновесие** | Смерть естественная, покой |
| -++ | Дионис | Эрос | Воля | **Экстаз** | Танец, праздник, любовь |
| -+- | Дионис | Эрос | Ничто | **Мечта** | Сон, фантазия |
| --+ | Дионис | Танатос | Воля | **Ярость** | Разрушение, взрыв, гнев |
| --- | Дионис | Танатос | Ничто | **Пустота** | Депрессия, небытие |

> Когда FrameWeaver кристаллизует Frame, его anchor-токен попадает в один из октантов.
> Это и есть его "характер" — не заданный вручную, а вычисленный из позиций участников.

## 3. Primitives-7: семь атомарных токенов

Эти семь форм — **минимальный универсальный базис** всех письменных систем Земли.
Латиница, кириллица, арабская вязь, китайские иероглифы, хангыль, деванагари —
все строятся из комбинаций этих форм. Математические символы — тоже.

Они живут в SUTRA (domain_id=100) как STATE_LOCKED якорные токены с большой массой.
Всё остальное кристаллизуется из них через FrameWeaver.

---

### P1 — dot (·) Точка

**Философия:** Чистое существование. Ноль измерений, только идентичность.  
В математике = элемент множества. В алгебре = точка на многообразии.  
Начало любого знака — перо касается бумаги.

**Архетипическое значение:** "Я есть". Первичное самоутверждение бытия.

```yaml
- id: "prim_dot"
  word: "точка"
  aliases: ["dot", "начало", "семя", "элемент", "есть"]
  tags: ["primitive", "p1_dot"]
  position: [0, 0, 6000]
  # Центр пространства (не порядок, не хаос),
  # слегка к Воле (Z+) — потому что точка ЕСТЬ, она заявляет о себе.
  shell: [10, 5, 0, 0, 20, 0, 0, 30]
  # L1=10 (физическое присутствие — маленькое, но есть)
  # L5=20 (когнитивный объект — "нечто одно")
  # L8=30 (абстрактная идентичность — "элемент как таковой")
  mass: 200
  temperature: 5
  state: STATE_LOCKED
  description: "Атомарная точка. Семя всех форм."
```

---

### P2 — vline (|) Вертикальная линия

**Философия:** Устойчивость, ствол, присутствие. Вертикаль = противостояние гравитации,
воля стоять. Ствол дерева, колонна, человеческая фигура (人 в японском = vline + dslash).

**Архетипическое значение:** "Я стою". Аполлоническая сила, организованная вертикально.

```yaml
- id: "prim_vline"
  word: "столб"
  aliases: ["vline", "вертикаль", "ствол", "стержень", "ось"]
  tags: ["primitive", "p2_vline"]
  position: [10000, 3000, 12000]
  # X+ (Аполлон) — вертикаль упорядочена, это структура.
  # Y+ немного (Эрос) — стержень держит, поддерживает.
  # Z+ сильно (Воля) — стоять вертикально = утверждать себя.
  shell: [30, 5, 20, 5, 5, 0, 5, 0]
  # L1=30 (сильно физическое — опора, материя)
  # L3=20 (мотор — держать позицию)
  mass: 180
  temperature: 3
  state: STATE_LOCKED
  description: "Вертикаль. Стержень. Физическое присутствие."
```

---

### P3 — hline (—) Горизонтальная линия

**Философия:** Горизонт, покой, течение времени, основание. Горизонт — это место где
небо встречает землю. Лежащее тело. Река. В китайском 一 (один) = hline.

**Архетипическое значение:** "Время идёт". Аполлоническая стабильность, но пассивная.

```yaml
- id: "prim_hline"
  word: "горизонт"
  aliases: ["hline", "горизонталь", "основание", "покой", "река"]
  tags: ["primitive", "p3_hline"]
  position: [8000, 0, -5000]
  # X+ (Аполлон) — горизонталь упорядочена, чёткая.
  # Y≈0 — нейтральна по Эросу/Танатосу.
  # Z- (лёгкое Ничто) — горизонтальное = лежачее = пассивное.
  shell: [20, 5, 5, 0, 5, 0, 25, 5]
  # L1=20 (физическая основа — земля, пол)
  # L7=25 (временная — горизонт = время, течение)
  mass: 180
  temperature: 3
  state: STATE_LOCKED
  description: "Горизонталь. Горизонт. Основание времени."
```

---

### P4 — dslash (/) Диагональ вверх-вправо

**Философия:** Движение, подъём, причинность направленная вперёд. Нож рассекающий
пространство. В японском ノ (но). Ноги идущего человека.

**Архетипическое значение:** "Движение к". Активная направленность.

```yaml
- id: "prim_dslash"
  word: "подъём"
  aliases: ["dslash", "диагональ", "движение", "вперёд", "взлёт"]
  tags: ["primitive", "p4_dslash"]
  position: [-3000, 5000, 10000]
  # X- (лёгкий Дионис) — движение динамично, не статично как столб.
  # Y+ (Эрос) — движение к чему-то, направленность = связь с целью.
  # Z+ (Воля) — активное действие.
  shell: [10, 10, 25, 5, 5, 0, 10, 5]
  # L3=25 (мотор — это движение прежде всего)
  # L2=10 (сенсорное — кинестетика движения)
  # L7=10 (временное — движение = изменение во времени)
  mass: 160
  temperature: 8
  state: STATE_LOCKED
  description: "Диагональ /. Подъём, движение вперёд."
```

---

### P5 — bslash (\\) Диагональ вниз-вправо

**Философия:** Падение, нисхождение, завершение движения. Рука опускается. Дождь.
В китайской каллиграфии 撇 (piě) — откидная черта влево.

**Архетипическое значение:** "Движение от". Завершение, спуск.

```yaml
- id: "prim_bslash"
  word: "спуск"
  aliases: ["bslash", "падение", "нисхождение", "отдача", "завершение"]
  tags: ["primitive", "p5_bslash"]
  position: [-3000, -5000, 8000]
  # X- (лёгкий Дионис) — снижение = движение к энтропии.
  # Y- (лёгкий Танатос) — спуск, уход от точки притяжения.
  # Z+ (Воля) — всё ещё активно, это не пассивность, это завершение.
  shell: [10, 10, 25, 10, 5, 0, 10, 5]
  # L3=25 (мотор — движение)
  # L4=10 (эмоциональное — падение несёт оттенок потери)
  mass: 160
  temperature: 8
  state: STATE_LOCKED
  description: "Диагональ \\. Спуск, завершение движения."
```

---

### P6 — hook (⌐) Крюк / поворот

**Философия:** Место где одно соединяется с другим. Сустав, крюк, узел.
В математике = операция (сложение, умножение — символы соединяют).
В китайском 趯 (tiào) — крюк, базовая черта.

**Архетипическое значение:** "Соединение". Самый эросный из примитивов.

```yaml
- id: "prim_hook"
  word: "соединение"
  aliases: ["hook", "крюк", "узел", "операция", "сустав", "стык"]
  tags: ["primitive", "p6_hook"]
  position: [5000, 12000, 5000]
  # X+ (Аполлон) — соединение структурно, оно следует правилам.
  # Y+ сильно (Эрос) — это буквально элемент связи.
  # Z+ умеренно (Воля) — активное действие соединения.
  shell: [5, 5, 20, 10, 15, 25, 0, 10]
  # L6=25 (социальное — это точка встречи, отношение)
  # L5=15 (когнитивное — операция как концепт)
  # L3=20 (мотор — физическое действие соединения)
  mass: 170
  temperature: 5
  state: STATE_LOCKED
  description: "Крюк/поворот. Точка соединения двух линий."
```

---

### P7 — arc (⌢) Дуга / кривая

**Философия:** Непрерывность, объятие, волна, предел. Кривая не знает углов — она
всегда плавно переходит. В математике = предел, непрерывная функция. Полукруг = объятие.
В арабской вязи дуга — основа большинства букв.

**Архетипическое значение:** "Течение". Наименее аполлонический из примитивов.

```yaml
- id: "prim_arc"
  word: "волна"
  aliases: ["arc", "дуга", "кривая", "объятие", "поток", "непрерывность"]
  tags: ["primitive", "p7_arc"]
  position: [-2000, 8000, 0]
  # X- (лёгкий Дионис) — кривая = отклонение от прямолинейного порядка.
  # Y+ (Эрос) — дуга обнимает, охватывает, притягивает к себе.
  # Z≈0 — нейтральна по воле, это сам процесс, не усилие.
  shell: [5, 20, 5, 15, 10, 5, 20, 15]
  # L2=20 (сенсорное — кривая ощущается плавностью)
  # L4=15 (эмоциональное — объятие, тепло)
  # L7=20 (временное — волна = ритм, поток во времени)
  # L8=15 (абстрактное — непрерывность как понятие)
  mass: 160
  temperature: 12
  state: STATE_LOCKED
  description: "Дуга. Кривая. Непрерывность и объятие."
```

## 4. Состав: как из примитивов строятся знаки

### 4.1 Буква = Frame первого уровня

FrameWeaver кристаллизует букву как Frame из нескольких примитивных токенов.
Буква не хранится в словаре — она возникает как стабильный паттерн.

**Примеры (Латиница/Кириллица):**

```
A = dslash + bslash + hline
    (две диагонали сходятся вверху, горизонтальная перекладина)

O = arc + arc
    (две дуги, образующие замкнутый контур)

I = vline
    (один вертикальный штрих)

R = vline + arc + dslash
    (столб + горбик + ножка)

T = hline + vline
    (горизонталь сверху + вертикаль вниз)
```

**Китайские иероглифы:**

```
一 (один) = hline
二 (два)  = hline + hline
三 (три)  = hline + hline + hline
人 (человек) = dslash + bslash
山 (гора)    = vline + vline + vline + hline (три пика)
永 (вечность) = dot + hline + vline + dslash + hook + bslash + arc + dslash
```

**Математические символы:**

```
+ = vline + hline    (крест из двух линий)
= = hline + hline    (два горизонтальных)
∞ = arc + arc        (два соединённых кольца = два arc через hook)
∫ = arc + vline      (вытянутая S-образная кривая)
√ = dslash + hline   (диагональ + горизонтальная крыша)
```

> Инсайт: математические символы и буквы — это **одни и те же комбинации** одних
> примитивов. Axiom не различает "текст" и "математику" на уровне примитивов.

### 4.2 Слово = Frame второго уровня

Frame из Frame-ов первого уровня. Слово "кот" = [буква_К + буква_О + буква_Т].
Позиция слова в пространстве = центроид всех входящих букв, смещённый гравитацией
семантических якорей.

### 4.3 Концепт = Frame третьего уровня

"Вечность" = Frame из слов-Frame-ов, притянутый к якорям L7 (Temporal) + L8 (Abstract).
"永远" (китайский) и "eternity" (английский) — разные пути в графе примитивов,
но конечная позиция в пространстве у них близкая: оба притягиваются к одним якорям.

## 5. Гравитация как вывод / доказательство

### Идея

Семантические "истины" — это тяжёлые якоря (mass >> 1000) в SUTRA. Лёгкий токен-
гипотеза бросается в пространство и за N тиков притягивается к ближайшей "истине".
Это и есть вывод: не алгоритм, а физика.

### Пример: математическое утверждение

```
Якоря в SUTRA:
  "истина"   → position=[15000, 15000, 15000], mass=5000  (+++: Аполлон + Эрос + Воля)
  "ложь"     → position=[-15000,-15000,-15000], mass=5000 (---: Дионис+Танатос+Ничто)
  "сложение" → position=[8000, 5000, 8000],   mass=800
  "равенство"→ position=[10000, 0, 3000],     mass=800

Гипотеза "2 + 2 = 5":
  Токен создаётся из примитивов: dot×2 + cross + dot×2 + hline×2 + dot×5
  За N тиков:
    → притягивается к "сложение" и "равенство"
    → но числа-якоря "4" и "5" находятся далеко друг от друга
    → конфликт → token движется к "ложь"

Гипотеза "2 + 2 = 4":
  → та же начальная точка
    → числа-якоря согласованы
    → движется к "истина"
```

### Практически для Axiom

Сейчас: TickForward обновляет position через gravity_batch (apply_gravity_batch).
Когда якоря расставлены правильно — это **уже работает**.
Нужно только: (1) добавить примитивные якоря, (2) правильно декомпозировать входной текст.

```
Этап 4:            layers/ L1-L4, L6-L8 (продолжение текущего L5)
Этап 5:            Embeddings в TextPerceptor (после OBS-01)
```

## 7. Как TextPerceptor использует это сейчас

**Текущий путь (FNV-1a fallback):**
```
текст → FNV-1a(текст) → произвольная позиция → не семантично
```

**Путь с примитивами (после внедрения):**
```
текст → разбиение на символы → каждый символ → декомпозиция в примитивы
     → позиция символа = центроид его примитивов, смещённый якорями Shell
     → позиция слова = центроид символов + гравитация семантических центров
```

**Промежуточный путь (якорное позиционирование уже работает):**
```
текст → если совпадает с якорем → позиция якоря (семантично)
      → иначе → FNV-1a (не семантично, но системой не ломается)
```

Чем больше якорей — тем меньше FNV-1a fallback, тем семантичнее пространство.

## 8. Для реализаторов (технические детали)

### Как читать Shell-профиль примитива

Shell [L1..L8] = [0..255]. Каждый Layer:
- L1 Physical: 255 = "это тело, вещь, материя"
- L2 Sensory: 255 = "это ощущение, восприятие"
- L3 Motor: 255 = "это движение, действие"
- L4 Emotional: 255 = "это чувство, заряд"
- L5 Cognitive: 255 = "это концепт, знание"
- L6 Social: 255 = "это отношение, связь между субъектами"
- L7 Temporal: 255 = "это ритм, время, последовательность"
- L8 Abstract: 255 = "это символ, чистая идея"

### Почему mass=160..200 для примитивов

Обычный инжектированный токен: mass ≈ 5-20.  
Семантические якоря осей: mass определяется AnchorSet.load() как константа.  
Примитивы должны быть тяжелее обычных токенов (иначе смоются), но легче осей
(иначе задавят пространство). Диапазон 160-200 — экспериментальная оценка.

### STATE_LOCKED

Примитивные якоря помечаются STATE_LOCKED = они не мутируются рефлексами Guardian,
не участвуют в Dream (не предлагаются на промоцию — они уже в SUTRA).
Это обеспечивается флагом `TOKEN_FLAG_FRAME_ANCHOR` + `STATE_LOCKED` при inject.

### lineage_hash для примитивов

Frame-якоря (TOKEN_FLAG_FRAME_ANCHOR) хранят в lineage_hash не путь,
а FNV-1a(sorted participant sutra_ids). Это их идентификатор.
Для примитивных токенов (они не Frame) lineage_hash = 0 (дефолт).
