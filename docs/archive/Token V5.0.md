### 1. TOKEN V5.0 (Голографическая Частица)

**Философия:** Токен больше не знает обо всех 8 измерениях сразу. Он знает только свое положение в текущем **Домене**. Освободившееся место мы отдали под **Lineage** (Историю пути) и **Momentum** (Инерцию).

**Размер:** 64 Байта (Строго).

Rust

```
#[repr(C, align(64))]
pub struct Token {
    // --- ИДЕНТИФИКАЦИЯ (8 Байт) ---
    // Ссылка на вечную запись в Библиотеке (Sutra)
    pub sutra_id: u32,       // Глобальный ID смысла ("Яблоко")
    // В каком Домене я сейчас нахожусь? (Logic, Dream...)
    pub domain_id: u16,      
    // Тип сущности (Object, Relation, Pattern...)
    pub type_flags: u16,     

    // --- ЛОКАЛЬНАЯ ФИЗИКА ПОЛЯ (16 Байт) ---
    // Где я сейчас? (X, Y, Z в текущем Field)
    pub position: [i16; 3],  
    // Куда я движусь? (Вектор скорости) - НОВОЕ!
    // Позволяет симулировать удар, отскок, притяжение.
    pub velocity: [i16; 3],  
    // Целевая точка (Attractor) - куда меня тянет?
    pub target: [i16; 3],    
    pub reserved_phys: u16,  // Паддинг

    // --- ТЕРМОДИНАМИКА (4 Байта) ---
    // Valence: Эмоциональный заряд (-127..+127)
    pub valence: i8,         
    // Mass: Гравитационный вес (1..255). Тяжелое тонет вниз.
    pub mass: u8,            
    // Temp: Пластичность (0..255). Горячее мутирует.
    pub temperature: u8,     
    // State: Active, Sleeping, Locked...
    pub state: u8,           

    // --- ФРАКТАЛЬНАЯ НАВИГАЦИЯ (36 Байт) ---
    // Самое важное изменение!
    // Вместо координат других слоев, мы храним "Нить Ариадны".
    
    // Hash пути: откуда я пришел? (Sutra -> Logic -> Math...)
    // Позволяет вернуться "наверх" (Lift Up).
    pub lineage_hash: u64,   

    // "Сжатый контекст". Это 28 байт, куда сворачиваются 
    // данные с предыдущих уровней. (Holographic Echo).
    pub context_payload: [u8; 28], 
}
```

#### Что изменилось по сравнению с твоим V4.1:

1. **Убрали:** `coordinates: [[i16; 3]; 8]` (48 байт). В текущем Домене нам не нужны координаты из соседних.
    
2. **Добавили:** `velocity` (Скорость). Теперь `GridHash` может предсказывать коллизии. Токены реально "летят" друг к другу.
    
3. **Добавили:** `lineage_hash`. Токен знает свой "адрес возврата". Это билет на Лифт Вверх.
    
4. **Добавили:** `context_payload`. Это "рюкзак". Когда токен падает вниз, он пакует важные параметры сверху в этот буфер.

### 2. CONNECTION V5.0 (Вектор Напряжения)

**Философия:** Связь — это не просто линия. Это **Пружина** или **Труба**.

Она имеет натяжение (`tension`) и пропускную способность (`permeability`).

**Размер:** 64 Байта.

Rust

```
#[repr(C, align(64))]
pub struct Connection {
    // --- ТОПОЛОГИЯ (16 Байт) ---
    pub source_id: u32,      // От кого (Sutra ID)
    pub target_id: u32,      // К кому (Sutra ID)
    pub domain_id: u16,      // В каком Домене эта связь существует?
    pub link_type: u16,      // (Synonym, Cause, Vertical_Drop...)
    pub flags: u32,          // (Active, Inhibited, Temporary...)

    // --- ДИНАМИКА (16 Байт) ---
    // Насколько сильно связь тянет (0.0 - 1.0)
    pub strength: f32,       
    // Текущее напряжение (стресс). Если > max, связь рвется.
    pub current_stress: f32, 
    // Длина пружины (идеальное расстояние)
    pub ideal_dist: f32,     
    // Коэф. упругости (как быстро возвращает)
    pub elasticity: f32,     

    // --- ШЛЮЗ (16 Байт) ---
    // Используется для Вертикальных связей (Лифтов)
    
    // Порог массы: Пропускать только если mass > X
    pub density_gate: u8,    
    // Порог температуры: Пропускать только если temp > Y
    pub thermal_gate: u8,    
    pub reserved_gate: [u8; 14],

    // --- МЕТАДАННЫЕ (16 Байт) ---
    pub created_at: u64,     // event_id создания (COM)
    pub last_active: u64,    // event_id последнего обновления (COM)
}
```

#### Что изменилось по сравнению с твоим Connection V4.0:

1. **Physics-First:** Поля `strength`, `rigidity` заменены на реальную физику пружин (`stress`, `elasticity`). Это позволяет использовать готовые физические движки для симуляции мыслей.
    
2. **The Gate:** Добавлены `density_gate`. Связь теперь работает как **Мембрана**. Она сама решает: "Ты слишком легкий, не пущу тебя на глубокий уровень".

### 3. ПОЧЕМУ ЭТО БЫСТРЕЕ? (Axiom Performance)

Ты боялся потерять скорость. Вот почему V5 будет даже быстрее:

1. **Локальность данных:**
    
    В V4 поиск соседей проверял 8 слоев (`8 * 3 = 24` координаты).
    
    В V5 поиск проверяет только текущий Домен (`3` координаты).
    
    _Ускорение математики в 8 раз._
    
2. **Предсказание (Velocity):**
    
    Имея вектор скорости, мы можем не пересчитывать GridHash на каждый event. Мы знаем, где токен _будет_ в следующий момент.
    
3. **Отсутствие Python-фильтра:**
    
    Как мы обсуждали, тебе не нужно фильтровать мусор. Если ты в Домене Логики, ты видишь только логические связи.
