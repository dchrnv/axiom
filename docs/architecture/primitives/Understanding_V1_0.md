## 0. Что эта спека описывает

Understanding — **эталонная таксономия когнитивного стека AXIOM**. Не набор примитивов, не механизм — а **карта**: где каждый уровень понимания живёт в архитектуре, что уже реализовано, что планируется, что за горизонтом.

Эта спека — точка входа для ответа на вопрос: "Где в AXIOM находится [X]?" Если X — прагматика, смотри L2. Если X — интуитивная физика, смотри L5. Если X — RF-сенсинг, смотри L9 (не планируется).

**Уровни 0–7** — когнитивные; реализуются через примитивы и Perceptors.  
**Уровни 8–14** — перцептивные; требуют физической сенсорики за пределами текстового ввода.

## 1. Таксономия: 15 уровней понимания

### Уровень 0 — Синтаксис

**Что:** Формальная структура без смысла. Токенизация, порядок, синтаксические деревья.

**В AXIOM:** FrameWeaver + Writing primitives (P1–P7). TextPerceptor → примитивы → буквы → слова.

**Статус:** ✅ Реализовано (Writing_V1_0, FrameWeaver_V1_3). Composition bonds → V7-A1.

---

### Уровень 1 — Семантика

**Что:** Значение слов и их комбинаций. Концепты, референция, смысл.

**В AXIOM:** AnchorSet (семантические якоря), SubsystemEnergy, SutraDepth, гравитация Shell-якорей.

**Статус:** ✅ Реализовано. Расширяется в V7 через L0/L1 структуру anchors.

---

### Уровень 2 — Прагматика

**Что:** Интент, контекст, ирония. "Холодно" → просьба закрыть окно.

**В AXIOM:** MetaDetector V2 (V7) + `pragmatic_primitives.yaml`. MetaDetector сейчас даёт частичный сигнал.

**Статус:** 🔶 40% (MetaDetector есть, прагматические примитивы в плане). V7 / UGS Phase 1.

---

### Уровень 3 — Дискурс

**Что:** Связность текста/диалога. Анафора, логические переходы, нарратив.

**В AXIOM:** ActivityTrace.long (256 записей) + `discourse_primitives.yaml` (V7). TransitionMatrix отслеживает дискурсивные переходы.

**Статус:** 🔶 30% (ActivityTrace.long есть, discourse yaml в плане). V7 / UGS Phase 1.

---

### Уровень 4 — Мультимодальная семантика

**Что:** Связь языка с другими модальностями. "Мяукает" ↔ звук кота.

**В AXIOM:** Architecture готова (Perceptors). Требует composition bonds (V7-A1) + реального VisionPerceptor (V7-E2) + cross-modal bonds.

**Статус:** 🔶 20% (архитектурно готово, нет реальных Perceptors). V7-E / UGS Phase 2.

---

### Уровень 5 — Интуитивная физика

**Что:** Причинность, гравитация, постоянство объектов. "Стакан упадёт".

**В AXIOM:** CausalPerceptor (UGS Phase 3) + `causal_primitives.yaml`. TransitionMatrix как статистика причинно-следственных паттернов.

**Статус:** ❌ 0% (не начато). UGS Phase 3. Связан с Time V1.0 (time_irreversible).

---

### Уровень 6 — Проприоцепция и воплощённое действие

**Что:** Знание своего тела. Координаты конечностей, усилия, равновесие.

**В AXIOM:** BodyPerceptor (UGS Phase 4) + `body_primitives.yaml`. Требует непрерывный поток, не дискретные токены.

**Статус:** ❌ 0% (не начато). UGS Phase 4. Специфичен: нужен sliding window адаптер.

---

### Уровень 7 — Пространственное мышление

**Что:** 3D структура, навигация, невидимые поверхности. Ментальная карта.

**В AXIOM:** XYZ-пространство SUTRA (архитектурно готово) + `spatial_primitives.yaml` (V7-A2) + SpatialPerceptor (UGS Phase 2).

**Статус:** 🔶 50% (внутреннее XYZ пространство есть, spatial yaml в плане). V7-A2 / UGS Phase 2.

---

### Уровень 8 — Акустическая сцена

**Что:** Понимание пространства через звук. Направление источника, реверберация.

**В AXIOM:** AudioPerceptor stub. Нет реальной реализации.

**Статус:** ❌ Задел для будущих версий. После UGS Phase 4.

---

### Уровень 9 — RF-зрение (радиочастотное)

**Что:** Восприятие через Wi-Fi, Bluetooth сигналы. Сквозь стены.

**В AXIOM:** Не планируется в обозримых версиях. Требует специализированного hardware.

**Статус:** ❌ Вне плана. Зафиксировано как теоретически возможное.

---

### Уровень 10 — Гибридное волновое поле

**Что:** Единая модель пространства как поля колебаний разных частот.

**В AXIOM:** Теоретически: расширение NeRF-подобных подходов. Практически не планируется.

**Статус:** ❌ Вне плана (V8+ эра, после реальных физических сенсоров).

---

### Уровень 11 — Коллективная перцепция

**Что:** Несколько агентов строят единую модель мира через обмен сенсорными паттернами.

**В AXIOM:** Потенциально: axiom-broadcasting расширение. Не в активном плане.

**Статус:** ❌ Задел. Связан с multi-instance архитектурой.

---

### Уровень 12 — Временна́я интуиция

**Что:** Предсказание эволюции состояния среды.

**В AXIOM:** World Models подход. Требует полной реализации L5–L7 как основы.

**Статус:** ❌ Задел. Зависит от L5–L7.

---

### Уровень 13 — Причинное вмешательство

**Что:** "Что будет, если изменить вот это?" Контрфактика.

**В AXIOM:** DREAM Phase частично. Полная реализация требует CausalPerceptor (L5) + DilemmaStore (контрфактические дилеммы).

**Статус:** 🔶 Частично (DREAM даёт rehearsal, не полное контрфактическое рассуждение). V8+.

---

### Уровень 14 — Абсолютный (гипотетический)

**Что:** Информационная метрика реальности. Квантовые флуктуации. Чистая умозрительность.

**В AXIOM:** Не планируется. Зафиксировано как теоретический горизонт.

**Статус:** ❌ Философский задел.

## 3. Критический путь (L0 → L7)

```
L0 Синтаксис       ✅ — Writing primitives + FrameWeaver
  └── L1 Семантика  ✅ — AnchorSet + гравитация
        └── L2 Прагматика ← V7: pragmatic_primitives + MetaDetector V2
              └── L3 Дискурс ← V7: discourse_primitives + TransitionMatrix
                    └── L4 Мультимодальная ← V7-E: VisionPerceptor + composition bonds
```

```
L7 Пространство ← V7-A2: spatial_primitives.yaml
L5 Физика       ← UGS Phase 3: CausalPerceptor
L6 Тело         ← UGS Phase 4: BodyPerceptor
```

L2–L4 — линейная зависимость (каждый требует предыдущего).  
L5–L7 — параллельные ветки (можно развивать независимо после L1).

## 6. Практический гайд: "Что реализовывать дальше?"

**Сейчас (V7 активный приоритет):** L2 (прагматика) и L3 (дискурс) — самые близкие и высокоценные.

**После V7:** L5 (интуитивная физика) или L7 (пространство) — параллельно.

**Критический блокер для L4+:** V7-A1 (composition bonds). Без него мультимодальное связывание невозможно.

**Для L5–L7:** нужны реальные Perceptors (Vision, Spatial, Causal). Данные из мира, не из текста.

**L8–L14:** не являются приоритетом пока L0–L7 не стабилизированы. L9–L11 требуют hardware за пределами текущей архитектуры.

## 8. Домены AshtiCore

15 уровней понимания распределяются по доменам — это отображение показывает, *где* в архитектуре живёт каждый уровень познания.

| L | Уровень | Основной домен | Причина |
|---|---------|----------------|---------|
| L0 | Синтаксис | **D0 SUTRA + D1 EXECUTION** | примитивы в SUTRA, обработка в EXECUTION |
| L1 | Семантика | **D4 MAP + D0 SUTRA** | якоря в SUTRA, подтверждённые связи в MAP |
| L2 | Прагматика | **D6 LOGIC + D2 SHADOW** | намерение = логика контекста + угрозная гипотеза |
| L3 | Дискурс | **D4 MAP** | нарратив = карта событий во времени |
| L4 | Мультимодальная | **D10 MAYA** | синтез модальностей = финализация и проекция |
| L5 | Интуитивная физика | **D2 SHADOW + D5 PROBE** | физические угрозы (SHADOW) + исследование (PROBE) |
| L6 | Проприоцепция | **D1 EXECUTION** | воплощённое действие — прямой контакт с реальностью |
| L7 | Пространство | **D4 MAP** | карта физического мира |
| L8 | Акустика | **D5 PROBE** | исследование через звуковой зонд |
| L12 | Временна́я интуиция | **D7 DREAM** | предсказание из фоновой рекомбинации паттернов |
| L13 | Причинное вмешательство | **D5 PROBE + D2 SHADOW** | "что если" = probe (исследование) + shadow (угроза) |
| L14 | Абсолютный (гипот.) | **D0 SUTRA** | знание на уровне абсолютной истины системы |

**Метапаттерн:** D2 SHADOW отвечает за понимание угроз и рисков на всех уровнях (L2, L5, L13). D4 MAP — за факты (L1, L3, L7). D5 PROBE — за исследование неизвестного (L5, L8, L13). DREAM (D7) — за интуицию и консолидацию (L12). MAYA (D10) — за синтез и проекцию (L4).
