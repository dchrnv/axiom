## Введение к Документу 3

### Что описывает этот документ

Каждое окно Workstation — что в нём, как выглядит, как работает, как реагирует. Псевдо-эскизы в ASCII. Перечень компонентов iced. Поведение при типичных сценариях.

### Чего НЕ описывает

Этот документ **не показывает финальные эскизы**. ASCII-композиции — это **направление и пропорции**, не точная картинка. Конкретная типографика, точные радиусы скруглений, миллиметры отступов — задача **этапа дизайна** (визуального макета в Figma или подобном), который происходит после согласования этого документа и до начала реализации.

То есть процесс такой:
1. Этот документ — **что и как должно работать**
2. Этап дизайна — **точные пиксели и оттенки**
3. Документ 4 — **порядок реализации**

### Какие принципы применяются

Все принципы из Документа 1 (Vision & Principles) применяются здесь по умолчанию. В частности:

- Школа Cupertino, скруглённые углы, белое пространство, ограниченная палитра
- Семантика цвета: 5 состояний (спокойствие, активность, сон, завершение, внимание)
- Принципы: композиция важнее метрики, спокойствие важнее яркости, наблюдение важнее взаимодействия
- Анти-паттерны: никаких прогрессбаров везде, никакой геймификации, никаких pop-ups

При проектировании каждого окна я опираюсь на эти принципы, не повторяя их каждый раз.

### Структура описания каждого окна

Для каждого окна:

1. **Назначение** — для чего это окно
2. **Композиция** — ASCII-эскиз с подписями
3. **Поведение** — как окно реагирует на события Engine и пользователя
4. **Состояния** — что показывает в разных ситуациях (norm / loading / error / disconnected)
5. **Detach-режим** — как окно работает в отдельном системном окне
6. **Открытые вопросы** — то, что определится на этапе дизайна или реализации

## 1. Welcome / First Run

### 1.1 Назначение

Первое впечатление при первом запуске Workstation. Согласно Vision-документу — это **спокойное появление рабочего места**, не "встреча с существом". Не должно быть драматического приветствия от первого лица или мастера установки.

Цель — задать **тон достойности**: видно, что вещь сделана внимательно. После этого пользователь оказывается в System Map и начинает работать.

### 1.2 Когда показывается

- Первый запуск Workstation (нет config-файла)
- **Не показывается** при последующих запусках — там сразу System Map с восстановленной топологией

Это окно — **одноразовый ритуал**. Если пользователь хочет увидеть его снова (просто посмотреть) — есть пункт меню "About / Show welcome", который воспроизводит экран без сохранения новых настроек.

### 1.3 Композиция

```
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                          AXIOM                                 │
│                                                                │
│                       Workstation                              │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│                  Connecting to engine...                       │
│                                                                │
│                  ◐ ◐ ◐ ◐ ◐ ◐ ◐ ◐                             │
│                                                                │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

Композиция:

- **Очень много пустого пространства**. Окно занимает весь экран при первом запуске (в дальнейшем — обычный размер).
- В центре — **только название** ("AXIOM" сверху, "Workstation" снизу) с минималистичной типографикой. Без слогана, без дополнительных пояснений.
- Под названием — индикатор состояния первого подключения. Когда engine не найден — текст меняется (см. ниже).
- **Никаких кнопок** в основном пространстве. Никаких "Get started", "Welcome to...", "Choose your theme".

### 1.4 Поведение

```
1. Окно открывается с fade-in эффектом (длительность ~ 800ms)
2. Название появляется первым (сначала "AXIOM", через ~ 500ms — "Workstation")
3. После полного появления — пауза ~ 1 секунда тишины
4. Workstation начинает попытку подключения к Engine
5. Под названием появляется текст состояния:
   - "Connecting to engine..." (если попытка идёт)
   - "Engine not running" (если не найдено)
   - "Connected" (на доли секунды, потом переход в System Map)
```

### 1.5 Если engine не найден

```
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│                                                                │
│                          AXIOM                                 │
│                       Workstation                              │
│                                                                │
│                                                                │
│                                                                │
│                                                                │
│              Engine is not running                             │
│                                                                │
│         To use Workstation, the engine must be started.        │
│                                                                │
│              [ Spawn local engine ]                            │
│              [ Configure connection ]                          │
│              [ Wait for engine ]                               │
│                                                                │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

Три кнопки:
- **Spawn local engine** — запустить engine как child process (режим разработки)
- **Configure connection** — открыть настройки адреса/порта
- **Wait for engine** — фоновое ожидание с retry

Это не модальное окно с alarm-стилем. Это **просто следующее состояние** того же экрана. Тон спокойный.

### 1.6 Переход в основное состояние

После успешного подключения:

```
1. Текст "Connected" появляется на ~ 500ms
2. Welcome-окно начинает fade-out (длительность ~ 800ms)
3. Открывается main window с активной вкладкой System Map
4. System Map отрисовывается со своим стартовым появлением (см. раздел System Map)
```

Этот переход — **единственный момент** в жизни Workstation, когда есть **специальная анимация перехода между состояниями**. В обычной работе таких ритуалов нет.

### 1.7 Что НЕ должно быть в Welcome

Чтобы избежать дрейфа в сторону "приветствия от персонажа":

- **Нет приветствия от первого лица.** Никаких "Hello, I'm AXIOM" или "Welcome to my mind".
- **Нет вопросов пользователю.** Никаких "What's your name?", "Choose your style".
- **Нет анимации мандалы.** Мандала живёт в System Map, не в Welcome.
- **Нет музыки/звуков.** В V1.0 звука нет вообще.
- **Нет введения "что такое AXIOM".** Если пользователь дошёл до запуска — он знает.
- **Нет настройки темы/языка/чего-либо.** Дефолты работают, настройки — потом.

### 1.8 Detach режим

Welcome **не имеет** detach-режима. Это не вкладка, это специальное стартовое окно, которое существует только при первом запуске.

### 1.9 Открытые вопросы

- Точная анимация появления названия (fade vs slide vs scale)
- Стоит ли скрывать индикатор подключения "◐ ◐ ◐ ◐..." или оставить как минимальный визуальный элемент
- Что делать, если первое подключение длится дольше 30 секунд — менять формулировку или просто продолжать

## 2. System Map — главное окно

### 2.1 Назначение

System Map — **центральное окно Workstation**. Карта архитектуры AXIOM в живом времени.

Здесь видно:
- Текущее состояние системы (WAKE / FALLING_ASLEEP / DREAMING / WAKING)
- Все 11 доменов с их активностью
- Потоки данных между доменами
- Над-доменный слой: GUARDIAN, FrameWeaver, DREAM Phase
- Текущий fatigue, последний DreamReport
- Активные импорты через External Adapters

System Map — это **живая карта мира AXIOM**. По ней понятно, что система работает и что в ней происходит **прямо сейчас**.

### 2.2 Главный визуальный элемент — мандала

В центре System Map — **мандала**, символический центр системы (см. Документ 1, раздел 6.2). Она отражает:

- Текущее состояние (через доминирующий цвет)
- Активность по доменам (через подсветку секторов)
- Темп работы (через медленную анимацию)

Мандала — **не интерактивный элемент**. На неё не кликают для действий. Это присутствие системы в видимой форме.

Структура:

- В центре — мандала (см. 2.4)
- Вокруг мандалы — расположение доменов как **окружение центра**:
  - Сверху и сбоку — кольцо ASHTI (101-108), восемь доменов
  - Над ASHTI — MAYA (110), точка сборки

- Линии потоков между доменами — **тонкие, светлые в покое, яркие при активности**

**Нижняя панель — сводные показатели:**
- Слева: текущее состояние, fatigue (sparkline), hot path метрика
- Справа: время с последнего сна, количество Frame, промоций сегодня

Эта нижняя панель — **единственное место**, где есть числа. Карта сама по себе показывает состояние через композицию и цвет, числа — для деталей.

### 2.4 Мандала — детально

Мандала — концентрические круги:

```
            ┌─────────────────┐
            │                 │
            │  ┌───────────┐  │
            │  │           │  │
            │  │  ┌─────┐  │  │
            │  │  │     │  │  │
            │  │  │  ◉  │  │  │     ← Внутреннее ядро: SUTRA
            │  │  │     │  │  │       (тёмный графитовый круг)
            │  │  └─────┘  │  │
            │  │           │  │     ← Среднее кольцо: ASHTI 1-8
            │  └───────────┘  │       (восемь секторов, разделённых
            │                 │        тонкими линиями. Активный
            │                 │        сектор подсвечивается цветом
            │                 │        состояния)
            │                 │     ← Внешнее кольцо: MAYA + EXPERIENCE
            │                 │       (тонкое кольцо, общий тон)
            └─────────────────┘
```

**Состояния мандалы:**

- **WAKE, спокойствие.** Все элементы в нейтральном тоне с лёгким холодным синим оттенком. Медленная пульсация (период ~ 4 секунды).
- **WAKE, активность.** Активные секторы ASHTI подсвечиваются тёплым янтарным. Внешнее кольцо более яркое.
- **FALLING_ASLEEP.** Медленное "оседание" цветов в сторону сиренево-индигового. Анимация ~ 5 секунд.
- **DREAMING.** Доминирующий тон — сиренево-индиговый. Внутреннее ядро (SUTRA) едва заметно "пульсирует" — это намёк на работу DREAM-фазы. Внешнее кольцо тише.
- **WAKING.** Постепенный возврат к WAKE-тону, обратная анимация ~ 2-3 секунды.
- **Внимание.** Если есть alert — на мандале появляется тонкое тёплое красное кольцо снаружи. Не мигает, просто присутствует.

**Анимация:**
- Период пульсации — длинный (3-5 секунд), не быстрый. Это создаёт ощущение **спокойного дыхания** системы.
- Переходы между состояниями — плавные, ease-in-out.
- В состоянии полного покоя анимация может почти останавливаться — это **молчание как полноправное состояние** (Принцип из Документа 1).

### 2.5 Домены вокруг мандалы

Вокруг мандалы — **схематические индикаторы каждого домена**.

```
              [ MAYA 110 ]
            ↑         ↑
    [ETHIC]→[LOGIC] ←...→ [SIMUL]→[PROBE]
       ↑                              ↓
       └─→ M A N D A L A ←─────────────┘
       ↓                              ↑
    [INTNT]←[GENESIS]←...→[DREAM]← [META]
            ↓         ↓
       [ EXPERIENCE 109 ]   [ SUTRA 100 ]
```

Каждый домен — **скруглённый блок** с именем (короткое сокращение из 4-5 букв) и индикатором активности. Размер блока — небольшой, чтобы все 11 помещались на экране.

**Состояния блока домена:**

- **Idle** — нейтральный фон, имя серым цветом
- **Active** — фон в цвете состояния (если LOGIC обрабатывает — тёплый янтарный), имя графитовым
- **High activity** — более насыщенный цвет
- **Disabled / not engaged** (например, во время DREAMING) — приглушённый, серый, с пометкой

**Линии между доменами:**

- Тонкие линии показывают **топологию**: какие домены могут передавать токены друг другу
- При активном потоке — линия **вспыхивает** в направлении потока (короткая анимация ~ 300-500ms)
- В покое линии тонкие, почти невидимые

### 2.6 Над-доменный слой — где он на карте

GUARDIAN, FrameWeaver, DREAM Phase — **не отдельные домены**. Они работают над всей системой. Поэтому их представление на System Map — **не блок на карте**, а **индикаторы в углах или периферии**.

Возможный вариант:

```
[GUARDIAN]                            [FrameWeaver]
   ●                                       ●
                  [ MAYA 110 ]
                ↑         ↑
        [ASHTI кольцо вокруг мандалы]
                ↓         ↓
       [ EXPERIENCE 109 ]   [ SUTRA 100 ]

[DREAM Phase]                                   [Other Weavers]
    ●                                          (deferred, пусто
                                                в V1.0)
```

Каждый над-доменный компонент — **точка с именем**, может подсвечиваться цветом активности.

**Клик на над-доменный компонент** — открывает соответствующее окно в подробностях:
- Клик на DREAM Phase → переход к табу Dream State
- Клик на FrameWeaver → переход к табу Patterns
- Клик на GUARDIAN → раскрывает popup со статистикой veto

### 2.7 Нижняя панель — сводные показатели

```
┌─────────────────────────────────────────────────────────────────┐
│ State: WAKE                  │ Last dream: 5 min ago             │
│ Fatigue: ▁▁▂▂▃ 47%           │ Frames in EXPERIENCE: 1247        │
│ Hot path: 238 ns             │ Promotions today: 3               │
└─────────────────────────────────────────────────────────────────┘
```

**Левая колонка:**
- **State** — текущее состояние, текстом
- **Fatigue** — sparkline последних 60 секунд + текущее значение в процентах
- **Hot path** — последний замер времени тика

**Правая колонка:**
- **Last dream** — сколько прошло с момента последнего сна
- **Frames in EXPERIENCE** — общее количество Frame
- **Promotions today** — сколько Frame продвинуто в SUTRA сегодня (с момента последнего рестарта engine)

Эти показатели — **обновляются с задержкой 500ms-1s**. Не мгновенно, не для каждого тика. Достаточно для понимания.

### 2.8 Поведение при изменениях

**Появление нового Frame в EXPERIENCE:**
- Нижняя панель: счётчик `Frames in EXPERIENCE` увеличивается
- На карте: линия от MAYA к EXPERIENCE кратковременно подсвечивается
- Без нотификации, без отдельного индикатора — это просто часть фоновой жизни

**Промоция Frame в SUTRA:**
- Это более редкое и значимое событие
- Линия от EXPERIENCE к SUTRA подсвечивается зелёным шалфеем (цвет завершения)
- Внутреннее ядро мандалы кратковременно "вспыхивает" мягким зелёным
- Счётчик `Promotions today` увеличивается
- Длительность визуального эффекта — около 1.5 секунды

**Переход в DREAMING:**
- Большое визуальное событие
- Все домены ASHTI постепенно приглушаются (5 секунд)
- Мандала переходит к сиренево-индиговому
- Появляется текст состояния "DREAMING" под мандалой
- Активные импорты приостанавливаются (если такие были — они паузятся, см. раздел Files)

**GUARDIAN veto:**
- Тонкое тёплое красное кольцо появляется вокруг мандалы
- Длится несколько секунд
- В углу с GUARDIAN — короткая вспышка
- В Conversation (если открыт) появляется запись о veto
- Без модальных окон, без звуков

### 2.9 Detach-режим

System Map — **главный кандидат на detach** на второй монитор. Описанная выше композиция работает в любых пропорциях окна:

- На широком мониторе — мандала по центру, домены вокруг с воздухом
- На квадратном — то же самое, более плотно
- На узком вертикальном — мандала в верхней половине, домены в нижней, нижняя панель сжимается в одну колонку

В detached-режиме окно работает как полноценный **живой пейзаж системы** — chrnv может оставить его на втором мониторе и работать с другим окном на главном.

При этом detached System Map обновляется с той же частотой, что и в main window. Никаких компромиссов.

### 2.10 Состояния окна

**Norm (Connected, snapshot есть):**
- Полная композиция как описано выше

**Loading (первое подключение):**
- Карта рисуется постепенно по мере получения snapshot
- Пока snapshot не пришёл — placeholder с текстом "Loading system state..."

**Reconnecting (соединение потеряно, был snapshot):**
- Frozen snapshot отображается
- Поверх — overlay с текстом "Connection lost — showing last known state"
- Overlay полупрозрачный, не блокирует чтение карты
- В углу — индикатор reconnecting с retry counter

**Disconnected (нет соединения):**
- Карта серая, тонированная
- В центре — текст "Engine not running"
- Кнопки действий внизу (как в Welcome, см. 1.5)

### 2.11 Что НЕ показывается на System Map

- **Числа потоков** (например, "120 токенов/сек"). Только композиция/цвет.
- **Метки на линиях между доменами**. Чисто визуальная карта.
- **Раскрывающиеся попапы при наведении на домен**. Клик — да (открывает Patterns или другое окно), наведение — нет.
- **Контекстное меню на правый клик**. В V1.0 — без правого клика.

### 2.12 Открытые вопросы

- **Точная форма мандалы.** Концентрические круги — самый простой вариант, но возможны и другие (8-лепестковая, спираль). Решается на этапе визуального дизайна.
- **Анимация линий потоков.** Просто вспышки vs движущиеся точки vs пульсация. Эстетический выбор.
- **Переключение карты на состояние "только домены без мандалы"** — может быть полезно для разработчика, который хочет видеть только активность. Стоит ли добавлять? Я склоняюсь к **нет** в V1.0. Мандала — обязательная часть.
- **Подсказки названий доменов** — если сокращения "LOGIC", "SIMUL" непонятны. Без tooltip это сложно. Возможен компромисс — короткое описание появляется в нижней панели при клике на домен (вместо общих метрик показывается описание выбранного).

## Резюме Части A

В этой части определены:

1. **Welcome** — стартовое окно, спокойное появление рабочего места без драматизма. Только первое включение, потом не показывается.
2. **System Map** — главное окно с мандалой в центре, доменами вокруг, потоками между ними и нижней сводной панелью. Кандидат на detach на второй монитор.

Это самые важные два окна — Welcome задаёт первое впечатление, System Map определяет основную работу. Остальные окна (Live Field, Patterns, Dream State, Conversation, Files, Configuration, Benchmarks) описываются в Частях B и C, опираясь на установленные здесь принципы.

После согласования Части A — переходим к Части B (окна наблюдения).
