
# Axiom Core Specification

Version: 0.1 (Draft)

## 1 Назначение

**Axiom Core** определяет инварианты системы.
Любая реализация, модуль или расширение обязаны соблюдать эти инварианты.
Нарушение инвариантов означает несовместимость с Axiom Core.

## 2 Термины

**Token** — базовая сущность системы.
Layer — семантический слой пространства.
Coordinate — положение токена в пространстве слоя.
State — внутреннее состояние токена.
Movement — изменение координат токена.

(Ты потом уточнишь определения. Сейчас важно зафиксировать словарь.)

## 3 Структура пространства

3.1. Система содержит два равноправных и изоморфных пространства.
3.2. Пространства используют общий набор осей.
3.3. Каждое пространство содержит N семантических слоёв (в текущей версии N = 8).
3.4. Слои используют одни и те же оси координат.

## 4 Токен

4.1. Токен является единой сущностью с проявлением в обоих пространствах.
4.2. Токен обладает состоянием.
4.3. Координаты токена являются функцией его состояния.
4.4. Токен может взаимодействовать с другими токенами в пределах одного слоя.

## 5 Движение

5.1. Движение определяется как изменение координат токена.
5.2. Изменение координат вызывается изменением состояния или взаимодействием.
5.3. Движение по умолчанию происходит в пределах одного слоя.
5.4. Межслойные переходы допустимы только при выполнении формально определённых условий.

## 6 Инварианты

6.1. Изоморфность пространств не может быть нарушена.
6.2. Количество слоёв в рамках одной мажорной версии фиксировано.
6.3. Определение токена как единой сущности неизменно.
6.4. Координаты не существуют независимо от состояния.

## 7 Расширения

7.1. Расширения не имеют права изменять инварианты.
7.2. Расширения могут вводить новые типы токенов.
7.3. Расширения могут вводить дополнительные параметры состояния.
7.4. Расширения могут реализовывать внешние модули оценки или обработки.

## 8 Версионирование

Изменение инвариантов требует увеличения мажорной версии.
Изменение поведения без нарушения инвариантов допускает минорное обновление.
