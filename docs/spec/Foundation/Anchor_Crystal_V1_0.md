# AXIOM — Anchor Crystal V1.0 (этап 1: аккумулятор)

**Статус:** Спецификация
**Версия:** 1.1
**Дата:** 2026-06-13
**Изменения V1.0 → V1.1:** геометрия перевёрнута в слои абстракции (поверхность C0 → дно C7, 8 слоёв жёстко); аргументация региона приведена к REPAIR-01 (N1–N3); стрела значимости и радиус-сложность свёрнуты в радиус-частоту внутри слоя
**Категория:** Стартовая библиотека якорей · формульная геометрия · этап 1 пары «аккумулятор → генератор»
**Crate:** `axiom-seed` (Crystal Layout как бэкенд Seed Compiler)
**Опирается на:** `Seed_Injection_V1_0` (этап 2 — генератор), `Anchor_Tokens_V1_0`, `BLUEPRINT` (SpatialHashGrid, resonance_search, инварианты), `Axiom_Semantic_Core` (оси/октанты)
**Источник идеи:** внешний диалог chrnv («Кристалл Первоосновы» / «Вселенский Кристалл Письменности») — адаптирован к реальной геометрии AXIOM (§1).

> **Стиль:** комментарии вместо кода.

## 0. Два этапа — аккумулятор и генератор

```
// Чтобы генератор заработал, машине нужен аккумулятор (chrnv).
//
// ЭТАП 1 (этот документ) — АККУМУЛЯТОР:
//   стартовая палитра графемных якорей, уложенная в SUTRA по ФОРМУЛЕ.
//   Тысячи символов — позиции вычисляются, не подбираются руками.
//
// ЭТАП 2 (Seed_Injection_V1_0) — ГЕНЕРАТОР:
//   grow-семена прорастают через MAYA→кристаллизация→DREAM.
//   Прорастание требует co-activation С ЧЕМ-ТО — этап 1 даёт это «что-то».
//
// Без аккумулятора grow-семена падают в пустоту (нечему резонировать).
// Без генератора аккумулятор мёртв (якоря лежат, понимание не растёт).
```

## 1. Главное решение — локальный кристалл, не глобальная перезапись осей

```
// После REPAIR-01 действует: координаты [0..32767] положительные,
//   позиция несёт ТОЛЬКО близость (N1), природа — только из подписи
//   и origin_domain (N2), SUTRA вне октантной философии (N3).
//
// Поэтому кристалл свободно живёт в положительном кубе — но как
//   ЛОКАЛЬНАЯ структура внутри региона, по двум причинам:
//   1. Непересечение с существующими якорями (STATE_LOCKED, позиции
//      кэшированы в SubsystemGravity навечно).
//   2. Локальность: соседство относительное — стреле/вееру/слоям всё
//      равно где регион, а грид и резонанс выигрывают от компактности.
//
// РЕШЕНИЕ: кристалл — ЛОКАЛЬНАЯ система координат внутри региона.
//
//   CrystalRegion { origin: [i16;3], size: [u16;3] }   — параметры, конфиг
//   Внутри региона — локальные оси (u, v, w) ∈ [0..size].
//   Глобальная позиция якоря = origin + local(u,v,w).
//
//   Вся красота сохраняется (стрела, веер, дуги, локальность —
//   соседство ОТНОСИТЕЛЬНО, ему всё равно где регион).
//   Глобальная семантика осей НЕ трогается.
//
// Размещение региона V1: рядом с существующей writing-зоной (графемы —
//   это письменность). Конкретный origin/size — конфиг, подбирается так,
//   чтобы регион не накрывал осевые/субсистемные якоря. Ориентир размера:
//   куб ~8000–10000 на сторону (достаточно для тысяч графем с запасом).
//
// СУЩЕСТВУЮЩИЕ ЯКОРЯ НЕ ДВИГАЮТСЯ. Кристалл — новая палитра ПОВЕРХ:
//   STATE_LOCKED навечно + позиции закэшированы в SubsystemGravity.
//   Кристалл добавляет графемный слой, не заменяет словарный.
```

## 2. Геометрия кристалла — слои абстракции (перевёрнутая, V1.1)

### 2.1 Ось d — Глубина Абстракции (8 слоёв)

```
// ПЕРЕВЁРНУТАЯ конструкция (chrnv): на ПОВЕРХНОСТИ — примитивы,
//   с каждым шагом ВГЛУБЬ — сложнее. Погружение = усложнение.
//
//   d ≈ 0 (поверхность): слой C0 — графемы. Горячая рабочая зона,
//     сканируется постоянно, кэш-локальна.
//   d → дно: слой C7. Холодная глубина, трогается по необходимости.
//
// Слой = плита толщиной size_d / 8. Жёстко 8 слоёв (граница GUARDIAN):
//   глубина фракталов ограничена, девятого этажа нет.
//
// ФИЗИКА СЛОИСТОСТИ: C(n+1) геометрически ПРИМЫКАЕТ к C(n) — слово
//   лежит сразу под слогами, из которых собрано → composition bonds
//   короткие, co-activation локальна, резонанс между уровнем и его
//   материалом дешёвый. Это топливо этапа 2 (прорастание).
//
// ScanningPlan получает буквальный смысл: обычный скан — поверхность
//   (C0–C2), погружение глубже — по требованию.
//
// НЕ ПУТАТЬ: слой (ГДЕ по абстракции, геометрия) ≠ SutraDepth
//   (НАСКОЛЬКО вросло, параметр укоренённости). Разные оси, обе нужны.
```

### 2.2 Таблица слоёв — расчёт «сколько миру надо»

```
// СЛОЙ | письменность      | математика       | музыка        | время
// C0   | графемы           | цифры/знаки      | ноты          | моменты
// C1   | слоги/биграммы    | пары («2+3»)     | интервалы     | последовательности
// C2   | слова             | числа/выражения  | аккорды/мотивы| циклы
// C3   | фразы             | формулы          | муз. фразы    | периоды
// C4   | предложения       | теоремы          | темы          | эпохи
// C5   | тексты/нарративы  | теории           | произведения  | —
//
// Миру достаточно ШЕСТИ (C0–C5) — вселенский кристалл V2 укладывается
//   (иероглиф = C0–C1 по знаку, его значение = C2).
// ДЛЯ МОДЕЛИ +2:
// C6   | МЕТА — композитные подсистемы (Calculus, Rhythm), формулы
//        над формулами, мета-режимы
// C7   | РЕЗЕРВ — пустой. Право системы погрузиться, когда реально надо.
```

### 2.3 Плоскость слоя — полярный веер (θ = природа, r = частота)

```
// В плоскости каждого слоя — полярная раскладка:
//
//   Угол θ — ПРИРОДА знака, секторами с ПУСТЫМИ ЗАЗОРАМИ между:
//     V1: крыло гласных | крыло согласных | узкий сектор цифр |
//         швы пунктуации/операторов (на стыках секторов:
//         операторы у цифр, языковая пунктуация между крыльями)
//     V2: + идеограммы / слоговые / алфавиты (веер культур)
//   Зазоры = защита от хаоса: цифры и буквы не путаются при
//     локальном поиске — между ними пустое окно. Даром.
//
//   Радиус r — ЧАСТОТА: частое в ЯДРЕ слоя (горячий центр),
//     редкое на ободе. Кэш-локальность: рабочее ядро компактно.
//
// УПРОЩЕНИЕ vs V1.0: стрела значимости и радиус-сложность свёрнуты
//   в один радиус-частоту. Сложность знака геометрией НЕ кодируется
//   (поле в charset остаётся справочным, при нужде — shell).
```

### 2.4 Самостоятельное погружение

```
// DREAM-промоция кладёт Frame на слой = его FrameComposition.
//   Система собрала композицию выше C6 → ложится в C7 (резерв).
//   Глубже C7 — НЕТ (GUARDIAN, жёсткая граница 8).
// «Разрешить самостоятельно погружаться» = резерв есть, прорыть
//   девятый этаж нельзя.
// Правка enum: FrameComposition C5_Plus раскрыть → C5, C6_Meta,
//   C7_Reserve (мелкая, согласовать с существующими использованиями).
```

### 2.5 Связь с L0-примитивами

```
// Штрихи/точки/пересечения (L0 perceptual, 22 якоря) кристалл
//   НЕ дублирует — они НАД поверхностью по смыслу (суб-графемный
//   уровень восприятия), живут где живут. Связь графема↔штрихи —
//   через composition bonds, не через геометрию.
```

## 4. Генерация — Crystal Layout в Seed Compiler

```
// Кристалл НЕ пишется руками. Это бэкенд компилятора из Seed_Injection §3:
//
//   axiom-seed compile --layout crystal
//        --charset charsets/ru_en_base.yaml      // графемы + ранги + классы
//        --region  "origin=[...], size=[...]"     // конфиг региона
//     → seeds/crystal_c0.yaml                     // anchor-mode seed set
//
// charset-файл: для каждой графемы — ранг частоты, класс (гласная/
//   согласная/цифра/пунктуация/оператор), сложность (число штрихов).
// Компилятор применяет формулы §2 → позиции → anchor-mode семена.
//
// Инъекция: štandartный anchor-режим Seed Injection (boot-time или
//   DREAMING через CODEX). Инвариант 11 соблюдён тем же способом.
//
// Mass графем: единый уровень C0-якоря (160–180, как принято для
//   примитивов) — частота уже закодирована ПОЗИЦИЕЙ (стрела), не дублируем
//   её массой. Shell — по классу знака (шаблон задаёт профили).
//
// Детерминизм: одинаковый charset + region → одинаковые позиции всегда.
//   Кристалл воспроизводим, diff-уем, версионируем как код.
```

## 5. Согласование с существующим

```
// match_text / anchor-matching (TextPerceptor 2-path):
//   графемные якоря кристалла ДОПОЛНЯЮТ покрытие на уровне символов —
//   там, где словарные якоря (word_signals) не матчатся, символьный
//   слой даёт позицию. Приоритет: словарный матч > графемный
//   (слово семантичнее буквы). Уточнить в реализации порядок путей.
//
// L0 perceptual: остаются исключёнными из match_text (инвариант 10);
//   кристалл — это НЕ L0, это C0-графемы (символьный уровень письменности).
//
// SubsystemGravity: новые якоря НЕ добавляют правил гравитации
//   (val_*/A5/A6 — отдельная история). Кристалл пассивен как все якоря.
//
// Мембраны доменов: семена-графемы при grow-использовании окрашиваются
//   как всё живое; сами якоря кристалла STATE_LOCKED в SUTRA — вне мембран.
//
// EXECUTION внимание: регион подобрать так, чтобы НЕ пересекался с
//   осевыми якорями (mass=255) и субсистемными зонами — проверка
//   коллизий в компиляторе (мин. дистанция до существующих якорей,
//   порог конфигурируем, по образцу emergent-правила distance<1000).
```

## 6. Что это даёт физике (резюме выгод)

```
// 1. Локальный резонанс: частые графемы в одном секторе → дешёвое
//    co-activation → слоги/биграммы прорастают сами (топливо этапа 2).
// 2. Кэш-дружелюбность: горячая зона компактна в SpatialHashGrid.
// 3. Защита от хаоса: цифры/буквы/пунктуация разведены геометрически.
// 4. Масштабируемость: тысячи графем по формуле, нуль ручного подбора.
// 5. Детерминизм и воспроизводимость: кристалл = функция(charset, region).
// 6. Задел на «периодическую таблицу письменности» (V2) без переукладки.
```

## 7. Инварианты

| Правило | Значение |
|---------|----------|
| Глобальные оси | НЕ перезаписываются — кристалл в локальных координатах региона |

## 9. Порядок реализации

```
1. charset ru_en_base.yaml (графемы + ранги частоты + классы природы)
2. crystal.rs: слои абстракции (d → 8 плит) + полярный веер в плоскости
   слоя (θ=природа секторами с зазорами, r=частота, ядро горячее)
3. FrameComposition enum: C5_Plus → C5 / C6_Meta / C7_Reserve
   (согласовать с существующими использованиями)
4. Регион: выбрать origin/size с chrnv (карта якорей → свободная зона) +
   collision-проверка
5. compile --layout crystal → seeds/crystal_c0.yaml (anchor-mode, слой C0)
6. Boot-инъекция, OBS-прогон: графемный слой матчится, частые символы
   в ядре слоя C0 (соседство в гриде), слой ≠ SutraDepth не спутаны
7. Этап 2 поверх: grow-семена C1 (слоги) ложатся на слой, примыкающий
   к C0 → короткие composition bonds → ПЕРВАЯ ПРОВЕРКА аккумулятор→генератор

Тесты: детерминизм раскладки; 8 слоёв, глубже C7 нельзя (GUARDIAN);
  частые графемы в ядре слоя; секторы природы с зазорами (нет коллизий
  цифра/буква); C0 на поверхности (d≈0); слой независим от SutraDepth.
```

## История

- **V1.1** (2026-06-13): геометрия перевёрнута в СЛОИ АБСТРАКЦИИ (chrnv) — поверхность d≈0 = C0 графемы (горячая рабочая зона), вглубь = сложнее, дно = C7. 8 слоёв жёстко (граница GUARDIAN, глубже нет). C(n+1) примыкает к C(n) → короткие composition bonds, дешёвый резонанс уровня с материалом. Расчёт: миру хватает C0–C5, модели +C6 (мета: Calculus/Rhythm) +C7 (резерв для самостоятельного погружения). Слой (ГДЕ по абстракции) ≠ SutraDepth (насколько вросло). В плоскости слоя — полярный веер: θ=природа (секторы с зазорами против хаоса), r=частота (ядро горячее). Стрела значимости и радиус-сложность V1.0 свёрнуты в радиус-частоту; сложность знака геометрией не кодируется. Аргументация региона приведена к REPAIR-01 (N1–N3, signed-модели больше нет). FrameComposition: C5_Plus → C5/C6_Meta/C7_Reserve.

- **V1.0** (2026-06-13): первая спека этапа 1 (см. ниже). Идея внешнего диалога chrnv (формульная топология якорей: стрела значимости, веер природы знака, луч цифр, швы пунктуации) адаптирована к реальному AXIOM: кристалл уложен в ЛОКАЛЬНЫЙ регион (глобальные оси заняты семантикой Аполлон/Эрос/Воля и октантами — куб [0..32k]³ был бы одним октантом и убил бы октантную энтропию). Существующие якоря не двигаются. Генерация через Crystal Layout бэкенд Seed Compiler (детерминизм, воспроизводимость). Инъекция anchor-режимом Seed Injection V1.0. Пара закрыта: этап 1 заряжает (палитра), этап 2 генерирует (прорастание).
