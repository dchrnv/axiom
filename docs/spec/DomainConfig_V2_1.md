# AXIOM MODULE SPECIFICATION: DomainConfig V2.1

**Статус:** Актуальная спецификация (core)  
**Версия:** 2.1 (Arbiter Integration)  
**Дата:** 2026-03-19  
**Формат:** Точно 128 байт, `repr(C, align(128))`  
**Связанные спеки:** Domain V1.3, COM V1.0, Token V5.2, Ashti_Core V2.0, Arbiter V1.0

---

## 1. Назначение

**DomainConfig** — 128-байтная "ДНК" Домена. Определяет физику, мембрану, семантику и поведение домена при взаимодействии с Arbiter.

Версия 2.1 перераспределяет 8 байт резерва мембраны (`reserved_membrane`) под блок настроек Arbiter, сохраняя общий размер 128 байт.

---

## 2. Структура (128 байт)

```rust
#[repr(C, align(128))]
pub struct DomainConfig {
    // --- 1. ИДЕНТИФИКАЦИЯ [16 Байт] ---
    pub reserved_id: u64,       // 8b | Резерв для будущих расширений
    pub domain_id: u16,         // 2b | Уникальный ID Домена
    pub parent_domain_id: u16,  // 2b | Родительский Домен
    pub domain_type: u8,        // 1b | Тип (до 255 вариаций)
    pub structural_role: u8,    // 1b | Роль в Ashti_Core (Sutra, Experience, Logic, Dream...)
    pub generation: u8,         // 1b | Поколение (эволюционный индекс)
    pub flags: u8,              // 1b | Битовая маска состояний (Active, Locked)
    // Offset: 16 байт

    // --- 2. ФИЗИКА ПОЛЯ [32 Байт] ---
    pub field_size: [f32; 3],   // 12b| Размеры поля (X, Y, Z)
    pub gravity_strength: f32,  // 4b | Гравитация (-MAX..+MAX)
    pub temperature: f32,       // 4b | Температура поля в Кельвинах
    pub time_dilation: u16,     // 2b | Замедление времени (х100)
    pub resonance_freq: u16,    // 2b | Базовая частота (Hz)
    pub pressure: u16,          // 2b | Давление (Pa)
    pub reserved_physics: u16,  // 2b | Резерв блока физики
    pub friction_coeff: u8,     // 1b | Трение (0..255 -> 0.0..1.0)
    pub viscosity: u8,          // 1b | Вязкость (0..255 -> 0.0..1.0)
    pub elasticity: u8,         // 1b | Упругость (0..255 -> 0.0..1.0)
    pub quantum_noise: u8,      // 1b | Квантовый шум (0..255 -> 0.0..1.0)
    // Offset: 48 байт

    // --- 3. СЕМАНТИЧЕСКИЕ ОСИ [16 Байт] ---
    pub axis_x_ref: u32,        // 4b | Референс концепции оси X
    pub axis_y_ref: u32,        // 4b | Референс концепции оси Y  
    pub axis_z_ref: u32,        // 4b | Референс концепции оси Z
    pub axis_config: u32,       // 4b | Конфигурация полюсов (Bit-packed u16x2)
    // Offset: 64 байт

    // --- 4. МЕМБРАНА И ARBITER [32 Байт] ---
    pub input_filter: u64,      // 8b | 64-bit Bloom Filter или хэш входа
    pub output_filter: u64,     // 8b | 64-bit Bloom Filter или хэш выхода

    // -- Блок Arbiter [8 Байт] (бывший reserved_membrane) --
    pub reflex_threshold: u8,   // 1b | Порог рефлекса (0..255 -> 0.0..1.0)
    pub association_threshold: u8, // 1b | Порог ассоциации (0..255 -> 0.0..1.0)
    pub arbiter_flags: u8,      // 1b | Битовая маска (см. раздел 3)
    pub reflex_cooldown: u8,    // 1b | Минимальный интервал между рефлексами (в пульсах)
    pub max_concurrent_hints: u8, // 1b | Макс. кол-во ассоциаций-подсказок одновременно
    pub feedback_weight_delta: u8, // 1b | Шаг изменения weight при обратной связи (0..255)
    pub reserved_arbiter: [u8; 2], // 2b | Резерв блока Arbiter

    pub gate_complexity: u16,   // 2b | Вычислительная сложность шлюзов
    pub threshold_mass: u16,    // 2b | Порог массы для прохождения
    pub threshold_temp: u16,    // 2b | Порог температуры для прохождения
    pub permeability: u8,       // 1b | Проницаемость (0..255 -> 0.0..1.0)
    pub membrane_state: u8,     // 1b | OPEN/CLOSED/SEMI/ADAPTIVE
    // Offset: 96 байт

    // --- 5. МЕТАДАННЫЕ [32 Байт] ---
    pub created_at: u64,        // 8b | COM event_id (Время создания)
    pub last_update: u64,       // 8b | COM event_id (Последнее обновление)
    pub token_capacity: u32,    // 4b | Максимальная емкость токенов
    pub connection_capacity: u32, // 4b | Максимальная емкость связей
    pub error_count: u16,       // 2b | Счетчик когнитивных ошибок
    pub processing_state: u8,   // 1b | IDLE/PROCESSING/FROZEN/CRASHED
    pub complexity_score: u8,   // 1b | Оценка сложности (0..255 -> 0.0..1.0)
    pub performance_score: u8,  // 1b | Производительность (0..255 -> 0.0..1.0)
    pub reserved_meta: [u8; 3], // 3b | Добивка до границы 128 байт
    // Итого: 128 байт. Offset: 128. Без скрытого паддинга.
}
```

## 3. Блок Arbiter — Семантика полей

### 3.1 reflex_threshold (u8)

Минимальный weight следа в EXPERIENCE(9), при котором Arbiter классифицирует ответ как рефлекс и отправляет предсказание в MAYA(10).

Шкала: 0..255 отображается на 0.0..1.0. Значение 0 означает "рефлексы отключены для этого домена" — ответ домена 9 никогда не будет отправлен как предсказание при обработке паттерна для этого домена.

### 3.2 association_threshold (u8)

Минимальный weight следа в EXPERIENCE(9), при котором ответ считается ассоциацией и прикрепляется как подсказка к паттерну при передаче в данный домен ASHTI(1-8).

Если weight ниже этого порога — тишина. Домен получает паттерн без подсказки.

### 3.3 arbiter_flags (u8)

Битовая маска поведения Arbiter для данного домена:

| Бит | Имя | Описание |
|-----|-----|----------|
| 0   | REFLEX_ENABLED | Рефлексы разрешены для этого домена |
| 1   | HINTS_ENABLED | Ассоциации-подсказки разрешены |
| 2   | FEEDBACK_ENABLED | Обратная связь (усиление/ослабление следов) активна |
| 3   | GUARDIAN_CHECK_REQUIRED | Рефлексы обязательно проходят проверку GUARDIAN |
| 4   | SLOW_PATH_MANDATORY | Медленный путь через 1-8 обязателен (без исключений) |
| 5-7 | Резерв | |

### 3.4 reflex_cooldown (u8)

Минимальное количество Heartbeat-пульсов между двумя последовательными рефлексами от домена 9 для данного домена. Защита от "рефлексного шторма" — ситуации когда домен 9 непрерывно выдаёт предсказания не давая медленному пути завершить обработку.

Значение 0 — без ограничений.

### 3.5 max_concurrent_hints (u8)

Максимальное количество ассоциаций-подсказок, которые Arbiter может прикрепить к паттерну при передаче в данный домен. Ограничивает шум: если домен 9 нашёл 20 похожих следов, домен-процессор получит не больше чем max_concurrent_hints самых релевантных.

Значение 0 — подсказки отключены (эквивалент сброса бита HINTS_ENABLED).

### 3.6 feedback_weight_delta (u8)

Шаг изменения weight при обратной связи. Определяет насколько сильно усиливается или ослабляется след в EXPERIENCE(9) после сравнения рефлекса с результатом 1-8.

Шкала: 0..255 отображается на 0.0..1.0. Большое значение — быстрое обучение, но нестабильные рефлексы. Маленькое — медленное обучение, но устойчивые рефлексы.

## 4. Примеры конфигурации Arbiter для доменов Ashti_Core

### SUTRA (Домен 0) — Источник

```
reflex_threshold: 0       // Рефлексы не применимы к SUTRA
association_threshold: 0   // Подсказки не применимы
arbiter_flags: 0b00000000 // Всё отключено
```

SUTRA — вечная библиотека. Arbiter не взаимодействует с ней напрямую.

### EXPERIENCE (Домен 9) — Память

```
reflex_threshold: 0       // Сам является источником рефлексов, не получателем
association_threshold: 0
arbiter_flags: 0b00000100 // Только FEEDBACK_ENABLED
```

Домен 9 не получает рефлексы — он их выдаёт. Но обратная связь для него активна: он получает результаты сравнения и обновляет weight следов.

### EXECUTION (Домен 1) — Реализация решений

```
reflex_threshold: 140     // ~0.55 — умеренный порог, рефлексы важны для быстрой реакции
association_threshold: 50  // ~0.20 — подсказки принимаются легко
arbiter_flags: 0b00011111 // Всё включено, GUARDIAN обязателен, медленный путь обязателен
reflex_cooldown: 0        // Без ограничений — быстрая реакция критична
max_concurrent_hints: 3
feedback_weight_delta: 30  // ~0.12 — умеренная скорость обучения
```

### DREAM (Домен 7) — Фоновая оптимизация

```
reflex_threshold: 0       // Рефлексы отключены — DREAM работает медленно по природе
association_threshold: 25  // ~0.10 — принимает даже слабые подсказки (фоновый поиск)
arbiter_flags: 0b00010010 // HINTS_ENABLED + SLOW_PATH_MANDATORY
reflex_cooldown: 0
max_concurrent_hints: 8   // Много подсказок — DREAM ищет неожиданные связи
feedback_weight_delta: 10  // ~0.04 — медленное, но устойчивое обучение
```

### LOGIC (Домен 6) — Точные вычисления

```
reflex_threshold: 230     // ~0.90 — очень высокий порог, рефлекс только при абсолютной уверенности
association_threshold: 100 // ~0.39 — подсказки только если достаточно релевантны
arbiter_flags: 0b00011111 // Всё включено, GUARDIAN обязателен
reflex_cooldown: 5        // Не чаще чем раз в 5 пульсов — логика не торопится
max_concurrent_hints: 2   // Минимум шума
feedback_weight_delta: 50  // ~0.20 — если логика подтвердила, след усиливается заметно
```

### SHADOW (Домен 2) — Симуляция

```
reflex_threshold: 180     // ~0.71 — высокий порог, симуляция должна быть осторожной
association_threshold: 40  // ~0.16 — принимает подсказки от прошлых симуляций
arbiter_flags: 0b00010111 // Рефлексы + подсказки + обратная связь + обязательный медленный путь
reflex_cooldown: 2
max_concurrent_hints: 4
feedback_weight_delta: 20  // ~0.08 — осторожное обучение
```

### MAP (Домен 4) — Карта мира

```
reflex_threshold: 200     // ~0.78 — высокий порог, факты требуют уверенности
association_threshold: 80  // ~0.31 — подсказки только от достаточно надёжного опыта
arbiter_flags: 0b00011111
reflex_cooldown: 3
max_concurrent_hints: 2
feedback_weight_delta: 40  // ~0.16 — подтверждённые факты усиливаются заметно
```

### MAYA (Домен 10) — Интерфейс

```
reflex_threshold: 0       // MAYA не получает рефлексы — она их принимает от Arbiter напрямую
association_threshold: 0
arbiter_flags: 0b00000000 // Arbiter не маршрутизирует В MAYA — он маршрутизирует ЧЕРЕЗ MAYA
```

MAYA — точка сборки, а не процессор. Arbiter отправляет рефлексы и результаты в MAYA, но не настраивает её через эти поля.

## 6. Совместимость с V2.0

Единственное изменение: 8 байт `reserved_membrane: u64` заменены на 6 полей Arbiter (6 байт) + 2 байта резерва. Все остальные поля, их размеры и смещения не изменились. Общий размер структуры: 128 байт.

Домены, не взаимодействующие с Arbiter (SUTRA, MAYA), заполняют блок Arbiter нулями. Нулевые значения означают "всё отключено" — безопасное поведение по умолчанию.

---

## 7. Аналитика (перенесена из V2.0)

Упаковка структуры в 128 байт — компромисс. Все trade-offs из V2.0 остаются актуальными:

1. Коэффициенты Arbiter имеют 256 шагов (0..255 → 0.0..1.0). Для порогов этого достаточно.
2. Bloom-фильтры мембран (u64) требуют отдельного модуля для минимизации коллизий.
3. Давление и замедление времени ограничены u16 — потребуется логарифмическая шкала.
4. SIMD-оптимизации возможны при обходе массива DomainConfig.

## 8. История изменений

- **V2.1**: Блок Arbiter (8 байт) размещён в бывшем reserved_membrane. Добавлен StructuralRole::Experience. Примеры конфигурации для всех доменов Ashti_Core v2.0.
- **V2.0**: Data-Packed версия. Точно 128 байт без паддинга.
- **V1.x**: Ранние версии с другим размещением полей.
