## Проблема

Сейчас AXIOM принимает только `&str` через TextPerceptor (CLI, WebSocket).
Нет пути подать:
- Документы (.md, .pdf, .docx)
- Структурированные датасеты (готовые наборы знаний)
- Потоки из внешних источников

Кроме того, пропущен уровень абстракции: L0 примитивы (штрихи) → C0 графемы.
Crystal C0 уложен в config статически — не через механизм ввода.

## Компонент 1 — AxiomDataset формат (.axiom.yaml)

Единый шаблон для описания набора знаний.

```yaml
# Пример: axiom-dataset формат V1
metadata:
  name: "russian_philosophy_basics"
  version: "1.0"
  author: "chrnv"
  subsystem_hint: "abstractions"   # опционально — подсказка доминирующей подсистемы
  language: "ru"

inject:
  mode: "streaming"         # streaming (по одному) | batch (всё сразу)
  ticks_between: 50         # пауза между инъекциями (тиков)
  max_chunk_tokens: 200     # макс. длина чанка в символах

entries:
  - id: "plato_forms"
    content: >
      Платон утверждал что мир форм первичен по отношению к материальному миру.
      Идея — прообраз вещи, вечная и неизменная.
    tags: ["philosophy", "abstractions", "plato"]
    subsystem: "abstractions"

  - id: "kant_categories"
    content: >
      Кант выделял двенадцать категорий рассудка — априорные формы мышления.
    tags: ["philosophy", "logic", "kant"]
    subsystem: "logic"

  - id: "hegel_dialectic"
    content: >
      Диалектика Гегеля: тезис, антитезис, синтез. Противоречие — движущая сила развития.
    tags: ["philosophy", "abstractions", "hegel"]
```

**Что даёт:**
- Версионируемые, diff-уемые наборы знаний
- Контролируемый темп инъекции
- Возможность указать целевую подсистему (обходит Path 1 детекцию если нужно)
- Одинаковый формат для SUTRA и training_data

## Компонент 2 — .md ридер (FileIngester V1)

Markdown — самый простой формат, нет внешних зависимостей.

**Стратегии чанкинга:**

**Вариант А — по абзацам (простой)**
```
# Заголовок          → inject как отдельный чанк (высокая mass)
Абзац текста...      → inject как чанк
Следующий абзац...   → inject как чанк
```
+ Просто реализовать  
− Теряет структуру документа

**Вариант Б — по секциям (семантический)**
```
## Раздел 1          → inject заголовок + все абзацы раздела как связанный кластер
BondTokens: заголовок ↔ каждый абзац (COMPOSITION bond)
```
+ Сохраняет иерархию → FrameWeaver строит Frame на уровне секции  
− Сложнее, нужен markdown parser

**Вариант В — sliding window (перекрывающийся)**
```
Предложения 1-3 → чанк 1
Предложения 2-4 → чанк 2  (перекрытие = 1 предложение)
...
```
+ Максимальная связность, частые понятия co-activируются часто  
− Избыточность инъекций, медленнее

**Рекомендация:** начать с Вариант А, добавить Б как опцию в metadata.

## Компонент 3 — .pdf / .docx (будущее)

**Варианты:**

| Формат | Библиотека | Сложность | Качество |
|--------|-----------|-----------|---------|
| .pdf | `pdfium-render` (C bindings) | высокая | хорошее |
| .pdf | `pdf-extract` (pure Rust) | средняя | ограниченное (нет OCR) |
| .docx | `docx-rs` (pure Rust) | средняя | хорошее |
| .odt | `quick-xml` (парсинг вручную) | низкая | хорошее |

**Предложение:** добавить как `feature = "pdf"` / `feature = "docx"` в axiom-agent.
Основная кодовая база не тянет зависимости — только при явном включении.

## Компонент 4 — CLI интеграция

```
# Существующий CLI axiom-agent
:load path/to/file.md              # загрузить и инжектировать .md
:load path/to/dataset.axiom.yaml   # загрузить датасет
```

Реализуется в `crates/axiom-agent/src/channels/cli.rs` как новая мета-команда.

## Куда идут данные

### Путь 1 — прямо в SUTRA (живой режим)
```
чанк → TextPerceptor::perceive_and_bond → InjectToken + BondTokens → SUTRA → engine
```
Движок обрабатывает в реальном времени. Токены накапливают Experience, попадают в DREAM.

**Плюсы:** работает сразу, видно эффект в реальном времени  
**Минусы:** темп инъекции ограничен `ticks_between`, большие датасеты — долго

### Путь 2 — в training_data.jsonl (нейронный)
```
чанк → TextPerceptor → позиция + subsystem_hint → training_data.jsonl
```
Для обучения ReactivationDepthModel (axiom-neural).

**Плюсы:** быстро, не зависит от темпа тиков  
**Минусы:** не влияет на живой опыт движка сразу

### Путь 3 — обе стороны (рекомендован для датасетов)
Streaming в SUTRA + параллельная запись в training_data.

## Связь с Crystal (L0 / C0 / C1)

```
L0 примитивы (штрихи)    ← FOUND-TD-01 (не реализовано)
       ↓ composition bonds
C0 графемы (crystal)      ← реализовано ✅
       ↓ grow-seeds (FOUND-TD-02)
C1 биграммы               ← ждёт FileIngester
       ↓ crystallization
C2 слова / понятия        ← строятся через Experience
       ↓
C3+ структуры              ← FrameWeaver
```

FileIngester даёт реальный текстовый поток для проверки биграммной плотности C1.
Без него Seed Injection (C1) сложно тестировать — корпус слишком мал.
