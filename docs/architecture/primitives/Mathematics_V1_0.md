## 0. Что эта спека описывает

Mathematics — одна из подсистем познания в AXIOM. Параллельная Writing, но описывает не графику, а **математические структуры**: элементы, функции, отношения, операции, пределы, группы, фракталы.

Mathematics использует ту же инфраструктуру, что и Writing (SUTRA, токены, гравитация, оси, Shell), но имеет **свой собственный набор примитивов** — не сводимый к Primitives-7 письменности.

Соответствует архитектурному варианту "б": Mathematics накладывается на одно пространство SUTRA вместе с Writing и будущими подсистемами. Один и тот же токен (например, символ `+`) может одновременно быть Frame письменности (vline + hline) и якорем-операцией математики.

---

## 1. Главный принцип Mathematics

**Математика — это структура отношений между элементами.**

В отличие от письменности, где примитивы — графические формы, в математике примитивы — **типы математических объектов**. Это не "что нарисовано", а "какую роль играет в структуре".

Семь математических примитивов:

1. **Element** (элемент) — атом, число, точка в пространстве
2. **Function** (функция) — отображение, морфизм
3. **Relation** (отношение) — связь между элементами, эквивалентность
4. **Operation** (операция) — способ комбинировать (сложение, умножение, композиция)
5. **Limit** (предел) — непрерывность, бесконечное приближение
6. **Group** (группа) — замкнутое множество с операцией, симметрия
7. **Fractal** (фрактал) — самоподобие, рекурсия

Это эмпирическое наблюдение: любая математическая конструкция (число, вектор, доказательство, теорема, аксиома) сводится к комбинации этих семи типов.

## 2. Math-Primitives-7: семь математических атомов

Все живут в SUTRA (domain_id=100) как `STATE_LOCKED` якоря. Массы 160–200, как у Writing. Координаты — в положительной области `[0..32767]³`.

**Принципиально:** математические примитивы — это **отдельные якоря**, не Frame из графических примитивов. Символ `f` (графически) — это Frame письменности (vline + hook). Но **математический примитив "function"** — это отдельный якорь в SUTRA, к которому притягиваются токены, обозначающие функции (`f`, `g`, `sin`, `mapsto`).

### M1 — Element (элемент)

**Философия:** Атом, идентичность без структуры. Точка в пространстве, число, элемент множества. Не имеет частей, только существование. Архетип: "Я — это нечто".

```yaml
- id: "math_element"
  word: "элемент"
  aliases: ["element", "число", "точка", "атом", "сущность"]
  tags: ["primitive", "mathematics", "m1_element"]
  position: [5000, 0, 8000]
  # X=5000 средний Аполлон: элемент дискретен, упорядочен
  # Y=0 нейтральный: элемент сам по себе, без связей
  # Z=8000 Воля: элемент существует, заявляет идентичность
  shell: [0, 0, 0, 0, 25, 0, 0, 30]
  # L5=25 когнитивное (концепт идентичности)
  # L8=30 абстрактное (чистая идея элемента)
  mass: 200
  temperature: 4
  state: STATE_LOCKED
```

---

### M2 — Function (функция)

**Философия:** Отображение из одного множества в другое. Морфизм. Связь типа "вход → выход". Архетип: "Я преобразую".

```yaml
- id: "math_function"
  word: "функция"
  aliases: ["function", "морфизм", "отображение", "f", "mapping"]
  tags: ["primitive", "mathematics", "m2_function"]
  position: [12000, 10000, 9000]
  # X=12000 высокий Аполлон: функция структурирована
  # Y=10000 Эрос: функция связывает элементы (домен → кодомен)
  # Z=9000 Воля: функция действует
  shell: [0, 0, 5, 0, 25, 0, 10, 25]
  # L5=25 когнитивное, L8=25 абстрактное, L7=10 темпоральное (вход потом выход)
  mass: 190
  temperature: 5
  state: STATE_LOCKED
```

### M3 — Relation (отношение)

**Философия:** Связь между элементами без преобразования. Эквивалентность, неравенство, "больше чем". Архетип: "Я связываю две сущности".

```yaml
- id: "math_relation"
  word: "отношение"
  aliases: ["relation", "связь", "эквивалентность", "сравнение", "="]
  tags: ["primitive", "mathematics", "m3_relation"]
  position: [10000, 12000, 6000]
  # X=10000 Аполлон: отношение структурно
  # Y=12000 высокий Эрос: отношение = чистая связь
  # Z=6000 средняя Воля
  shell: [0, 0, 0, 0, 20, 5, 0, 30]
  # L8=30 абстрактное (отношение — чистая идея связи)
  mass: 185
  temperature: 4
  state: STATE_LOCKED
```

### M4 — Operation (операция)

**Философия:** Способ комбинировать элементы. Сложение, умножение, композиция функций, тензорное произведение. Архетип: "Я соединяю в новое".

```yaml
- id: "math_operation"
  word: "операция"
  aliases: ["operation", "сложение", "умножение", "композиция", "+"]
  tags: ["primitive", "mathematics", "m4_operation"]
  position: [9000, 8000, 11000]
  # X=9000 Аполлон: операция упорядочена
  # Y=8000 Эрос: операция = создание новой связи
  # Z=11000 высокая Воля: операция активно действует
  shell: [0, 0, 10, 0, 25, 0, 5, 20]
  # L3=10 моторное (операция действует), L5=25 когнитивное
  mass: 185
  temperature: 6
  state: STATE_LOCKED
```

### M5 — Limit (предел)

**Философия:** Бесконечное приближение к точке. Непрерывность. Граница, к которой стремится последовательность. Архетип: "Я приближаюсь, но не достигаю".

```yaml
- id: "math_limit"
  word: "предел"
  aliases: ["limit", "стремление", "непрерывность", "→", "схождение"]
  tags: ["primitive", "mathematics", "m5_limit"]
  position: [7000, 7000, 12000]
  # X=7000 средний Аполлон: предел структурен
  # Y=7000 Эрос: предел = вечное приближение
  # Z=12000 высокая Воля: устремление есть сила
  shell: [0, 0, 5, 0, 20, 0, 25, 20]
  # L7=25 темпоральное (предел = бесконечное во времени)
  mass: 180
  temperature: 7
  state: STATE_LOCKED
```

### M6 — Group (группа)

**Философия:** Замкнутое множество с операцией. Симметрия. Все возможные преобразования куба образуют группу. Архетип: "Я возвращаюсь к себе".

```yaml
- id: "math_group"
  word: "группа"
  aliases: ["group", "симметрия", "замкнутость", "цикл", "инвариант"]
  tags: ["primitive", "mathematics", "m6_group"]
  position: [14000, 9000, 10000]
  # X=14000 высокий Аполлон: группа = совершенная структура
  # Y=9000 Эрос: связь всех со всеми внутри группы
  # Z=10000 Воля: замкнутость есть устойчивость
  shell: [0, 0, 0, 0, 25, 5, 5, 30]
  # L8=30 абстрактное, L6=5 социальное (группа = коллектив)
  mass: 195
  temperature: 4
  state: STATE_LOCKED
```

## 3. Композиция математических объектов

### 3.1 Число = Element + конкретное значение

Натуральное число `5` — это токен, притягивающийся к якорю `math_element`. Конкретное значение (5) кодируется через `mass` или через позицию относительно якорей-чисел (которые будут добавлены отдельно как `math_number_1`, `math_number_2` и т.д. — но это уже расширение).

### 3.2 Вектор = три Element-токена в трёх осях

Вектор — Frame из трёх токенов-элементов, каждый привязан к одной из осей пространства.

### 3.3 Функция f(x) = Function-токен со связями

Функция `f(x) = x²` — это:
- Frame-анкер, привязанный к `math_function`
- Связь типа "domain" к области определения (множество X)
- Связь типа "codomain" к области значений (множество Y)
- Связь "rule" к Frame, описывающему правило (operation: умножение, повторённое)

### 3.4 Доказательство = последовательность Frame-ов

Доказательство — это Frame высокого уровня, объединяющий цепочку Frame-ов (аксиомы → леммы → теорема). Гравитация направляет его к якорю "истина" в Shell L8.

### 3.5 Уровни абстракции (см. `Абстракции.md`)

Математические объекты различаются **уровнем абстракции**:

| Уровень | Тип объекта | Якорь |
|---------|-------------|-------|
| 0 | Конкретное число (5) | math_element |
| 1 | Переменная (x) | math_element + flag |
| 2 | Функция (f) | math_function |
| 3 | Функтор (отображение между категориями) | math_function высшего порядка |
| 4 | Аксиоматическая система | Frame из якорей |
| 5 | Метаматематика (теорема о теоремах) | Frame высшего уровня |

Этот уровень кодируется через `mass` Frame-анкера или через специальный флаг.

## 4. Гравитация как математический вывод

Из `Математика.md`: математическое утверждение — это **траектория токена в гравитационном поле якорей**.

### Пример: проверка истинности

```
Якоря в SUTRA:
  "истина"   → position=[15000, 15000, 15000], mass=5000
  "ложь"     → position=[500, 500, 500],       mass=5000
  math_operation (сложение) → ...
  math_relation (равенство) → ...

Утверждение "2 + 2 = 5":
  - Создаётся Frame из токенов 2, 2, 5 и якорей operation, relation
  - Запускается гравитация
  - Числовые якоря "2+2" и "5" расходятся (4 ≠ 5)
  - Frame движется к якорю "ложь"

Утверждение "2 + 2 = 4":
  - Числовые якоря согласованы
  - Frame движется к якорю "истина"
```

Это работает потому, что:
1. Якоря "истина" и "ложь" — реальные точки в SUTRA с большой массой
2. Гравитация (apply_gravity_batch) уже реализована в SPACE_V6_0
3. Frame-анкеры FrameWeaver уже умеют формироваться

Что нужно добавить:
- Якоря "истина"/"ложь" (часть будущего `semantic_centers.yaml`)
- Якоря для чисел (часть `math_numbers.yaml`)
- Правила связывания (как именно Frame-анкер формируется для математического утверждения)

## 5. Связь с другими подсистемами

### 5.1 С Writing

Один и тот же символ может быть и Writing, и Mathematics:

- `+` графически = vline + hline (Writing Frame)
- `+` математически = математический примитив operation (`math_operation`)
- `=` графически = hline + hline (Writing Frame)  
- `=` математически = math_relation (равенство)

В контексте чтения **математического выражения** активна интерпретация Mathematics. Один и тот же графический Frame даёт разные ассоциации в разных контекстах. Это и есть архитектура "б".

### 5.2 С Time (будущая подсистема)

Математика и время пересекаются:
- Предел `lim` имеет временной аспект (бесконечное приближение)
- Производная — мгновенное изменение во времени
- Дифференциальное уравнение — связь функции и её изменения

Эти связи будут описаны в `Time_V1_0.md`.

### 5.3 С Abstractions (будущая подсистема)

Из `Абстракции.md`: математика — это часть лестницы абстракции. Math-Primitives отражают уровни 1–6 этой лестницы. Уровни 7 (конструктивизм) — это **способность создавать новые математические системы**, отдельный механизм.

## 9. Домены AshtiCore

| Примитив / процесс | Домен | Причина |
|---|---|---|
| M1–M7 (все примитивы) | **D0 SUTRA** | STATE_LOCKED якоря — эталон математических структур |
| Математические вычисления | **D6 LOGIC** | чистый дедуктивный процессор, цепочки причинно-следственных связей |
| Аксиомы, правила вывода | **D3 CODEX** | конституционный фильтр — математические законы неизменны |
| Подтверждённые теоремы | **D4 MAP** | факты, верифицированные через DREAM, встраиваются в карту |
| Недоказанные гипотезы | **D2 SHADOW** | работают в режиме Z=Гипотеза, потенциальная угроза/шанс |
| Пробные вычисления | **D5 PROBE** | генерирует пробные паттерны для проверки |
| Ложное утверждение (2+2=5) | **D8 VOID** | не резонирует ни с одним якорем → аномалия |
| Вывод доказательства | **D10 MAYA** | финализация и проекция результата |

**Путь математического утверждения:**
```
Утверждение → D6 LOGIC (вывод) → D3 CODEX (проверка аксиом)
                  ↓ (верно)              ↓ (противоречие)
             D4 MAP (факт)          D8 VOID / D2 SHADOW
```
