**Платформа:** Linux x86-64

## axiom-core — базовые структуры

| Операция | Время | Комментарий |
|----------|-------|-------------|
| `Token::new` | 24.4 ns | Инициализация 64-byte структуры |
| `Token::compute_resonance` | 7.6 ns | Арифметика на полях — быстро |
| `Token copy` (Copy trait) | 32.0 ns | Копирование 64 bytes в памяти |
| `Event::new` | 29.0 ns | Инициализация 64-byte структуры |
| `Connection::default` | 23.9 ns | Zero-init 64 bytes |
| `struct size assert` | ~0.9 ps | Компайл-тайм проверка, практически нулевая стоимость |

**Вывод:** Все базовые структуры укладываются в 25-32 ns на создание. Это соответствует одному кэш-промаху (L1 cache line = 64 bytes = размер структур). Выравнивание работает как ожидалось.

## axiom-runtime — AxiomEngine

### Инициализация

| Операция | Время | Комментарий |
|----------|-------|-------------|
| `AxiomEngine::new` | 151.3 ns | Пустой движок (пустые HashMap) |
| `add_domain × 10` | 1.36 ms | **136 µs на домен** — см. анализ ниже |

⚠️ `add_domain` занимает ~136 µs на домен. Это одноразовая операция при старте, но при 11 доменах AshtiCore = ~1.5 ms инициализации. Приемлемо для startup, но стоит отслеживать при дальнейшем росте.

### Обработка команд

| Операция | Время | Комментарий |
|----------|-------|-------------|
| `InjectToken` | 2.45 µs | Включает пересоздание engine (iter_batched) |
| `TickForward` / 0 токенов | 169.9 ns | Overhead цикла без нагрузки |
| `TickForward` / 10 токенов | 300 ns ⚠️ | Высокая вариация (210–477 ns) |
| `TickForward` / 50 токенов | 161.0 ns | |
| `TickForward` / 100 токенов | 159.4 ns | |

⚠️ **Аномалия TickForward:** с 50 и 100 токенами *быстрее*, чем с 10. Предположительно: при малом количестве токенов branch prediction хуже, или детекция коллизий имеет overhead даже при нулевых результатах. Требует детального профилирования при развитии физики.

### Snapshot

| Операция | Конфигурация | Время |
|----------|-------------|-------|
| `capture` | 0 токенов/домен | 178.7 ns |
| `capture` | 10 токенов/домен | 338.1 ns |
| `capture` | 100 токенов/домен | 1.82 µs ⚠️ высокая вариация |
| `restore_from` | 0 токенов/домен | 29.7 µs |
| `restore_from` | 10 токенов/домен | 30.5 µs |
| `restore_from` | 100 токенов/домен | 31.5 µs |

**Вывод capture:** Линейно по количеству токенов, аллокации Vec. 100 токенов × 3 домена = 1.8 µs — хорошо.

**Вывод restore_from:** Практически не зависит от количества токенов (29.7 → 31.5 µs). Доминирует пересоздание структур AxiomEngine (HashMap init). Это приемлемо — restore не горячий путь.

## Замеченные аномалии — требуют наблюдения

1. **resonance_search/500** — высокий разброс (5.6–7.7 µs). Граничный эффект HashMap при росте.
2. **TickForward/10 токенов** — высокая вариация (210–477 ns), быстрее при 50 и 100.
3. **capture/100** — вариация 0.97–3.65 µs. Аллокаторные паузы.
4. **add_domain** — 136 µs/домен. Одноразово, но отслеживать при масштабировании.
