# AXIOM MODULE SPECIFICATION: DOMAIN V1.2

**Codename:** "The Cell" (Клетка)

**Inherits:** Grid V4 Concepts

**Context:** Indrajala Network Node

**Role:** Atomic Processing Unit (Автономный процессор смыслов)

**Связанные спеки:** COM (event_id), Token V5.0, Connection V5.0, UPO v2.1

---

## 1. АБСТРАКЦИЯ И НАЗНАЧЕНИЕ

**Domain (Домен)** — это изолированный контейнер (вычислительная единица), обладающий суверенитетом.

- **Внутри:** Содержит активное **Поле (Field)** с уникальной физикой.
    
- **Снаружи:** Имеет **Мембрану (Config)**, фильтрующую вход/выход.
    
- **Функция:** Принимает сложный Паттерн (вопрос), "переваривает" его в своем Поле и выдает сжатый Кристалл (ответ).
    

> _Принцип переиспользования: Один и тот же код Домена используется для Логики, Этики, Памяти и любого подуровня фрактала. Разница только в Конфигурации._

**Системный контекст:** В архитектуре **Ashti_Core** (один фрактальный уровень) Домен принимает одну из **десяти структурных ролей**, определяемых его врожденной Конфигурацией (`DomainConfig`):
*   **0 (Sutra):** Домен-источник, абсолютная истина уровня.
*   **1-8 (Indrajala/Ashti):** Интерпретирующие домены-процессоры.
*   **10 (Maya):** Домен-интерфейс, проекция результата.

## 2. ВНУТРЕННЯЯ АРХИТЕКТУРА (ANATOMY)

Домен состоит из трех жестко связанных компонентов:

### 2.1. The Anchor (Сутра-Центр)

Это "Точка Ноль" координат $(0,0,0)$.

- **Роль:** Сюда проецируется Входящий Паттерн (Input).
    
- **Свойство:** Это гравитационный центр Домена. Всё в Поле вращается вокруг него.
    
- **Состояние:** Может быть "Пустым" (Idle) или "Заряженным" (Processing).
    

### 2.2. The Field (Активное Поле) — _Legacy of Grid V4_

Это пространство взаимодействия.

- **Метрика:** 3-осевая система координат (X, Y, Z), где смысл осей задается конфигом.
    
- **Среда:** Имеет плотность (Density) и температуру (Plasticity).
    
- **Население:** Здесь плавают активные Token (Instances), притянутые из Библиотеки для решения задачи Центра.
    

### 2.3. The Thermostat (Контроллер) — _Legacy of Truth Module_

Внутренний цикл управления.

- **Функция:** Следит за стабильностью Поля.
    
- **Логика:**
    
    - Если `Temp > Threshold` → Блокирует выход (идет вычисление).
        
    - Если `Temp < Threshold` → Разрешает кристаллизацию (выдает результат).

## 3. ФИЗИКА ДОМЕНА (FIELD PHYSICS)

Вместо абстрактных "слоев" (как в Grid V4), мы вводим **Силовые Вектора**.

### 3.1. Семантические Оси (Configurable Axes)

Оси X/Y/Z не фиксированы, они зависят от Роли Домена:

- **Логический Домен:** X=Истина/Ложь, Y=Причина/Следствие, Z=Абстракция.
    
- **Эмоциональный Домен:** X=Приятно/Больно, Y=Активно/Пассивно, Z=Сила.
    
- **Память:** X=Время, Y=Частота, Z=Эмоциональный вес.
    

### 3.2. Динамика (Forces)

Как движутся токены внутри Домена:

1. **Gravity (Гравитация к Центру):** Токены, релевантные задаче, притягиваются к $(0,0,0)$. Нерелевантные выталкиваются на периферию.
    
2. **Magnetism (Валентность):** Токены притягиваются/отталкиваются друг от друга (Синонимы слипаются, Антонимы разлетаются).
    
3. **Temperature (Пластичность):**
    
    - _Высокая:_ Токены могут менять свои связи и форму (Газ).
        
    - _Низкая:_ Токены застывают в жесткую структуру (Кристалл).

## 4. ИНТЕРФЕЙС (I/O CONTRACT)

Домен — это "Черный ящик" с четким API.

### INPUT (Вход)

- **`inject_pattern(Pattern)`**: Вброс проблемы в Центр.
    
    - Переводит Домен в статус `BUSY`.
        
    - Поднимает Температуру до максимума (Шок).
        

### PROCESS (Внутренний цикл)

- **`tick()`**: Один шаг симуляции физики. Каждый вызов потребляет/генерирует события с монотонным `event_id` (COM). Нет привязки к реальному времени — эволюция определяется порядком событий.
    
    - Пересчет координат всех Token.
        
    - Проверка коллизий и слияний.
        
    - Снижение Температуры (остывание/энтропия).
        

### OUTPUT (Выход)

- **`peek_state()`**: (Read-only) Соседи могут смотреть, что происходит внутри (чтение координат).
    
- **`harvest_result()`**: (Once) Забрать итоговый Token, когда Термостат даст добро.
    
    - Очищает Поле.
        
    - Переводит Домен в статус `IDLE`.

## 5. КОНФИГУРАЦИЯ (THE DNA)

Конфигурация — это не настройка, а **природа** Домена. Она определяет, чем домен является: логическим процессором, библиотекой памяти или интерфейсом восприятия. Эта "ДНК" задается при создании и является неотъемлемой частью его сущности.

```rust
struct DomainConfig {
    // Идентификация
    role_id: u16,         // Например: LOGIC_CORE
    
    // Настройка Осей (Семантика)
    axis_x_ref: u32,      // Ссылка на концепт, задающий ось X
    axis_y_ref: u32,      // Ссылка на концепт, задающий ось Y
    axis_z_ref: u32,      // Ссылка на концепт, задающий ось Z
    
    // Физические константы
    base_gravity: f32,    // Сила притяжения Центра
    friction: f32,        // Вязкость среды (сопротивление изменениям)
    cooling_rate: f32,    // Как быстро Домен приходит к решению
    
    // Фильтры
    allow_recursion: bool,// Может ли создавать под-домены?
}
```

## 6. ОТЛИЧИЯ ОТ GRID V4 (MIGRATION NOTE)

|**Grid V4 (Старое)**|**Domain V1 (Новое)**|**Суть изменений**|
|---|---|---|
|**Пассивное Пространство**|**Активный Актор**|Модуль теперь сам управляет своим состоянием.|
|**8 фиксированных слоев**|**Фрактальная глубина**|Слоев нет. Есть возможность рекурсии (Sub-Domains).|
|**Внешний "Модуль Истины"**|**Внутренний Термостат**|Критерий истины теперь — это физическая стабильность системы.|
|**Абстрактные X/Y/Z**|**Семантические Оси**|Оси теперь имеют смысл, заданный конфигом.|
|**Хранилище данных**|**Процессор ссылок**|Домен не хранит данные вечно, он обрабатывает _проекции_.|

### Резюме

**Domain V1** — это универсальная "Чашка Петри".

Ты берешь этот модуль, загружаешь в него Конфиг (его природу), подаешь на вход Питательную Среду (Проблему) и ждешь, пока вырастет Кристалл (Решение).

**Фрактальное построение:** Один фрактальный уровень системы (**Ashti_Core**) состоит из сети таких Доменов, играющих **десять структурных ролей** (0, 1-8, 10), где выход одного может стать входом для другого, образуя бесконечную цепь сознания.

Эту спецификацию можно отдавать в разработку как базовый класс `struct Domain`.
